# 配置指南

本项目有两种运行形态——**TUI 终端客户端** 和 **PWA 浏览器应用**，它们的配置方式截然不同。TUI 依赖 `.env` 文件与 `bsky-tui.config.json` 双重配置，而 PWA 完全通过浏览器内置的 `localStorage` 存储凭据。理解这些差异是顺利上手的关键。

## TUI 配置：双文件系统

TUI 的配置分为两个文件，遵循**敏感凭证与非凭证设置分离**的原则：

| 文件 | 内容 | 是否应提交到 Git |
|------|------|:---:|
| `.env` | Bluesky 帐号凭据 + LLM API Key | ❌ |
| `bsky-tui.config.json` | AI 模型、翻译偏好、场景覆盖等 | ⚠️ 建议 `.gitignore` |

[来源](packages/tui/src/cli.ts#L21-L30)

### 环境变量（.env）

`.env` 文件是 TUI 的**身份凭证**来源。以下是所有支持的变量：

| 变量 | 必需 | 默认值 | 说明 |
|------|:---:|--------|------|
| `BLUESKY_HANDLE` | ✅ | — | Bluesky 用户名（如 `user.bsky.social`） |
| `BLUESKY_APP_PASSWORD` | ✅ | — | Bluesky 应用密码（Settings → App Passwords 创建） |
| `LLM_API_KEY` | ✅ | — | LLM 供应商 API 密钥 |
| `LLM_BASE_URL` | ❌ | `https://api.deepseek.com` | OpenAI 兼容 API 的 base URL |
| `LLM_MODEL` | ❌ | `deepseek-v4-flash` | 模型名称 |
| `LLM_THINKING_ENABLED` | ❌ | `true` | 是否启用思考模式（显示推理过程） |
| `LLM_VISION_ENABLED` | ❌ | `false` | 是否启用视觉模式（图像分析，仅视觉模型支持） |
| `TRANSLATE_TARGET_LANG` | ❌ | `zh` | 翻译目标语言，可选 `zh/en/ja/ko/fr/de/es` |

[来源](packages/tui/src/cli.ts#L57-L67) | [来源](.env.example)

### .env 路径解析规则

TUI 启动时按**优先级顺序**查找 `.env` 文件：

```
monorepo 根目录 → 当前工作目录 (CWD)
```

具体实现是：

```typescript
const envPaths = [
  path.resolve(__dirname, '..', '..', '..', '.env'),   // monorepo 根目录
  path.resolve(process.cwd(), '.env'),                   // 当前工作目录
];
```

这意味着：如果你在 `bsky/` 根目录下运行 `pnpm tui`，它会先加载项目根目录下的 `.env`；如果你在 `packages/tui/` 中运行，它会尝试加载该子目录下的 `.env`。两个路径都尝试加载（`dotenv.config` 不存在的路径不会报错）。

[来源](packages/tui/src/cli.ts#L21-L30)

### config.json 配置详解（bsky-tui.config.json）

`bsky-tui.config.json` 存储非敏感的结构化配置。如果文件不存在，系统会使用默认值启动。

**完整字段表：**

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `targetLang` | `string` | `"zh"` | 翻译目标语言（`zh/en/ja/ko/fr/de/es`） |
| `translateMode` | `"simple" \| "json"` | `"simple"` | 翻译输出格式 |
| `aiConfig.baseUrl` | `string` | `"https://api.deepseek.com"` | LLM API 地址 |
| `aiConfig.model` | `string` | `"deepseek-v4-flash"` | 默认 AI 模型 |
| `aiConfig.provider` | `string` | `"deepseek"` | 供应商标识，用于查找对应 API Key |
| `aiConfig.reasoningStyle` | `string` | `"reasoning_content"` | 推理风格，详见[思考模式与视觉模式](思考模式与视觉模式.md) |
| `aiConfig.thinkingEnabled` | `boolean` | `true` | 是否显示思考过程 |
| `aiConfig.visionEnabled` | `boolean` | `false` | 是否启用图像识别 |
| `apiKeys` | `Record<string, string>` | `{}` | 多供应商 API Key 映射 |
| `scenarioModels.aiChat` | `string` | `""` | AI 聊天专用模型（`"provider/model"` 格式） |
| `scenarioModels.translate` | `string` | `""` | 翻译专用模型 |
| `scenarioModels.polish` | `string` | `""` | 润色专用模型 |

[来源](packages/tui/src/config/configStore.ts#L11-L25) | [来源](bsky-tui.config.example.json)

### config.json 路径解析

与 `.env` 一致，`bsky-tui.config.json` 也采用相同的查找顺序：

```
monorepo 根目录 → 当前工作目录 (CWD)
```

当保存配置时，默认写回 `CWD`（适合首次安装的场景）。

[来源](packages/tui/src/config/configStore.ts#L33-L38)

### API Key 解析优先级

TUI 启动时，AI 功能的 API Key 按以下优先级确定：

1. **环境变量 `LLM_API_KEY`** — 如果设置了，直接使用（兼容单供应商场景）
2. **`config.json` 中的 `apiKeys[providerId]`** — 根据当前 `aiConfig.provider` 查找对应 Key
3. **空字符串** — 两处都找不到时返回空

这意味着：如果你只有一个 LLM 供应商，最简单的做法是在 `.env` 中设置 `LLM_API_KEY`，无须在 `config.json` 中重复填写。

[来源](packages/tui/src/cli.ts#L69-L73)

### 交互式安装引导

如果 `.env` 不存在或缺少 `BLUESKY_HANDLE` / `BLUESKY_APP_PASSWORD`，TUI 会自动启动 **SetupWizard** 交互式引导。它会逐个询问：

1. Bluesky 用户名 → 应用密码 → 选择 LLM 供应商 → 选择模型 → 输入 API Key → 选择界面语言 → 配置场景模型 → 完成

向导完成后，凭据写入 `.env`，非敏感配置写入 `bsky-tui.config.json`。

[来源](packages/tui/src/components/SetupWizard.tsx#L89-L117)

## PWA 配置：无 .env，凭据存于浏览器

PWA 运行在浏览器环境中，**不存在文件系统**，因此完全不需要 `.env` 或 `config.json`。

### Bluesky 凭据

用户在登录页面输入用户名和密码后，鉴权成功返回的 `accessJwt` / `refreshJwt` 等会话信息被序列化存入 `localStorage`，键名为 `bsky_session`：

```typescript
localStorage.setItem('bsky_session', JSON.stringify({
  accessJwt, refreshJwt, handle, did
}));
```

后续页面刷新会从 `localStorage` 恢复会话，无需重复登录。登出时删除该键。

[来源](packages/pwa/src/hooks/useSessionPersistence.ts#L1-L25)

### AI 配置

PWA 的所有 AI 相关设置存储在 `localStorage` 的 `bsky_app_config` 键下。字段与 TUI 的 `config.json` 完全一致，通过设置面板（SettingsModal）进行修改。

| localStorage 键 | 存储内容 |
|:---|---|
| `bsky_session` | Bluesky 会话（JWT 令牌 + 用户标识） |
| `bsky_app_config` | AI 配置、翻译语言、界面主题、场景模型等 |

[来源](packages/pwa/src/hooks/useAppConfig.ts#L1-L54) | [来源](packages/pwa/src/components/SettingsModal.tsx#L92-L108)

### 设置面板

点击 PWA 界面的齿轮图标（⚙️），会打开 SettingsModal，提供四个 Tab：

- **Account** — 修改/更新 Bluesky 凭据
- **AI** — 选择供应商、模型、API Key、思考/视觉模式
- **Scenario** — 为 AI 聊天、翻译、润色各自分配不同的模型
- **General** — 翻译目标语言、UI 语言、深色模式

修改后点击保存，配置立即写入 `localStorage` 并生效。

[来源](packages/pwa/src/components/SettingsModal.tsx#L31-L401)

## 场景模型覆盖机制

无论是 TUI 还是 PWA，都支持**按场景分配不同模型**。`scenarioModels` 的三个字段使用 `"provider/model"` 格式，例如 `"deepseek/deepseek-v4-flash"` 或 `"mistral/mistral-medium"`。

| 场景 | 字段 | 说明 |
|------|------|------|
| AI 对话 | `aiChat` | 聊天与工具调用使用的模型 |
| 翻译 | `translate` | 翻译功能专用模型 |
| 润色 | `polish` | 草稿润色专用模型 |

如果某个场景留空（`""`），则回退到 `aiConfig.model` 中配置的默认模型。

解析逻辑：当模型字符串包含 `/` 时，拆分出 provider 和 model，查找对应供应商的 `baseUrl`、`reasoningStyle`，并从 `apiKeys` 中获取该供应商的 Key。

[来源](packages/pwa/src/App.tsx#L55-L77) | [来源](packages/tui/src/config/configStore.ts#L19-L24)

## 最佳实践

1. **TUI 新手**：直接删除 `.env` 和 `bsky-tui.config.json`，让 SetupWizard 引导你完成首配。
2. **多供应商场景**：在 `.env` 中只放 `BLUESKY_HANDLE` 和 `BLUESKY_APP_PASSWORD`，所有 API Key 放在 `config.json` 的 `apiKeys` 对象中。
3. **PWA 使用**：直接在浏览器中打开应用，登录后通过设置面板配置 AI，所有数据自动存于浏览器。
4. **不要将 `.env` 或 `config.json` 提交到 Git**，它们包含敏感凭据。
