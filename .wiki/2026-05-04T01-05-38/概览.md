# 概览

## 项目定位

**Bluesky Client** 是一个双端 Bluesky 社交网络客户端，同时提供 **TUI（终端界面）** 和 **PWA（渐进式 Web 应用）** 两种运行形态。它与官方客户端的核心差异在于深度集成了 AI 能力——不是简单的"搜索框里加个 AI 按钮"，而是一套完整的 AI 对话系统，赋予 LLM 31 个直接操作 Bluesky 平台的工具，让 AI 能读帖子、查资料、发帖、点赞、转发甚至分析图片。

项目采用 TypeScript 全栈开发，基于 AT Protocol 协议与 Bluesky 联邦网络通信，后端逻辑与前端渲染完全解耦。[来源](README.md)

```
@bsky/core ──→ @bsky/app ──→ @bsky/tui (终端)
                          └─→ @bsky/pwa (浏览器)
```
[来源](README.md#L7-L9)

## 双端对比：TUI vs PWA

同一套业务逻辑，两种截然不同的交互体验。

| 维度 | TUI（终端） | PWA（浏览器） |
|------|:-----------:|:------------:|
| **运行环境** | 终端（Windows Terminal、iTerm2、Kitty） | 现代浏览器（Chrome/Firefox/Safari） |
| **渲染引擎** | Ink（React-on-terminal） | React DOM + Tailwind CSS |
| **界面风格** | 纯文本终端渲染，键盘驱动 | 图形界面，鼠标/触摸驱动 |
| **导航方式** | 键盘快捷键（`t` 时间线、`p` 个人主页、`n` 通知……） | Hash 路由（`#/feed`、`#/search?q=...`） |
| **认证方式** | `.env` 配置文件（Handle + App Password） | 浏览器登录表单，会话存 localStorage |
| **AI 配置** | `.env` 环境变量 | 设置页面表单 |
| **聊天记录存储** | JSON 文件（FileChatStorage） | IndexedDB |
| **Markdown 渲染** | 自定义终端解析器 | react-markdown + remark-gfm + rehype-highlight |
| **图片显示** | CDN 终端渲染 + 本地保存 | Lightbox 预览 |
| **AI 会话 URL** | 不支持 | `#/ai?session=uuid` 持久化 |
| **滚动恢复** | 不支持 | 支持（Profile/Search/Bookmarks） |
| **虚拟滚动** | 视口预计算（postToLines） | @tanstack/react-virtual |
| **暗色模式** | 终端主题继承 | CSS 变量切换 |
| **SVG 图标** | 无 | 50+ lucide 风格图标 |
| **鼠标支持** | ANSI 鼠标跟踪（滚动/点击） | 浏览器原生 |

详细对比见 [用户界面：TUI 与 PWA](用户界面-tui-与-pwa.md)。[来源](README.md#features)

## 三层架构设计

项目严格遵循**分层依赖**原则：下层不依赖于上层。这是确保代码可维护、可测试的核心设计决策。

```
┌─────────────────────────────────────┐
│          @bsky/core                 │
│  纯 TypeScript，零 UI 依赖           │
│  BskyClient · AIAssistant · 31 个工具 │
│  · 类型定义 · 系统提示词 · Provider   │
└────────────┬────────────────────────┘
             │ 依赖注入
┌────────────▼────────────────────────┐
│          @bsky/app                  │
│  React Hooks + 纯 Store             │
│  useAuth · useTimeline · useThread  │
│  useAIChat · useTranslation · i18n  │
│  ChatStorage 接口 · Widget 注册     │
└────────────┬────────────────────────┘
             │
    ┌────────┴────────┐
    │                  │
┌───▼────────┐  ┌─────▼──────────┐
│ @bsky/tui  │  │  @bsky/pwa     │
│ Ink 终端   │  │  React DOM PWA │
│ 键盘驱动    │  │  Hash 路由     │
│ 视口渲染    │  │  Tailwind CSS  │
│ 无 DOM     │  │  虚拟滚动       │
└────────────┘  └────────────────┘
```

三个包之间的依赖关系清晰地反映在 `package.json` 中：

- **`@bsky/core`** 只依赖 `ky`（HTTP 客户端）和 `dotenv`——没有任何 UI 框架依赖。[来源](packages/core/package.json#L16-L17)
- **`@bsky/app`** 依赖 `@bsky/core` 和 `react`，提供 20+ 个共享 React Hooks 和纯 Store。[来源](packages/app/package.json#L13-L14)
- **`@bsky/tui`** 依赖 `@bsky/app` 和 `@bsky/core`，使用 Ink 渲染终端界面。[来源](packages/tui/package.json#L16-L20)
- **`@bsky/pwa`** 同样依赖 `@bsky/app` 和 `@bsky/core`，使用 React DOM + Vite + Tailwind。[来源](packages/pwa/package.json#L13-L17)

这种设计的精妙之处在于：`@bsky/core` 可以脱离任何 UI 独立测试；`@bsky/app` 的 Hooks 可以被 TUI 和 PWA **完全复用**，每个 Hook 只需写一次；TUI 和 PWA 作为纯渲染层，各自只负责将共享逻辑适配到终端的 Ink 和浏览器的 React DOM。[来源](docs/ARCHITECTURE.md#dependency-flow)

详细架构分析见 [三层架构设计](三层架构设计.md)。

## AI 集成：区别于官方客户端的核心价值

这是整个项目存在的原因。Bluesky 官方客户端没有 AI 功能，而本项目让 AI 成为真正的内容助手。

**AI 对话**是核心入口。用户可以在聊天界面中与 LLM（目前支持 DeepSeek、Mistral 等多供应商）进行自然语言交互，AI 可以调用 31 个 Bluesky 平台工具来执行操作——从查询帖子、读取资料到发帖、点赞、转发，全部通过对话完成。[来源](packages/core/src/ai/assistant.ts#L93)

这 31 个工具分为两类：
- **27 个只读工具**：搜索帖子、获取个人资料、浏览线程、查看时间线、获取通知、查看书签等
- **4 个写工具**：发帖、点赞、转发、关注

写操作触发的确认门控（Write Confirmation Gate）是安全设计的亮点——AI 不能不经用户确认就执行写操作，每次写操作都会暂停并弹出确认提示。[来源](README.md#ai-integration)

其他 AI 特性还包括：
- **流式输出**（SSE）：TUI 和 PWA 都使用 Server-Sent Events 实现逐 token 输出，实时显示 AI 思考过程
- **思考模式**：DeepSeek 的 `reasoning_content` 可见，以 💭 块展示
- **视觉模式**：通过 `view_image` 工具让 AI 分析图片内容
- **智能翻译**：7 种语言，simple/json 双模式输出，指数退避重试
- **AI 润色**：按风格要求润色帖子草稿

详细文档见 [AI 助手与工具调用系统](ai-助手与工具调用系统.md) 和 [流式输出与思考模式](流式输出与思考模式.md)。[来源](packages/core/src/index.ts#L22-L29)

## 快速上手

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
pnpm install
pnpm -r build
```

**启动 TUI**：配置 `.env` 后执行 `cd packages/tui && npx tsx src/cli.ts`
**启动 PWA**：执行 `cd packages/pwa && pnpm dev`，在浏览器打开 `http://localhost:5173`

完整步骤见 [快速开始](快速开始.md)。[来源](README.md#quick-start)
