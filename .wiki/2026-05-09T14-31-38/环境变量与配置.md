# 环境变量与配置

BSky 的配置系统采用 **双层架构**：`bsky-tui.config.json` 文件管理非敏感设置（模型、翻译、场景覆盖等），`.env` 文件管理凭证（Bluesky 密码、API Key）。**凭证不进入 JSON 文件**，这是隐私优先设计的直接体现。

双客户端使用不同的持久化策略——**TUI 依赖文件系统**（.env + JSON），**PWA 依赖浏览器 `localStorage`**[来源](packages/pwa/src/hooks/useAppConfig.ts#L46-L58)。两者共享一个名为 `@bsky/app` 的公共包，其中包含 `useAuth`、草稿存储、国际化等跨平台逻辑[来源](packages/app/src/index.ts#L1-L5)。

## 一、蓝图：配置项全景

| 配置项 | 存放位置 | 用途 | 必填 |
|---|---|---|---|
| `BLUESKY_HANDLE` | `.env` | Bluesky 用户名（例如 `your-handle.bsky.social`） | **是** |
| `BLUESKY_APP_PASSWORD` | `.env` | Bluesky 应用密码（**不是**账号密码） | **是** |
| `BLUESKY_PDS` | `.env` | 自定义 PDS 服务地址（留空则自动发现 `bsky.social`） | 否 |
| `LLM_API_KEY` | `.env`（优先）或 `bsky-tui.config.json` 的 `apiKeys` | AI 提供者的 API 密钥（单提供者场景用 `.env`） | 取决于提供者 |
| `TRANSLATE_TARGET_LANG` | `.env` | 翻译目标语言（`zh/en/ja/ko/fr/de/es`） | 否，默认 `zh` |
| 其余所有 AI 配置 | `bsky-tui.config.json`（TUI）/ `localStorage`（PWA） | 模型、提供者、场景覆盖、自定义提示词等 | 有默认值 |

[来源](.env.example#L1-L14) | [来源](packages/tui/src/cli.ts#L64-L93)

## 二、TUI 的配置读取流程

### 2.1 `.env` 加载

TUI 启动时通过 `dotenv` 加载 `.env` 文件，搜索路径按优先级从高到低为：

1. **Monorepo 根目录**：`packages/tui/../../../.env`（即项目根 `.env`）
2. **当前工作目录**：`process.cwd()/.env`

两个路径都会尝试加载，后者可以覆盖前者。如果缺少 `BLUESKY_HANDLE` 或 `BLUESKY_APP_PASSWORD`，TUI 会自动弹出 **SetupWizard** 交互式引导，逐步骤填写：Bluesky 凭证 → 可选 PDS 地址 → AI 提供者选择 → 模型选择（支持注册表模型或自定义模型）→ API Key → 场景模型覆盖 → 界面语言[来源](packages/tui/src/cli.ts#L21-L28)。

### 2.2 API Key 的解析规则

`resolveAiApiKey` 函数采用"环境变量优先"原则：

```
LLM_API_KEY（来自 .env）存在？  → 使用它
否且 config.apiKeys[providerId] 存在？ → 使用 per-provider key
否则 → 返回空字符串
```

这种设计让你可以：
- **单提供者**：只在 `.env` 写一个 `LLM_API_KEY` 即可
- **多提供者**：在 `bsky-tui.config.json` 的 `apiKeys` 中分别存放各提供者的密钥，`.env` 中不设 `LLM_API_KEY`

[来源](packages/tui/src/cli.ts#L55-L62)

### 2.3 `bsky-tui.config.json` 的结构

示例配置文件 `bsky-tui.config.example.json` 顶部通过 `$schema` 字段链接到 JSON Schema 定义，支持编辑器的自动补全和校验。它遵循以下 TypeScript 接口[来源](bsky-tui.config.example.json#L1-L22)：

```typescript
interface TuiConfig {
  targetLang: string;                         // 翻译目标语言
  translateMode: 'simple' | 'json';           // 翻译模式
  aiConfig: {
    baseUrl: string;                          // LLM API 地址
    model: string;                            // 模型名
    provider?: string;                        // 提供者 ID
    reasoningStyle?: 'reasoning_content'
                     | 'structured_content'
                     | 'none';                // 推理风格
    thinkingEnabled?: boolean;                // 是否启用思考过程
    visionEnabled?: boolean;                  // 是否启用视觉能力
    customSystemPrompt?: string;              // 自定义 AI 提示词
  };
  apiKeys: Record<string, string>;            // 按提供者存放的 API Key（如 deepseek、mistral）
  scenarioModels: {
    aiChat: string;                           // AI 聊天场景模型覆盖
    translate: string;                        // 翻译场景模型覆盖
    polish: string;                           // 润色场景模型覆盖
  };
  dmEmojis?: string[];                        // DM 自定义快捷表情
}
```

默认值文件是 `bsky-tui.config.example.json`，实际使用时应复制为 `bsky-tui.config.json`（已被 `.gitignore` 忽略）。加载时使用 **深度合并**（deep merge）：`aiConfig`、`scenarioModels`、`apiKeys` 三个子对象分别单独合并，未配置的字段保留默认值。[来源](packages/tui/src/config/configStore.ts#L62-L79)

### 2.4 配置文件的查找路径

与 `.env` 同理，`configStore.ts` 先在 monorepo 根目录查找，再回退到当前工作目录，都不存在时使用默认值。首次保存时写入当前工作目录。[来源](packages/tui/src/config/configStore.ts#L52-L60)

## 三、PWA：localStorage 配置

PWA 没有文件系统访问权限，配置全部托管在浏览器的 `localStorage` 中，key 为 `bsky_app_config`。其中 `aiConfig` 字段的类型 `AIConfig` 由 `@bsky/core` 统一定义，确保 TUI 和 PWA 的 AI 配置结构一致[来源](packages/core/src/ai/assistant.ts#L79-L88)。

```typescript
// packages/pwa/src/hooks/useAppConfig.ts
interface AppConfig {
  aiConfig: AIConfig;              // 与 @bsky/core 统一（含 apiKey、baseUrl、model 等）
  targetLang: string;
  translateMode: 'simple' | 'json';
  darkMode: boolean;               // PWA 独有：暗色模式
  thinkingEnabled: boolean;
  visionEnabled: boolean;
  apiKeys: Record<string, string>;
  scenarioModels: { aiChat, translate, polish };
  enabledWidgets: string[];        // PWA 独有：右侧 Widget 面板
  customSystemPrompt?: string;     // PWA 独有：自定义 AI 提示词
}
```

读取和写入分别由 `getAppConfig()` 和 `saveAppConfig()` 完成，配置变更通过 Settings UI 界面操作，无需编辑文件。Bluesky 凭证在 PWA 中通过登录表单输入，成功后 JWT 令牌由 `useSessionPersistence` 持久化到 `localStorage`（key `bsky_session`），而非以明文存储密码。[来源](packages/pwa/src/hooks/useAppConfig.ts#L26-L44) | [来源](packages/pwa/src/hooks/useSessionPersistence.ts#L1-L9) | [来源](packages/pwa/src/components/LoginPage.tsx#L20-L32)

## 四、多 AI 提供者切换

项目的 AI 引擎支持**多提供者注册表**，内置两个提供者，定义在 `packages/core/src/ai/providers.json`：

| 提供者 ID | 名称 | 基础 URL | 推理风格 |
|---|---|---|---|
| `deepseek` | DeepSeek | `https://api.deepseek.com` | `reasoning_content` |
| `mistral` | Mistral | `https://api.mistral.ai` | `structured_content` |

每个提供者下挂多个模型：

**DeepSeek 模型**：`deepseek-v4-flash`（默认，支持思考）、`deepseek-v4-pro`（支持思考）

**Mistral 模型**：`mistral-small-latest`（24B，支持思考+视觉）、`pixtral-large-latest`（视觉）、`mistral-medium-latest`（128B，视觉）、`ministral-3b-latest`（快速）

配置文件中的 `aiConfig.model` 和 `aiConfig.baseUrl` 与提供者的对应关系由 `providers.ts` 的工具函数维护：

- `getProviderById(id)` — 按 ID 查找提供者元信息
- `getProviderByBaseUrl(baseUrl)` — 按地址反向匹配
- `getModelInfo(providerId, modelId)` — 查询模型的 thinking/vision 能力
- `isCustomModel(providerId, modelId)` — 判断是否为注册表外的自定义模型
- `shouldSendThinkingParam(providerId)` — 仅 DeepSeek 需要发送非标准 `thinking` 参数

[来源](packages/core/src/ai/providers.json#L1-L24) | [来源](packages/core/src/ai/providers.ts#L26-L56)

### 如何切换提供者

**TUI**：编辑 `bsky-tui.config.json`，修改 `aiConfig.provider`、`aiConfig.baseUrl`、`aiConfig.model` 三个字段。例如从 DeepSeek 切到 Mistral：

```json
{
  "aiConfig": {
    "provider": "mistral",
    "baseUrl": "https://api.mistral.ai",
    "model": "mistral-small-latest",
    "reasoningStyle": "structured_content"
  },
  "apiKeys": {
    "mistral": "your-mistral-api-key"
  }
}
```

**PWA**：在 Settings UI 的下拉菜单中选择提供者和模型，无需手动编辑文件。

**SetupWizard**（TUI 首次启动）：按步骤交互选择提供者、模型、API Key 等，自动写入 `.env` 和 `bsky-tui.config.json`。

### 场景模型覆盖

`scenarioModels` 字段允许为 AI 聊天、翻译、润色三个场景分别指定不同的模型。值为空字符串时使用 `aiConfig.model`。格式为 `"provider/model"`（例如 `"mistral/pixtral-large-latest"`）。[来源](packages/tui/src/config/configStore.ts#L24-L28)

## 五、自定义 AI 提示词（customSystemPrompt）

`aiConfig.customSystemPrompt` 允许用户附加一段自定义指令到每条 AI 对话的系统提示词末尾。TUI 在 `bsky-tui.config.json` 中配置，PWA 在 Settings UI 的文本框中编辑，两者均持久化到各自的存储介质中。[来源](packages/tui/src/config/configStore.ts#L19) | [来源](packages/pwa/src/hooks/useAppConfig.ts#L23-L24) | [来源](packages/pwa/src/components/SettingsModal.tsx#L44)

> **使用场景**：让 AI 以特定角色（例如"你是一位热爱 Bluesky 的网络考古学家"）回答问题，或在每次对话中自动附加固定指令。

## 六、翻译语言支持

`TRANSLATE_TARGET_LANG`（.env）或 `targetLang`（JSON/localStorage）指定翻译目标语言，支持七种：`zh`（中文）、`en`（英文）、`ja`（日文）、`ko`（韩文）、`fr`（法文）、`de`（德文）、`es`（西班牙文）。`translateMode` 控制输出格式：`simple` 只输出翻译结果，`json` 输出双语对照 JSON。[来源](.env.example#L14) | [来源](bsky-tui.config.example.json#L3-L4)

## 七、DM 自定义快捷表情

TUI 和 PWA 的 DM 功能支持自定义快捷表情面板。TUI 中通过 `bsky-tui.config.json` 的 `dmEmojis` 字段配置（字符串数组），PWA 通过设置面板的 DM 表情配置界面操作。两者均持久化到各自的存储介质中。[来源](packages/tui/src/config/configStore.ts#L30) | [来源](packages/app/src/hooks/useDmEmojiConfig.ts#L12-L30)
