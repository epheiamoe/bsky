# 快速开始

两条路径，一个代码库。本文带你分别在 **5 分钟内** 启动 TUI（终端）和 PWA（浏览器）客户端。

---

## 前置条件

- **Node.js >= 18**
- **pnpm**（推荐 `npm i -g pnpm` 安装）
- 一个 **Bluesky 账号**（注册于 [bsky.app](https://bsky.app)）
- 一个 **Bluesky App Password**：进入 Bluesky 设置 → **App Passwords** → 新建一个，权限选择默认即可
- 一个 **LLM API Key**（如 [DeepSeek](https://platform.deepseek.com) / [OpenAI](https://platform.openai.com) / [Mistral](https://console.mistral.ai) 等）

> 如果你只想浏览时间线和发帖，**LLM API Key 不是必须的**——只是 AI 聊天/翻译/润色等功能无法使用。

[来源](packages/tui/src/cli.ts#L64-L67)

---

## 路径一：TUI（终端客户端）

TUI 是一个基于 **Ink**（React 终端渲染引擎）的全键盘操作界面，适合键盘党使用。

### 第 1 步：克隆仓库

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
```

### 第 2 步：安装依赖并构建

```bash
pnpm install
pnpm -r build
```

`pnpm -r build` 会依次构建 `@bsky/core` → `@bsky/app` → `@bsky/tui` 三个包。如果你只需启动开发服务器，也可以跳过建构建直接用 `cd packages/tui && pnpm dev`（`tsx` 会在运行时编译 TypeScript）。

[来源](packages/tui/package.json#L20-L20)

### 第 3 步：配置环境变量

复制示例环境变量文件：

```bash
cp .env.example .env
```

编辑 `.env`，填入你的 Bluesky 凭证：

```env
# Bluesky 账号（必填）
BLUESKY_HANDLE=your-handle.bsky.social
BLUESKY_APP_PASSWORD=your-app-password

# LLM API Key（AI 功能需要，也支持在 bsky-tui.config.json 中按提供商配置）
LLM_API_KEY=sk-your-api-key

# 自定义 PDS 主机（可选，留空则自动发现 bsky.social）
# BLUESKY_PDS=https://your-pds.example.com

# 翻译目标语言（可选，默认 zh）
TRANSLATE_TARGET_LANG=zh
```

此外，TUI 还会读取项目根目录下的 `bsky-tui.config.json`。该文件存放非凭证设置（如模型、提供商、按场景分配不同模型等）。文件不存在时自动使用默认配置。完整配置项见 [`bsky-tui.config.example.json`](bsky-tui.config.example.json) 和 [环境变量与配置](环境变量与配置.md)。

> **关于 API Key**：`LLM_API_KEY` 环境变量是全局后备方案。你也可以在 `bsky-tui.config.json` 中按提供商独立配置，例如为 DeepSeek 和 Mistral 分别设置不同的 key。SetupWizard 引导设置时也会写入 `bsky-tui.config.json` 中的 `apiKeys`。

> **安全提示**：`.env` 和 `bsky-tui.config.json` 包含你的凭据，已列入 `.gitignore`，**不要提交到 Git**。

[来源](packages/tui/src/config/configStore.ts#L33-L50) · [来源](packages/tui/src/cli.ts#L55-L62) · [来源](bsky-tui.config.example.json#L1-L22)

### 第 4 步：启动 TUI

```bash
cd packages/tui
pnpm dev
```

或者直接使用 `tsx`：

```bash
cd packages/tui
npx tsx src/cli.ts
```

如果 `.env` 中缺少 `BLUESKY_HANDLE` 或 `BLUESKY_APP_PASSWORD`，TUI 会自动进入 **设置向导（SetupWizard）**，引导你逐步完成：Bluesky 句柄 → App Password → PDS 主机（可选）→ LLM 提供商选择 → 模型选择（含自定义模型）→ API Key → 界面语言 → 按场景分配模型（AI 聊天/翻译/润色可分别指定不同模型）。

启动后的界面结构如下：

- **顶部状态栏**：当前视图名称、连接状态
- **主内容区**：时间线、帖子详情、AI 聊天等
- **底部栏**：快捷键提示 + 当前时间
- **raw mode 警告条**（如果终端不兼容 raw mode）

[来源](packages/tui/src/cli.ts#L64-L67) · [来源](packages/tui/src/cli.ts#L95-L116) · [来源](packages/tui/src/components/SetupWizard.tsx#L1-L320)

### 常见问题：raw mode 不兼容

部分终端（如 Windows 的 cmd.exe）不支持 **raw mode**（逐键输入模式）。程序启动时会自动检测：

```typescript
// packages/tui/src/cli.ts
let isRawMode = false;
try {
  const stdin = process.stdin as ReadStream;
  if (stdin.isTTY) {
    stdin.setRawMode(true);
    isRawMode = true;
  }
} catch {}
```

如果检测失败，程序会**自动降级**：创建一个模拟的输入流管道，并将原始 `stdin` 数据桥接过去。此时：

- 大部分键盘操作仍可工作
- 鼠标点击不会拦截文本选择
- 屏幕底部会显示橙色警告条：**"当前终端不支持 raw mode。请在 Windows Terminal / iTerm2 中运行。"**

**建议**使用 Windows Terminal、iTerm2 或任何现代终端模拟器以获得最佳体验。

[来源](packages/tui/src/cli.ts#L119-L144) · [来源](packages/app/src/i18n/locales/zh.ts)

## 路径二：PWA（浏览器客户端）

PWA 是一个基于 **React DOM + Tailwind CSS** 的浏览器界面，支持离线安装。**无需任何 `.env` 配置**——所有登录和 AI 设置都在浏览器内完成。

你也可以直接访问已部署的版本：**[ai-bsky.pages.dev](https://ai-bsky.pages.dev)**

### 第 1 步：克隆并安装

如果你已按 TUI 路径完成了克隆和安装，跳过此步。否则：

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
pnpm install
```

> PWA 不需要预先 `pnpm -r build`，Vite 开发服务器会自动处理依赖构建。

### 第 2 步：启动开发服务器

```bash
cd packages/pwa
pnpm dev
```

Vite 开发服务器默认在 `http://localhost:5173` 启动，并自动打开浏览器。

### 第 3 步：在浏览器中完成配置

PWA 没有 `.env`，所有配置通过浏览器 UI 完成：

1. **登录**：在登录页面输入你的 Bluesky 句柄（如 `user.bsky.social`）和 App Password。可选输入自定义 PDS 主机。登录后会话自动持久化到 `localStorage`，刷新页面**不会掉登录**。
2. **欢迎引导**（首次访问）：弹出 Welcome Card 引导你设置 AI Key，选择提供商并填写 API Key。可随时跳过。
3. **AI 配置**（可选）：进入设置页面，选择 LLM 提供商、模型，填写 API Key。支持按场景分配不同模型（AI 聊天/翻译/润色）。
4. **个性化**：切换语言（中文/English/日本語）、切换深色模式、开启/关闭右侧 Widget。

所有配置自动保存到浏览器的 `localStorage`，刷新页面不会丢失。

[来源](packages/pwa/src/hooks/useSessionPersistence.ts#L1-L27) · [来源](packages/pwa/src/hooks/useAppConfig.ts#L26-L44) · [来源](packages/pwa/src/components/LoginPage.tsx#L40-L135) · [来源](packages/pwa/src/App.tsx#L55-L241)

### 构建生产版本（可选）

```bash
cd packages/pwa
pnpm build
```

构建产物输出到 `packages/pwa/dist/`，可部署到 Cloudflare Pages、Netlify 等静态托管平台。详见 [PWA 部署指南](pwa-部署指南.md)。

## 配置对比速查

| 配置项 | TUI 配置方式 | PWA 配置方式 |
|--------|-------------|-------------|
| Bluesky 句柄 | `.env` 文件 | 浏览器 UI 登录 |
| Bluesky App Password | `.env` 文件 | 浏览器 UI 登录 |
| 自定义 PDS | `.env` 可选 `BLUESKY_PDS` | 浏览器 UI 登录页可选 |
| LLM API Key | `.env`（全局）或 `bsky-tui.config.json`（按提供商） | 浏览器 UI 设置页（按提供商） |
| AI 模型/提供商 | `bsky-tui.config.json` | 浏览器 UI 设置页 |
| 按场景分配模型 | SetupWizard 或 `bsky-tui.config.json` | 浏览器 UI 设置页 |
| 翻译目标语言 | `.env` 或 `bsky-tui.config.json` | 浏览器 UI 设置页 |
| 界面语言 | SetupWizard 选择 | 浏览器 UI 设置页 |
| 凭证存储位置 | 本地文件系统 | 浏览器 `localStorage`（会话持久化） |
| 自定义 AI 系统提示词 | `bsky-tui.config.json` | 浏览器 UI 设置页 |

详细配置说明见 [环境变量与配置](环境变量与配置.md)。
