# 快速开始

5 分钟内从零开始运行这个双界面 Bluesky 客户端。你只需要一个终端、一个 Bluesky 账号和一个 LLM API Key。

---

## 第一步：系统要求

确保你的开发环境满足以下条件：

| 工具 | 版本要求 |
|------|---------|
| **Node.js** | >= 18 |
| **pnpm** | 任意版本 |

验证方式：

```bash
node --version   # 应输出 v18.x.x 或更高
pnpm --version   # 应输出版本号（如 8.x.x）
```

如果尚未安装 pnpm，执行 `npm install -g pnpm`。

[来源](package.json#L13-L15)

---

## 第二步：克隆、安装并构建

```bash
# 克隆仓库
git clone https://github.com/epheiamoe/bsky.git
cd bsky

# 安装所有依赖（pnpm workspace 会自动处理内部包链接）
pnpm install

# 构建所有内部包（core → app → tui/pwa）
pnpm -r build
```

`pnpm -r build` 会按依赖顺序依次构建 `@bsky/core`、`@bsky/app`、`@bsky/tui` 和 `@bsky/pwa`。这一步是**必需的**——TUI 和 PWA 都依赖已编译的内部包。

构建完成后不会有报错就是成功。关于四个包的职责划分，见[包架构深度解析](包架构深度解析.md)。

[来源](package.json#L6-L8)

---

## 第三步：配置凭据

将环境变量模板复制为 `.env`：

```bash
cp .env.example .env
```

编辑 `.env`，至少填入三个必填项：

```env
# 你的 Bluesky 用户名（完整 handle）
BLUESKY_HANDLE=your-handle.bsky.social

# Bluesky 应用密码（不是登录密码）
# 在 Settings → App Passwords 中生成
BLUESKY_APP_PASSWORD=xxxx-xxxx-xxxx-xxxx

# LLM API Key（用于 AI 对话、翻译、润色）
LLM_API_KEY=sk-your-api-key
```

> **如何获取 Bluesky 应用密码？** 登录 bsky.app → Settings → Privacy and Security → App Passwords → 新建一个。应用密码通常形如 `xxxx-xxxx-xxxx-xxxx`。
>
> **LLM API Key？** 项目默认使用 DeepSeek（`api.deepseek.com`），你也可以改用 OpenAI、Mistral 等其他提供商——在 TUI 的 SetupWizard 中可选。

完整的配置项清单（含可选配置）见[环境变量与配置](环境变量与配置.md)。

[来源](.env.example#L1-L8)

## 第四步：启动 TUI（终端界面）

```bash
cd packages/tui && npx tsx src/cli.ts
```

**首次启动会自动进入 SetupWizard（配置向导）**，引导你依次完成：

1. **Bluesky Handle** — 输入你的用户名（如 `user.bsky.social`）
2. **Bluesky App Password** — 输入应用密码（不回显）
3. **LLM 提供商** — 用上下键选择（DeepSeek / OpenAI / Mistral 等）
4. **模型** — 选择具体模型（或用自定义模型名）
5. **API Key** — 输入该提供商对应的 API Key
6. **场景模型** — 可选，为翻译/润色指定不同模型
7. **语言** — 选择界面语言（中文 / English / 日本語）

完成后 SetupWizard 会自动写入 `.env` 文件和 `bsky-tui.config.json`，然后进入主界面。

如果直接进入主界面而不是向导，说明 `.env` 中已填写了所有必填信息。SetupWizard 的详细操作指南见 [TUI 终端界面入门](tui-终端界面入门.md)。

> **关于 `npx tsx`**：tsx 是一个 TypeScript 执行器，免去手动编译的步骤。项目已在 `devDependencies` 中声明了 tsx，`npx` 会自动使用本地版本。

[来源](packages/tui/src/cli.ts#L91-L108)，[来源](packages/tui/src/components/SetupWizard.tsx#L29-L43)

## 第五步：启动 PWA（网页界面）

```bash
cd packages/pwa && pnpm dev
```

Vite 开发服务器会在 **http://localhost:5173** 启动，并自动在浏览器中打开。

PWA **不需要 .env 文件**——所有凭据都在浏览器中输入：

- 点击登录表单，输入 Bluesky Handle + App Password
- 进入设置页填入 AI API Key 和模型参数
- 凭据仅存储在浏览器 localStorage 中

PWA 的使用指引见 [PWA 网页应用入门](pwa-网页应用入门.md)。

[来源](packages/pwa/vite.config.ts#L29-L31)，[来源](packages/pwa/package.json#L7)

## 验证：你已成功运行了什么？

启动成功后，你将拥有一个**完整的 Bluesky 客户端**，能浏览时间线、查看帖子、点赞转发，更强大的是集成了 AI 助手——可以直接在终端或浏览器里用自然语言操作 Bluesky（查帖子、搜用户、分析内容、写草稿、翻译……）。

背后的架构是四层 monorepo：

```
@bsky/core  →  @bsky/app  →  @bsky/tui (终端)
                            └→ @bsky/pwa (浏览器)
```

- `@bsky/core`：AT Protocol 客户端、AI 引擎、31 个工具——**0 UI 依赖**
- `@bsky/app`：共享 React Hooks、Store、i18n——**UI 无关**
- `@bsky/tui` / `@bsky/pwa`：纯渲染层，只负责画界面和转发用户输入

TUI 和 PWA 共享同一套业务逻辑。如果你想深入理解每一层，推荐阅读[包架构深度解析](包架构深度解析.md)。

[来源](README.md#L155-L171)
