# 组件系统

PWA 右侧面板的 Widget 系统采用 **注册-查询** 模式，与 React 状态管理解耦。所有 Widget 在 `@bsky/app` 层的模块级 Map 中集中注册，视图层通过 `viewType` 过滤查询，无需 Context Provider 或全局 Store。

---

## 注册表：`widgetRegistry.ts`

`widgetRegistry.ts` 维护一个模块级的 `Map<string, WidgetEntry>`，提供四个核心导出函数：

| 函数 | 签名 | 作用 |
|------|------|------|
| `registerWidget` | `(def: WidgetDefinition, render: (props: WidgetProps) => ReactNode) => void` | 注册一个 Widget，存入 Map |
| `getWidget` | `(id: string) => WidgetEntry \| undefined` | 按 id 查找 |
| `getWidgets` | `() => WidgetEntry[]` | 返回所有注册项 |
| `getWidgetsForView` | `(viewType: string) => WidgetEntry[]` | 按视图过滤，`views` 为空数组的 Widget 出现在所有视图 |

[来源](packages/app/src/hooks/widgetRegistry.ts#L42-L58)

### WidgetDefinition 接口

```typescript
interface WidgetDefinition {
  id: string;            // 唯一标识，如 'polish'、'aiChat'
  titleKey: string;      // i18n 键，如 'action.polish'、'ai.widgetTitle'
  icon: string;          // Icon 组件名称，如 'file-text'、'at-sign'
  views: string[];       // 可见视图列表；空数组 = 所有视图
  defaultOpen: boolean;  // 首次加载时是否默认启用
  headerButtons?: React.ComponentType<{ goTo, onClose }>;  // 可选的头部按钮组件
}
```

`views` 字段是视图约束的核心机制：设 `views: ['compose']` 则该 Widget **只**在 compose 视图可见；设 `views: []`（空数组）则在所有视图均可见。该逻辑在 `getWidgetsForView` 中实现——**空数组不过滤，包含才匹配**。

[来源](packages/app/src/hooks/widgetRegistry.ts#L18-L31)

### WidgetProps 与 WidgetContext

渲染函数接收 `WidgetProps`：

```typescript
interface WidgetProps {
  onClose: () => void;
  context?: WidgetContext;
}
```

`WidgetContext` 是跨视图的上下文注入通道，包含 `composeDraft`、`viewType`、`client`、`threadUri`、`goTo` 等字段。每个视图的 `Layout` 组件在渲染 `WidgetPanel` 时，根据当前 `AppView` 组装 context 对象传入。

[来源](packages/app/src/hooks/widgetRegistry.ts#L37-L40)

## 状态管理：`widgetStore.ts`

`widgetStore.ts` 维护两个模块级状态：**启用顺序列表**和 **AI Chat 会话桥**。

### 启用/禁用与排序

```typescript
let _order: string[] = [];  // 有序的启用 Widget id 列表
```

核心 API：

| 函数 | 行为 |
|------|------|
| `initEnabledWidgets(ids)` | 用指定数组覆盖 `_order`，保证顺序 |
| `enableWidget(id)` | 追加到末尾（若已存在则忽略） |
| `disableWidget(id)` | 从数组移除 |
| `toggleWidget(id)` | 切换启用状态，通知 `_onWidgetToggle` 回调 |
| `getEnabledWidgetIds()` | 返回 `_order` 副本 |
| `getEnabledWidgetsForView(viewType)` | 合并查询：先过滤视图，再标记 `enabled` 状态 |
| `isWidgetEnabled(id)` | 检查是否在 `_order` 中 |

所有变更操作（`toggleWidget`、`initEnabledWidgets`）都触发 `_onWidgetToggle` 回调，由 `Layout` 组件挂载该回调以同步写入 AppConfig（即 `localStorage`）。

[来源](packages/app/src/hooks/widgetStore.ts#L1-L58)

### Compose Draft 桥

`widgetStore.ts` 还维护两个特殊的模块级桥接变量：

- **`_composeDraft` / `_composeDraftSetter`**：让右侧面板的 Widget（如 PolishWidget）能读取和替换 Compose 页面的草稿文本。`setComposeDraftForWidgets` 由 Compose 页面在每次输入变更时调用，`replaceComposeDraft` 由 Widget 调用以写回。
- **`_focusedProfileActor`**：让 `ProfilePreviewWidget` 在 thread 视图中能够获取当前帖子的作者 handle，而无需直接从 Widget 内部解析 threadUri。

[来源](packages/app/src/hooks/widgetStore.ts#L60-L73)

## 六个内置 Widget

六个 Widget 在 `packages/pwa/src/App.tsx` 的 `useEffect` 中一次性注册：

```typescript
useEffect(() => {
  registerWidget({ id: 'polish', ... }, ...);
  registerWidget({ id: 'profilePreview', ... }, ...);
  registerWidget({ id: 'suggestedFollows', ... }, ...);
  registerWidget({ id: 'suggestedFeeds', ... }, ...);
  registerWidget({ id: 'trends', ... }, ...);
  registerWidget({ id: 'aiChat', ... }, ...);
}, []);
```

[来源](packages/pwa/src/App.tsx#L86-L130)

| Widget ID | 视图绑定 | `defaultOpen` | 数据源 | 功能简述 |
|-----------|----------|---------------|--------|----------|
| `polish` | `compose` 专用 | `true` | `polishDraft()` 调用 LLM | 读取草稿，输入要求，调用 AI 润色后提供复制/替换 |
| `profilePreview` | `thread` 专用 | `true` | `getPostThread` → `getProfile` | 显示当前帖子作者的头像、简介、关注数，支持关注/取关 |
| `suggestedFollows` | **所有视图** | `false` | `getSuggestedFollows` | 显示 8 个推荐关注，点击头像可导航至 profile，一键关注 |
| `suggestedFeeds` | **所有视图** | `false` | `getPopularFeedGenerators` | 显示 8 个热门 Feed，一键订阅（调用 `addFeed`） |
| `trends` | **所有视图** | `false` | `getTrends` | 显示 15 条趋势话题，点击跳转搜索页 |
| `aiChat` | **所有视图** | `false` | `useAIChat` + `IndexedDBChatStorage` | 完整 AI 对话界面（见下节） |

**视图绑定语义**：设 `views: ['compose']` 的 PolishWidget 只出现在 compose 视图的右侧面板；设 `views: []` 的四个 Widget 出现在 timeline、thread、profile、search 等所有视图。

### PolishWidget

从 `composeDraft`（优先通过 context，fallback 到模块级桥）获取当前草稿文本，用户输入要求（如"更正式""更幽默"），调用 `@bsky/core` 的 `polishDraft` 函数。返回结果后提供 **复制到剪贴板** 和 **替换草稿**（通过 `onComposeDraftChange` 或 `replaceComposeDraft`）两个操作。

[来源](packages/pwa/src/components/widgets/PolishWidget.tsx#L1-L109)

### ProfilePreviewWidget

接收 `threadUri`，通过 `client.getPostThread(uri, 0, 0)` 获取帖子作者，再调用 `client.getProfile(handle)` 获取完整资料。显示头像、显示名、简介、关注/粉丝/帖子数，并提供关注/取关按钮。

[来源](packages/pwa/src/components/widgets/ProfilePreviewWidget.tsx#L1-L104)

### SuggestedFollowsWidget / SuggestedFeedsWidget / TrendsWidget

三个数据 Widget 共享相同的生命周期模式：`useEffect` 中异步获取数据，loading 状态显示加载指示器，空数据或错误显示提示文本。TrendsWidget 点击趋势项会调用 `goTo({ type: 'search', query, searchTab: 'top' })` 导航到搜索页。

[来源](packages/pwa/src/components/widgets/SuggestedFollowsWidget.tsx#L1-L81)
[来源](packages/pwa/src/components/widgets/SuggestedFeedsWidget.tsx#L1-L77)
[来源](packages/pwa/src/components/widgets/TrendsWidget.tsx#L1-L57)

## 组件管理页面：ComponentsPage

`ComponentsPage` 是 PWA 中的一个全屏管理页面（可从侧边栏的"组件"入口进入），提供 Widget 的**排序**与**启用/禁用**功能。

### 排序

页面将已启用的 Widget 按 `_order` 顺序渲染，每个卡片提供上移/下移箭头。移动操作通过 `moveWidget(fromIdx, toIdx)` 实现：

1. 调用 `getEnabledWidgetIds()` 获取当前顺序数组
2. `splice` 移动元素位置
3. 调用 `initEnabledWidgets(ids)` 写回模块级状态
4. 调用 `saveAppConfig` 持久化到 AppConfig（localStorage）

### 启用/禁用

已启用区每个卡片右侧显示红色 **禁用** 按钮；页面底部"已关闭的组件"区域列出所有未启用的 Widget，每个显示 **启用** 按钮。点击调用 `toggleWidget(id)`，该函数：

1. 切换 `_order` 中包含/排除该 id
2. 触发 `_onWidgetToggle` 回调（由 Layout 挂载），同步写配置
3. 返回新的启用状态

### 视图限制提示

设 `views` 非空的 Widget（如 PolishWidget、ProfilePreviewWidget）在 ComponentsPage 中只显示简介和标签，不渲染实际组件内容，提示"请在对应页面查看"。

[来源](packages/pwa/src/components/ComponentsPage.tsx#L1-L145)

## WidgetPanel：右侧面板渲染

`WidgetPanel` 接收 `viewType`、`enabledIds` 和 `context`，执行两层过滤：

1. `getWidgetsForView(viewType)` → 获取当前视图可见的所有 Widget
2. 按 `enabledIds` 顺序保留已启用的，跳过未启用的

每个 Widget 渲染为带统一样式头部（图标 + 标题 + 上/下移箭头 + 关闭按钮）的卡片。Widget 的 `headerButtons` 组件（如 AIChatWidget 的"全屏打开"和"新建对话"按钮）渲染在箭头左侧。

默认情况下（首次加载，配置为空），`Layout` 的 `useEffect` 会自动启用所有 `defaultOpen: true` 的 Widget（即 PolishWidget 和 ProfilePreviewWidget），并将该配置持久化。

[来源](packages/pwa/src/components/Layout.tsx#L64-L82)
[来源](packages/pwa/src/components/WidgetPanel.tsx#L15-L84)

## AI Chat Widget：会话持久化与 `/view` 上下文注入

AIChatWidget 是功能最复杂的 Widget，它集成了完整的 AI 对话能力。

### 会话持久化：`_aiChatSessionId`

AIChatWidget 的会话 ID 通过 `widgetStore.ts` 的模块级变量 `_aiChatSessionId` 管理：

```typescript
let _aiChatSessionId = '';
```

关键流程：

1. **初始化**：`useMemo(() => getAIChatSessionId() || resetAIChatSession(), [widgetKey])`。当用户首次打开或点击"新建对话"时，调用 `resetAIChatSession()` 生成 `crypto.randomUUID()` 并写入模块级变量。
2. **共享**：`chatId` 被传递给 `useAIChat` hook 的 options 中，驱动 `IndexedDBChatStorage` 的读写键。
3. **跨页面跳转**：Widget 头部的"全屏打开"按钮通过 `goTo({ type: 'aiChat', sessionId: chatId })` 导航到 `AIChatPage`，该页面读取 `_aiChatSessionId` 恢复同一会话。

这种设计使得 Widget 中的对话与全屏 AI Chat 页面共享同一个会话，消息不会丢失。

[来源](packages/app/src/hooks/widgetStore.ts#L28-L36)
[来源](packages/pwa/src/components/widgets/AIChatWidget.tsx#L30-L36)

### `/view` 上下文注入命令

AIChatWidget 支持一个特殊的命令前缀 `/view`，用于将当前页面的上下文注入 AI 提示：

```typescript
const viewContext = useMemo(() => {
  const vt = context?.viewType;
  if (vt === 'thread' && context?.threadUri) return `帖子: ${context.threadUri}`;
  if (vt === 'profile' && context?.profileActor) return `用户: @${profileActor}`;
  if (vt === 'search' && context?.searchQuery) return `搜索: ${searchQuery}`;
  return null;
}, [...]);
```

当用户在输入框中输入以 `/view` 开头的内容时：

1. 输入框上方显示一个蓝色预览卡片，展示当前页面上下文（如 "📋 帖子: at://did:plc:xxx/app.bsky.feed.post/yyy"）
2. 发送时自动将 `/view` 前缀替换为 `<currently_viewing>...</currently_viewing>` XML 标签包裹的上下文信息
3. 用户消息中的标签部分在 UI 中呈现为蓝色上下文卡片，与正文分离

例如输入 `/view 这条帖子说了什么？`，实际发送给 AI 的是：

```
<currently_viewing>用户当前正在浏览: 帖子: at://...。这个信息可能有帮助，但如果用户没有要求你使用，请不要提及。</currently_viewing>
这条帖子说了什么？
```

UI 中会清晰区分显示上下文卡片和用户正文消息。

[来源](packages/pwa/src/components/widgets/AIChatWidget.tsx#L59-L79)

### 消息分组与紧凑渲染

AIChatWidget 将原始消息流（user / thinking / tool_call / tool_result / assistant）按视觉逻辑分组：

- **thinking** → `ThinkingCard`（可折叠，compact 模式）
- **tool_call + tool_result** → 合并为 `ToolCard`（显示工具名、参数、结果）
- **user** → 用户气泡（含 `/view` 上下文卡片分离）
- **assistant** → `AssistantMessage`（compact 模式）

这种分组避免了多轮工具调用时消息列表的视觉碎片化。

[来源](packages/pwa/src/components/widgets/AIChatWidget.tsx#L102-L127)

### Widget 可见性的自动管理

`Layout.tsx` 中还实现了一个特殊逻辑：当用户导航到全屏 AI Chat 页面（`currentView.type === 'aiChat'`）时，自动 `disableWidget('aiChat')` 移除右侧 Widget 中的 AI Chat 副本，避免消息被清除；离开时自动 `initEnabledWidgets(widgetOrderRef.current)` 恢复。

[来源](packages/pwa/src/components/Layout.tsx#L95-L109)
