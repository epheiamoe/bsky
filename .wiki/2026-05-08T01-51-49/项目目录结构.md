# 项目目录结构

```
bsky/
├── packages/              # 四层 monorepo 核心代码
│   ├── core/              #     Layer 0: 零 UI 依赖
│   ├── app/               #     Layer 1: React Hooks + 状态层
│   ├── tui/               #     Layer 2: 终端 UI
│   └── pwa/               #     Layer 3: 网页 UI
├── contracts/             # 架构契约（JSON Schema / 提示词）
├── docs/                  # 开发文档 Wiki
├── public/                # 静态资源（当前为空）
├── temp/                  # 临时文件
├── pnpm-workspace.yaml    # monorepo 工作区定义
├── tsconfig.base.json     # 共享 TypeScript 配置基座
└── .env.example → .env    # 环境变量模板
```

[来源](pnpm-workspace.yaml#L1-L4), [来源](tsconfig.base.json)

## packages/core/src — 零 UI 核心层

系统的心脏，**不依赖任何 UI 框架**。详见 [包架构深度解析](包架构深度解析.md) 和 [BskyClient 深度解析](bskyclient-深度解析.md)。

```
packages/core/src/
├── index.ts               # public API 聚合导出
├── at/                    # AT Protocol HTTP 客户端
│   ├── client.ts          #   BskyClient 主类（ky 实例 + JWT 刷新）
│   ├── feeds.ts           #   内置订阅源枚举（BUILTIN_FEEDS 等）
│   └── types.ts           #   类型定义（PostView、ProfileView 等）
└── ai/                    # AI 引擎
    ├── assistant.ts       #   AIAssistant 主类 + 翻译/润色辅助函数
    ├── tools.ts           #   31 个工具定义（ToolDescriptor 工厂）
    ├── prompts.ts         #   系统提示词分片（P_/PF_ 前缀约定）
    ├── providers.ts       #   多提供商注册表（DeepSeek、OpenAI 等）
    └── providers.json     #   提供商配置数据
```

### 关键职责速查

| 文件 | 一句话职责 |
|------|-----------|
| `at/client.ts` | 封装 Bluesky API 的所有 HTTP 调用，自动刷新 JWT |
| `ai/assistant.ts` | OpenAI 兼容接口调用、多轮工具循环、流式/非流式双模式 |
| `ai/tools.ts` | 定义 27 个读工具 + 4 个写工具，统一 ToolDescriptor 接口 |
| `ai/prompts.ts` | 集中管理所有系统提示词，支持上下文注入 |
| `ai/providers.ts` | 从 JSON 注册表加载模型配置，适配推理参数 |

[来源](packages/core/src/index.ts#L1-L93), [来源](packages/core/src/ai/tools.ts)

## packages/app/src — 共享 React 层

封装所有业务逻辑为 React Hooks 和纯状态 Store，**PWA 和 TUI 共享**。详见 [核心 Hooks 参考](核心-hooks-参考.md)。

```
packages/app/src/
├── index.ts               # public API 聚合导出
├── hooks/                 # 20+ 个 React Hooks
│   ├── useAIChat.ts       #   AI 对话管理（流式/编辑/撤销）
│   ├── useAuth.ts         #   登录态管理
│   ├── useTimeline.ts     #   时间线加载与分页
│   ├── useThread.ts       #   讨论串展平（FlatLine 格式）
│   ├── useCompose.ts      #   发帖编辑（草稿/媒体）
│   ├── useDrafts.ts       #   草稿箱管理
│   ├── useNavigation.ts   #   导航状态绑定
│   ├── usePostActions.ts  #   点赞/转帖/等操作
│   ├── useProfile.ts      #   个人资料加载
│   ├── useSearch.ts       #   搜索逻辑
│   ├── useNotifications.ts#   通知管理
│   ├── useBookmarks.ts    #   书签管理
│   ├── useLists.ts        #   列表管理（订阅源列表）
│   ├── useListDetail.ts   #   单个列表详情
│   ├── useConvoList.ts    #   私信会话列表
│   ├── useChatMessages.ts #   单会话消息流
│   ├── useTranslation.ts  #   翻译/润色集成
│   ├── useChatHistory.ts  #   聊天历史加载
│   ├── useActiveFeed.ts   #   当前订阅源跟踪
│   ├── useScrollRestore.ts#   滚动位置恢复
│   ├── widgetRegistry.ts  #   右侧面板 Widget 注册
│   └── widgetStore.ts     #   Widget 状态管理
├── state/                 # 纯函数状态（无 React 依赖）
│   ├── navigation.ts      #   导航状态机（AppView push/pop）
│   ├── feedConfig.ts      #   订阅源配置持久化
│   └── viewStateStore.ts  #   滚动/视图状态
├── stores/                # Zustand-like 状态 Store
│   ├── auth.ts            #   认证 Store
│   ├── postDetail.ts      #   帖子详情 Store
│   └── timeline.ts        #   时间线 Store
├── services/              # 存储抽象接口
│   ├── chatStorage.ts     #   ChatStorage 接口 + 文件实现
│   └── draftStorage.ts    #   DraftStorage 接口 + 文件实现
├── i18n/                  # 国际化系统
│   ├── index.ts           #   聚合导出 + availableLocales
│   ├── store.ts           #   单例 Store
│   ├── types.ts           #   类型定义
│   ├── useI18n.ts         #   React Hook
│   └── locales/           #   语言包
│       ├── zh.ts          #     中文
│       ├── en.ts          #     英文
│       └── ja.ts          #     日文
└── utils/
    └── imageUrl.ts        #   CDN 图片 URL 处理
```

**关键分层逻辑**：`state/` 目录下的文件是纯函数，可在任何 JS 环境运行；`hooks/` 依赖 React；`stores/` 提供类 Redux 的响应式存储。详见 [国际化系统](国际化系统.md) 和 [导航状态机](导航状态机.md)。

[来源](packages/app/src/index.ts#L1-L82), [来源](packages/app/src/state/navigation.ts)
