# 键盘快捷键完整参考

TUI 的键盘系统基于 Ink 框架的 `useInput` 钩子，采用 **多处理器链式调度** 架构。理解这个调度模型是掌握快捷键冲突管理的关键。

## 架构核心：5 个 `useInput` 的注册顺序与 Guard 规则

整个应用在组件树中注册了 **5 个 `useInput` 处理器**（外加 SetupWizard 的首运行专用处理器）。Ink 的行为是：**每次按键，按注册顺序依次调用所有处理器**，直到某个处理器未做 `return` 才继续传播。这意味着 **处理器顺序决定了快捷键优先级**。

| 序号 | 处理器所在组件 | 源码位置 | 激活条件 |
|------|---------------|---------|---------|
| 1 | `App.tsx`（全局） | `packages/tui/src/components/App.tsx#L217` | **始终激活**，处理 Tab、Esc、Ctrl+G、全局导航、Feed/Bookmarks/Lists 视图专用键 |
| 2 | `UnifiedThreadView` | `packages/tui/src/components/UnifiedThreadView.tsx#L68` | 仅当 `currentView.type === 'thread'` |
| 3 | `AIChatView`（聊天模式） | `packages/tui/src/components/AIChatView.tsx#L169` | 仅当 `!showHistory` |
| 4 | `AIChatView`（历史模式） | `packages/tui/src/components/AIChatView.tsx#L254` | 仅当 `showHistory` |
| 5 | `NotifView` | `packages/tui/src/components/NotifView.tsx#L20` | 仅当 `currentView.type === 'notifications'` |

此外，`SetupWizard` 在首运行时独占键盘[来源](packages/tui/src/components/SetupWizard.tsx#L104-L120)，App.tsx 还注册了一个原始 `process.stdin.on('data')` 处理器专用于鼠标滚轮追踪[来源](packages/tui/src/components/App.tsx#L920)。

**冲突管理规则**：

- **全局处理器（App.tsx）中：条件从上到下依次守卫**。例如：`Tab` → `Esc` → `aiChat + focusedPanel === 'ai'` → `showFeedConfig` → 箭头键 → `Enter` → `Ctrl+G` → `,`（设置）→ 导航键。每个条件匹配后立即 `return`，阻断后续传播。[来源](packages/tui/src/components/App.tsx#L217-L498)
- **视图级处理器通过 `currentView.type` 守卫**。例如 Thread 的处理器仅在 `currentView.type === 'thread'` 时激活[来源](packages/tui/src/components/UnifiedThreadView.tsx#L68-L69)，AIChatView 的 chat/history 两个处理器互斥守卫[来源](packages/tui/src/components/AIChatView.tsx#L170)(packages/tui/src/components/AIChatView.tsx#L255)。
- **Compose 和 Search 视图主动阻断全局导航键**：`currentView.type === 'compose'` 时，全局处理器在导航键段之前 `return`[来源](packages/tui/src/components/App.tsx#L298-L483)。
- **`focusedPanel === 'ai'` 时全局处理器提前 `return`**，将键盘完全交给 AI 聊天面板[来源](packages/tui/src/components/App.tsx#L245)。

## 全局键（所有视图通用）

### 高优先级键（Tab / Esc / Ctrl+G）

| 键 | 动作 | 说明 |
|----|------|------|
| `Tab` | 切换焦点面板 `main` ↔ `ai`；Compose 中逐个帖子 | 多个帖子时循环编辑索引 |
| `Esc` | 见下方按视图区分表 | 返回上一级 |
| `Ctrl+G` | `goTo({ type: 'aiChat' })` | 线程视图中携带当前帖子 URI 启动 AI 对话 |
| `,`（逗号） | 打开设置面板（.env 编辑器） | 全局 |

[来源](packages/tui/src/components/App.tsx#L219-L292)

### Esc 按视图的行为

| 视图 | 第一下 Esc | 第二下 Esc |
|------|-----------|-----------|
| aiChat + `focusedPanel === 'ai'` | 取消 AI 焦点 → `focusedPanel = 'main'` | `goBack()` |
| compose + 正在输入图片路径 | 取消图片输入 | `goBack()`（有内容时弹出保存草稿提示） |
| compose + 无图片输入 | `goBack()`（有内容时弹出保存草稿提示） | — |
| feed | 无操作 | — |
| thread / profile / notifications / search / bookmarks | `goBack()` | — |

[来源](packages/tui/src/components/App.tsx#L226-L243)

### 全局导航键

这些键在所有非 compose / 非 search 视图中有效：

| 键 | 动作 | 描述 |
|----|------|------|
| `t` | `goHome()` | 时间线 — 清空导航栈回到 feed |
| `n` | `goTo({ type: 'notifications' })` | 通知视图 |
| `p` | `goTo({ type: 'profile', actor: handle })` | 个人资料（当前用户） |
| `s` | `goTo({ type: 'search' })` | 搜索 |
| `a` | `goTo({ type: 'aiChat' })` | AI 对话（非 aiChat 视图时） |
| `c` | `goTo({ type: 'compose' })` | 发帖（非 thread 视图时） |
| `b` | `goTo({ type: 'bookmarks' })` | 书签 |
| `m` | `goTo({ type: 'dm' })` | 私信（feed 视图中 `m` 被重载为"加载更多"） |
| `L`（大写） | `goTo({ type: 'lists' })` | 列表 |
| `?` | `goTo({ type: 'about' })` | 关于页面 |

[来源](packages/tui/src/components/App.tsx#L488-L498)

### 全局键保留规则

以下键被永久保留，**不得**在任何视图中重新用作本地快捷键：

| 键 | 原因 |
|----|------|
| `t`, `n`, `p`, `s`, `a`, `c`, `b`, `m`, `L` | 全局导航 |
| `Esc` | 通用返回 |
| `Tab` | AI 焦点切换 |
| `Ctrl+G` | AI 对话启动器 |

新增视图快捷键时可选的可用键：`f`, `z`, `x`, `w`, `u`, `o`, `g`, `q`, `e`, `d`（书签除外）, `l`（thread/AI-history 除外）, `h`（thread 除外）, `y`（thread 除外）, `i`（compose 除外）, `,`（逗号）。[来源](docs/KEYBOARD.md#L66-L75)

## Compose 视图

键盘焦点委托给 `TextInput`（通过 `onSubmit`）。全局快捷键在此视图中被 **阻断**。

### 正常（文字）模式

| 键 | 动作 |
|----|------|
| `Enter` | 提交帖子（先检查 ALT 文本完整性） |
| `Esc` | 返回（若有内容弹出保存草稿提示） |
| `i` / `I` | 进入媒体路径输入模式（图片或视频，最多 4 图/1 视频） |
| `D` | 打开草稿列表（保存/加载/删除/同步） |
| `P` | 新增帖子到线程 |
| `X` | 从线程中移除当前帖子 |
| `f` / `F` | 进入润色需求输入模式 |
| `Tab` | 循环切换帖子编辑索引（多帖时） |

### 媒体路径输入模式

| 键 | 动作 |
|----|------|
| `Enter` | 校验 + 上传媒体，进入 ALT 文本输入 |
| `Esc` | 取消媒体输入 |

### ALT 文本输入模式

| 键 | 动作 |
|----|------|
| `Enter` | 确认 ALT 文本 → 保存媒体 |
| `Esc` | 跳过 ALT → 以空 ALT 保存媒体 |

### 润色需求输入模式

| 键 | 动作 |
|----|------|
| `Enter` | 提交需求 → 调用 AI 润色 |
| `Esc` | 取消润色 |

### 润色结果模式

| 键 | 动作 |
|----|------|
| `R` | 用润色结果替换当前帖子文字 |
| `C` | 复制结果到剪贴板（stderr） |
| `Esc` | 关闭结果 |

**底部提示行**：`Enter:发送 · Esc:取消 · i:媒体 · D:草稿 · f:润色 · P:加帖 · X:删帖`

## 新增快捷键流程

1. 检查[全局键保留规则](#全局键保留规则) — 不重用保留键
2. 检查[键冲突表](#键冲突表) — 避免与现有视图含义冲突
3. 在对应组件中添加 `useInput` / switch case
4. 更新 i18n 语言文件中的底部提示行（`keys.*` 键）
5. **更新本文档**
