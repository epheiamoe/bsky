# 环境变量与配置

本项目的配置体系有两种形态：**TUI（终端界面）** 依赖 `.env` 文件 + `bsky-tui.config.json`，**PWA（网页应用）** 完全在浏览器内完成。理解这套配置体系，是成功运行项目的第一步。

## 快速对照

| 配置项 | TUI 存储位置 | PWA 存储位置 |
|--------|-------------|-------------|
| Bluesky 凭证 | `.env` 文件 | 登录表单 → localStorage 会话 |
| AI API 密钥 | `.env` 或 `bsky-tui.config.json` | 设置页面 → localStorage |
| AI 模型/提供商 | `bsky-tui.config.json` | localStorage `bsky_app_config` |
| 翻译设置 | `bsky-tui.config.json` | localStorage `bsky_app_config` |
| 界面主题 | 终端固有 | localStorage `bsky_app_config` |

## 一、环境变量（.env 文件）

`.env` 文件位于项目根目录，存放**所有凭证类配置**。首次使用请复制模板：

```bash
cp .env.example .env
```

然后编辑 `.env`，填入你的真实信息：

| 变量名 | 是否必需 | 默认值 | 用途说明 |
|--------|---------|--------|---------|
| `BLUESKY_HANDLE` | ✅ 是 | — | 你的 Bluesky 用户名，如 `user.bsky.social` |
| `BLUESKY_APP_PASSWORD` | ✅ 是 | — | Bluesky **应用密码**（设置 → App Passwords 生成），**不是**登录密码 |
| `LLM_API_KEY` | ❌（TUI 必需） | — | OpenAI 兼容 API 的密钥。PWA 无需此变量，在设置页面填写 |
| `TRANSLATE_TARGET_LANG` | ❌ | `zh` | 翻译目标语言：`zh`/`en`/`ja`/`ko`/`fr`/`de`/`es` |

> ⚠️ **`LLM_API_KEY` 在 TUI 中优先级最高**：即使 `bsky-tui.config.json` 的 `apiKeys` 表中有其他密钥，环境变量也会覆盖它。PWA 完全不解 `LLM_API_KEY`。 [来源](packages/tui/src/cli.ts#L54-L61)

> 💡 **为什么 `.env` 只存凭证？** AI 的模型选择、提供商、思考模式等非敏感设置已从 `.env` 迁移到结构化的 JSON 配置文件中。这样分开管理更清晰，也便于按场景切换模型。

[来源](docs/ENV.md#L1-L31) · [来源](.env.example#L1-L13)

## 二、TUI 的结构化配置（bsky-tui.config.json）

TUI 使用 `bsky-tui.config.json` 管理所有非凭证设置。文件同样位于项目根目录，已被 `.gitignore` 排除。

模板见 `bsky-tui.config.example.json`，完整结构如下：

```json
{
  "targetLang": "zh",
  "translateMode": "simple",
  "aiConfig": {
    "baseUrl": "https://api.deepseek.com",
    "model": "deepseek-v4-flash",
    "provider": "deepseek",
    "reasoningStyle": "reasoning_content",
    "thinkingEnabled": true,
    "visionEnabled": false
  },
  "apiKeys": {
    "deepseek": "",
    "mistral": ""
  },
  "scenarioModels": {
    "aiChat": "",
    "translate": "",
    "polish": ""
  }
}
```

### 2.1 顶层字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `targetLang` | string | `"zh"` | 翻译目标语言 |
| `translateMode` | `"simple"` / `"json"` | `"simple"` | 翻译模式：纯文本 / 保留 JSON 结构 |
| `aiConfig` | object | — | AI 核心配置（见下方） |
| `apiKeys` | object | `{}` | 按提供商存储的多组 API 密钥，key 为提供商 ID |
| `scenarioModels` | object | — | 三个场景各自的模型覆盖 |

### 2.2 aiConfig 字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `baseUrl` | string | `"https://api.deepseek.com"` | OpenAI 兼容 API 的端点地址 |
| `model` | string | `"deepseek-v4-flash"` | 默认模型 ID |
| `provider` | string | `"deepseek"` | 提供商标识，用于查询 API 密钥和注册表 |
| `reasoningStyle` | string | `"reasoning_content"` | 推理风格：`reasoning_content` / `structured_content` / `none` |
| `thinkingEnabled` | boolean | `true` | 是否在流式输出时显示思考过程 |
| `visionEnabled` | boolean | `false` | 是否启用图片视觉识别能力 |

[来源](packages/tui/src/config/configStore.ts#L9-L47) · [来源](bsky-tui.config.example.json#L1-L22)

### 2.3 scenarioModels：场景级模型覆盖

`scenarioModels` 允许你为三个不同场景指定**不同的模型**，覆盖 `aiConfig.model` 的默认值：

| 场景 | 字段 | 格式示例 | 说明 |
|------|------|---------|------|
| AI 对话 | `aiChat` | `"deepseek/deepseek-chat"` | 与 AI 助手聊天时的模型 |
| 翻译 | `translate` | `"mistral/mistral-large"` | 翻译帖子时的模型 |
| 润色 | `polish` | `"deepseek/deepseek-v4-flash"` | 润色草稿时的模型 |

格式为 `"提供商ID/模型ID"`。留空（`""`）则使用 `aiConfig.model`。 [来源](packages/tui/src/cli.ts#L44-L52)

> 关于「提供商」和「模型注册表」的完整说明，参见 [多提供商支持与模型注册表](多提供商支持与模型注册表.md)。

## 三、PWA 的配置（localStorage `bsky_app_config`）

PWA 不读取任何本地文件。所有配置通过**浏览器设置界面**操作，存储在 localStorage 的 `bsky_app_config` 键中。

`AppConfig` 接口定义如下：

```typescript
interface AppConfig {
  aiConfig: AIConfig;                    // apiKey, baseUrl, model
  targetLang: string;                    // 默认 'zh'
  translateMode: 'simple' | 'json';     // 默认 'simple'
  darkMode: boolean;                     // 默认 false
  thinkingEnabled: boolean;              // 默认 true
  visionEnabled: boolean;                // 默认 false
  apiKeys: Record<string, string>;       // 按提供商存储
  scenarioModels: {
    aiChat: string;
    translate: string;
    polish: string;
  };
  enabledWidgets: string[];              // 右侧面板启用的 widget
}
```

[来源](packages/pwa/src/hooks/useAppConfig.ts#L5-L42)

PWA 的 `AIConfig` 包含 `apiKey` 字段（TUI 的 `aiConfig` 不含，因为 TUI 的 API 密钥从外部注入），其余字段与 TUI 的 `aiConfig` 一致。

## 四、TUI vs PWA：配置异同对比

| 对比维度 | TUI | PWA |
|---------|-----|-----|
| Bluesky 凭证来源 | `.env` 文件 | 登录表单输入 |
| Bluesky 凭证持久化 | 磁盘 `.env` 文件 | localStorage 会话令牌 |
| AI 密钥来源 | 环境变量 > `apiKeys` 表 | 设置页面 → `aiConfig.apiKey` |
| 配置编辑方式 | 编辑 `bsky-tui.config.json` | 设置页面 UI |
| 首次配置 | SetupWizard 向导（自动写文件） | 登录页 + 设置页 |
| 界面主题 | 无（终端固有） | `darkMode`（localStorage） |
| 右侧 Widget 状态 | 无（TUI 无 widget 面板） | `enabledWidgets`（localStorage） |
| 默认值机制 | 深度合并：读文件后用 `DEFAULT_TUI_CONFIG` 填充缺省字段 | 读 localStorage，缺失字段用 `DEFAULT_CONFIG` 填充 |

TUI 默认值合并的简化示意：

```
bsky-tui.config.json  →  DEFAULT_TUI_CONFIG  →  最终配置
  (仅部分字段)            (完整默认值)           (合并后完整)
```

[来源](packages/tui/src/config/configStore.ts#L59-L77) · [来源](packages/pwa/src/hooks/useAppConfig.ts#L44-L52)

**关于「首次配置向导」**，参见 [TUI 终端界面入门](tui-终端界面入门.md) 的 SetupWizard 部分。

## 五、凭证安全：金科玉律

这条规则对所有配置都适用，但对初学者来说最容易犯错，所以单独强调：

> **绝不将凭证硬编码到任何提交文件中。**

具体来说：

1. **`.env` 已被 `.gitignore` 排除**，你编辑后的真实凭证不会进入 Git
2. **`bsky-tui.config.json` 同样在 `.gitignore` 中**，其中 `apiKeys` 字段的密钥不会提交
3. **不要**创建 `env.local`、`config.json` 等非标准文件名提交到仓库
4. **不要**在代码中直接写 `apiKey: 'sk-xxx'` 这样的字面量
5. 多人协作时，只分发 `.env.example` 和 `bsky-tui.config.example.json`，让每个人填入自己的值
