# 包架构深度解析

## 四层分治：一条不可逆的依赖链

```
┌──────────────────────────────────────────────────┐
│                   @bsky/core                      │
│  纯 TypeScript，零 UI 依赖                        │
│  BskyClient · AIAssistant · createTools · types   │
└────────────┬─────────────────────────────────────┘
             │  依赖: 仅 ky + dotenv(dev)
             │
┌────────────▼─────────────────────────────────────┐
│                   @bsky/app                       │
│  React Hooks + 纯 Stores · PWA 可直接复用          │
│  useAuth · useTimeline · useThread · useAIChat    │
│  useCompose · useProfile · useTranslation          │
│  ChatStorage 接口 · 导航状态机 · Widget 注册表      │
└────────────┬─────────────────────────────────────┘
             │  依赖: @bsky/core + react(peer)
             │
             ├──────────────────────────────┐
             │                              │
┌────────────▼──────────┐    ┌──────────────▼───────────┐
│     @bsky/tui          │    │      @bsky/pwa            │
│  Ink/React 终端 UI     │    │  React DOM PWA            │
│                        │    │                           │
│  cli.ts → App.tsx      │    │  main.tsx → App.tsx       │
│  CJK 文本换行           │    │  Tailwind CSS             │
│  ANSI 鼠标追踪          │    │  Hash 路由                │
│  JSON 文件存储          │    │  IndexedDB 存储           │
│  Markdown 渲染          │    │  虚拟滚动 · HLS 视频      │
└────────────────────────┘    └───────────────────────────┘
```

**核心约束**：依赖箭头只朝一个方向。`core` 不知道 `app` 的存在，`app` 不知道 `tui` 和 `pwa` 的存在。任何试图从 `tui` 或 `pwa` 向 `app` 反向注入业务逻辑的行为，都是架构违规。 [来源](docs/ARCHITECTURE.md#L63-L86)

## 黄金法则：业务逻辑只写一次

这个架构最根本的设计原则可以用一句话概括：**任何业务逻辑，无论 TUI 还是 PWA 需要，都在 `core` 或 `app` 中实现一次，两个界面层只负责渲染**。

具体分工：

| 逻辑类别 | 归属包 | 理由 |
|---------|--------|------|
| AT Protocol HTTP 通信 | `core` | 协议无关，纯数据 |
| AI 多轮工具调用循环 | `core` | 无 UI 交互诉求 |
| 31 个 AI 工具定义 | `core` | JSON Schema + 纯函数 handler |
| Prompt 模板系统 | `core` | 纯字符串拼接，零依赖 |
| 登录认证与会话管理 | `core` → `app` | 认证逻辑在 core，React hook 在 app |
| 导航状态机 | `app` | 通用视图状态管理 |
| 发帖/回复草稿 | `app` | 纯数据状态 + Store 接口 |
| 翻译与润色 | `core` → `app` | AI 调用在 core，hook 在 app |
| 聊天记录存储接口 | `app` | 接口定义，具体实现在各界面层 |
| CJK 终端文本换行 | `tui` | 纯终端渲染问题 |
| Tailwind 响应式布局 | `pwa` | 纯浏览器渲染问题 |
| IndexedDB 存储实现 | `pwa` | 浏览器特有 API |

验证方式：在 `tui` 或 `pwa` 的源代码中搜索业务逻辑关键词（如"refreshSession"、"createTools"、"sendMessageStreaming"），应当找不到——这些全部在 `core` 中。 [来源](docs/ARCHITECTURE.md#L100-L104)

**依赖**：`ky`（HTTP 客户端） + `dotenv`（开发依赖） + `@types/node`（开发依赖）  
**零 UI 依赖**：无 React、无 Ink、无 DOM。 [来源](packages/core/package.json#L22-L25)

### 四大模块

| 模块 | 文件 | 职责 |
|------|------|------|
| **BskyClient** | `src/at/client.ts` | AT Protocol HTTP 客户端，认证与会话管理，JWT 自动刷新，30+ API 端点 |
| **AIAssistant** | `src/ai/assistant.ts` | OpenAI 兼容 API 调用，多轮工具调用循环，流式输出，翻译/润色 |
| **createTools** | `src/ai/tools.ts` | 31 个 AI 工具（ToolDescriptor）的定义工厂，含 JSON Schema + handler |
| **Types** | `src/at/types.ts` | 所有 AT Protocol TypeScript 类型定义 |

### 为什么 `core` 能零 UI 依赖？

因为它的所有功能都是"询问-应答"模式：
- `BskyClient.getTimeline()` → 返回 `PostView[]`
- `AIAssistant.chat()` → 异步迭代器产出 `ChatMessage`
- `createTools()` → 返回 `ToolDescriptor[]`

这些函数不关心数据怎么展示、用户怎么操作、键盘还是鼠标。它们只处理数据和协议。 [来源](docs/ARCHITECTURE.md#L100-L101)

### 架构模式：Store + Hook 双层

`app` 层的内部架构遵循一个简洁的模式：

```
┌───────────────────────────────────────┐
│   stores/                             │  纯状态逻辑
│   auth.ts, timeline.ts, postDetail.ts │  subscribe(fn) → unsubscribe
└──────────┬────────────────────────────┘
           │  store.subscribe(tick)
           │
┌──────────▼────────────────────────────┐
│   hooks/                              │  React 桥接
│   useAuth, useTimeline, usePostDetail │  useEffect → store.subscribe
└───────────────────────────────────────┘
```

**Store 模式**：每个 store 是一个普通 JavaScript 对象（非 class），包含状态数据、修改方法和 `_notify()` 通知机制。通过 `subscribe(fn) => unsubscribe` 暴露给外部。React Hook 在 `useEffect` 中调用 `store.subscribe(tick)`，当状态变化时 `tick` 触发 re-render。这是单监听器模型，无需 Redux 或 Zustand。 [来源](docs/PACKAGES.md#L85-L86)

### 为什么 `app` 是 PWA 就绪的？

因为 TUI 和 PWA 共享同一套 hooks。PWA 只需要写渲染组件（`PostCard.tsx`、`AIChatPage.tsx`），所有数据获取、状态管理、AI 对话逻辑都来自 `app`。TUI 也一样——`App.tsx` 中调用 `useTimeline()`、`useThread()` 等 hook，用 Ink 的 `<Text>` 和 `<Box>` 渲染输出。 [来源](docs/ARCHITECTURE.md#L102)

## Layer 2：两个渲染层

### `@bsky/tui` — 终端 UI

**依赖**：`@bsky/core` + `@bsky/app` + `ink` + `ink-text-input` + `ink-spinner`  
**入口**：`src/cli.ts` → `tsx src/cli.ts` [来源](packages/tui/package.json#L22-L30)

TUI 的职责仅限渲染和键盘交互：

| 文件 | 职责 |
|------|------|
| `src/cli.ts` | 环境准备、原始模式、渲染启动 |
| `src/components/App.tsx` | 视图路由器 + 全局键盘分发 + 布局计算 |
| `src/components/PostList.tsx` | Viewport 帖子列表 |
| `src/components/PostItem.tsx` | 单帖渲染（CJK 截断、gutter lines） |
| `src/components/UnifiedThreadView.tsx` | 讨论串（光标/焦点双模式） |
| `src/components/AIChatView.tsx` | AI 对话（历史、工具调用、自动保存） |
| `src/components/Sidebar.tsx` | 导航侧栏 |
| `src/components/NotifView.tsx` | 通知列表 |
| `src/components/ProfileView.tsx` | 用户档案 |
| `src/components/SearchView.tsx` | 搜索结果 |
| `src/components/SetupWizard.tsx` | 首次配置向导 |
| `src/utils/text.ts` | CJK 感知文本换行 |
| `src/utils/mouse.ts` | ANSI 鼠标追踪 |

TUI 特有的渲染工具（PWA 不需要）：

| 工具 | 位置 | PWA 等价方案 |
|------|------|-------------|
| `visualWidth(str)` | `text.ts` | CSS `word-wrap` |
| `wrapLines(text, cols, indent)` | `text.ts` | CSS `word-wrap` |
| `enableMouseTracking()` | `mouse.ts` | 浏览器 `scroll` 事件 |
| `parseMouseEvent(buf)` | `mouse.ts` | 浏览器 `scroll` 事件 |

[来源](docs/ARCHITECTURE.md#L88-L96)

### `@bsky/pwa` — 网页应用

**依赖**：`@bsky/core` + `@bsky/app` + `react-dom` + `react-markdown` + `remark-gfm` + `@tanstack/react-virtual` + `hls.js`  
**构建**：Vite + Tailwind CSS [来源](packages/pwa/package.json#L13-L26)

PWA 的职责同样仅限渲染和浏览器交互：

| 文件 | 职责 |
|------|------|
| `src/main.tsx` | `ReactDOM.createRoot` 入口 |
| `src/App.tsx` | Hash 路由分发 + 会话恢复 |
| `src/components/Layout.tsx` | 壳布局：Header + Sidebar + 内容区 |
| `src/components/FeedTimeline.tsx` | 无限滚动时间线（虚拟滚动） |
| `src/components/ThreadView.tsx` | 讨论串（嵌套缩进） |
| `src/components/PostCard.tsx` | 单帖卡片（媒体、Markdown、链接） |
| `src/components/AIChatPage.tsx` | AI 对话页（Markdown 渲染、工具卡片） |
| `src/components/ComposePage.tsx` | 发帖/回复编辑器（图片上传） |
| `src/components/LoginPage.tsx` | 登录表单 |
| `src/components/SettingsModal.tsx` | 设置弹窗 |
| `src/components/ConvoListPage.tsx` | DM 会话列表 |
| `src/components/DMChatPage.tsx` | DM 对话视图 |
| `src/hooks/useHashRouter.ts` | Hash 路由 (`#/feed`, `#/thread?uri=...`) |
| `src/hooks/useSessionPersistence.ts` | localStorage 会话持久化 |
| `src/services/indexeddb-chat-storage.ts` | IndexedDB 聊天存储实现 |
| `src/stubs/fs.ts` | Node `fs` 浏览器桩模块 |

PWA 特有的关键差异：
- **Hash 路由**：`useHashRouter` hook 编解码 `AppView` 为 `#/path` 格式 URL，通过 `history.pushState` + `popstate` 实现导航，适配静态托管。 [来源](docs/ARCHITECTURE.md#L111)
- **IndexedDB 存储**：实现 `ChatStorage` 接口，代替 TUI 的 JSON 文件存储。详见 [存储与持久化](存储与持久化.md)。
- **虚拟滚动**：使用 `@tanstack/react-virtual` 处理大量 feed 数据。

## 依赖流向总结

```
                       contracts/
                      /    |    \
                     ▼     ▼     ▼
    @bsky/core ────► @bsky/app
       │                  │
       │          ┌───────┼───────┐
       │          ▼       ▼       ▼
       └─────► @bsky/tui  @bsky/pwa
```

| 依赖关系 | 方向 | 证据 |
|---------|------|------|
| `core → 无` | 纯叶节点 | `package.json` 无 workspace 依赖 |
| `app → core` | 单向 | `package.json` 有 `@bsky/core: "workspace:*"` |
| `tui → core + app` | 单向 | `package.json` 有 `@bsky/core` + `@bsky/app` |
| `pwa → core + app` | 单向 | `package.json` 有 `@bsky/core` + `@bsky/app` |
| `tui ~/ pwa` | 无 | 互不依赖，仅共享 core + app |

[来源](packages/core/package.json#L22-L25)，[来源](packages/app/package.json#L20-L21)，[来源](packages/tui/package.json#L22-L24)，[来源](packages/pwa/package.json#L13-L15)
