# 快速开始

本文档带你在 10 分钟内从零搭建并运行 **双端客户端**——包括终端 TUI 和网页 PWA。你只需要一个 Bluesky 账号和基本的终端操作能力。

---

## 前置准备

确保你的开发环境中已安装以下工具：

| 工具 | 最低版本 | 检验命令 | 说明 |
|------|---------|---------|------|
| **Node.js** | 18+ | `node -v` | 项目的运行时基础 |
| **pnpm** | 最新 | `pnpm -v` | 包管理器，需全局安装 |

如果尚未安装 pnpm，运行以下命令：

```bash
npm install -g pnpm
```

项目在 `package.json` 中声明了 node 引擎要求，低于 18 的版本会安装失败。[来源](../package.json#L10-L12)

---

## 第一步：克隆仓库

从 GitHub 拉取源码到本地：

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
```

这是项目唯一的远程仓库地址。[来源](../README.md#L5-L12)

克隆后目录结构如下：

```
bsky/
├── packages/
│   ├── core/        # 核心逻辑层：API 封装、AI 引擎、工具系统
│   ├── app/         # React Hooks 层：状态管理、授权、数据订阅
│   ├── tui/         # 终端渲染层：基于 Ink (React-on-Terminal)
│   └── pwa/         # 浏览器渲染层：React DOM + Tailwind + PWA
├── pnpm-workspace.yaml
├── package.json
└── .env.example
```

关于各包的职责边界，详见 [三层架构设计](三层架构设计.md)。

---

## 第二步：安装依赖

在项目根目录执行：

```bash
pnpm install
```

该命令通过 `pnpm-workspace.yaml` 中定义的 workspace 配置，一次性安装 `packages/` 下所有子包以及 `contracts` 目录的依赖。[来源](../pnpm-workspace.yaml#L1-L3)

> **注意**：如果你在国内网络环境遇到安装缓慢或超时，可以配置 npm 镜像：
> ```bash
> pnpm config set registry https://registry.npmmirror.com
> ```

---

## 第三步：构建所有包

```bash
pnpm -r build
```

`-r` 标志表示递归（recursive）执行，对 workspace 中的每个包依次运行其 `build` 脚本：

- `@bsky/core` → TypeScript 编译输出 `dist/`
- `@bsky/app` → TypeScript 编译输出 `dist/`
- `@bsky/tui` → TypeScript 编译输出 `dist/`
- `@bsky/pwa` → TypeScript 编译 + Vite 打包输出 `dist/`

构建完成后，所有包的 TypeScript 源代码被编译为 JavaScript，供后续运行使用。[来源](../package.json#L6-L8)

---

## 第四步：运行 TUI 终端客户端

TUI 需要 Bluesky 凭据来登录。项目使用 `.env` 文件管理配置。

### 4.1 创建环境变量文件

从模板复制一份：

```bash
cp .env.example .env
```

### 4.2 填写凭据

用文本编辑器打开 `.env`，填入你的信息：

```env
# Bluesky 账号
BLUESKY_HANDLE=your-handle.bsky.social
BLUESKY_APP_PASSWORD=xxxx-xxxx-xxxx-xxxx

# LLM API（兼容 OpenAI 格式，推荐 DeepSeek）
LLM_API_KEY=sk-your-api-key
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-v4-flash

# AI 思考模式（显示推理过程）
LLM_THINKING_ENABLED=true

# AI 视觉模式（启用图片分析，需视觉模型支持）
LLM_VISION_ENABLED=false

# 翻译目标语言（zh/en/ja/ko/fr/de/es）
TRANSLATE_TARGET_LANG=zh
```

> **Bluesky 应用密码**：前往 Bluesky 设置 → **App Passwords** 生成专用密码，**不要使用你的登录密码**。

各字段的详细说明见 [环境变量与认证](环境变量与认证.md)。

### 4.3 启动 TUI

```bash
cd packages/tui && npx tsx src/cli.ts
```

首次运行且 `.env` 中凭据不完整时，会自动弹出 **SetupWizard** 交互式向导引导你填写配置。配置完成后，向导会自动将数据写入 `.env` 并启动主界面。[来源](../packages/tui/src/cli.ts#L55-L82)

启动成功后，你将看到终端界面，默认显示 **Following 时间线**。关于界面布局和快捷键，参见 [TUI 终端客户端入门](tui-终端客户端入门.md) 和 [键盘快捷键完整系统](键盘快捷键完整系统.md)。

## 第五步：运行 PWA 网页客户端

PWA 不需要 `.env` 文件——所有凭据通过浏览器端的登录表单输入，并存储在浏览器 `localStorage` 中。

### 5.1 启动开发服务器

```bash
cd packages/pwa && pnpm dev
```

Vite 开发服务器启动后，终端会输出访问地址，默认是：

```
http://localhost:5173
```

### 5.2 打开浏览器

在浏览器中访问上述地址，你会看到登录页面。

### 5.3 登录

填写你的 Bluesky 账号信息：

- **Handle**：例如 `your-handle.bsky.social`
- **App Password**：与应用密码一致

登录成功后自动跳转到时间线页面。你也可以在 **Settings** 页面配置 AI 相关参数（API Key、模型、思考模式等）。[来源](../README.md#L95-L100)

关于 PWA 的完整使用指引，参见 [PWA 网页客户端入门](pwa-网页客户端入门.md)。

## 运行效果速览

| 终端 | 浏览器 |
|------|--------|
| 键盘驱动的文本界面 | 鼠标+触摸的图形界面 |
| 多视图侧边栏导航 | 底部 Tab 栏导航 |
| `?` 键查看快捷键 | 点击图标触发操作 |
| 原生终端性能 | 响应式 + 可安装 PWA |

两端的核心功能完全对等：时间线浏览、帖子交互（点赞/转推/回复）、AI 对话、翻译、搜索、通知等。差异仅在于交互方式和渲染风格。[来源](../README.md#L47-L90)

## 验证安装成功

用以下检查清单确认一切正常：

- [ ] `pnpm -r build` 无报错，退出码为 0
- [ ] TUI 启动后显示时间线帖子列表
- [ ] PWA 访问 `localhost:5173` 显示登录表单
- [ ] 登录后可浏览帖子、切换 Feed 视图
