# 组件树与渲染层

## 入口与路由分发

整个 PWA 的渲染起点是 `main.tsx`，它在 DOM 根节点挂载 `<App />` 组件，同时注册 Service Worker。`App.tsx` 是这个单页应用的核心调度器，其渲染逻辑可分为三个层次：

```
main.tsx → <App /> → useHashRouter() → switch(currentView.type) → Page Component
```

**第一层：认证守卫。** `App` 组件通过 `useAuth()` 获取 `client`、`session` 和 `authLoading` 状态。当认证加载中，渲染加载动画；当未登录，直接渲染 `<LoginPage />`，阻断所有后续渲染。登录成功后将 `session` 持久化到 **localStorage**——这与 TUI 端通过 `.env` 文件加载凭据的方式不同。

**第二层：路由分发。** 使用 `useHashRouter()` 解析当前 URL hash 为类型安全的 `AppView` 对象，然后执行 `switch(currentView.type)`。每个 `case` 对应一个页面组件，将 `goTo`、`goBack`、`client` 等核心基础设施作为 props 传入。分发逻辑一目了然：

```typescript
switch (currentView.type) {
  case 'feed'          → <FeedTimeline />
  case 'thread'        → <ThreadView />
  case 'compose'       → <ComposePage />
  case 'profile'       → <ProfilePage />
  case 'notifications' → <NotifsPage />
  case 'search'        → <SearchPage />
  case 'aiChat'        → <AIChatPage />
  case 'bookmarks'     → <BookmarkPage />
}
```

**第三层：Layout 包裹。** 无论哪个页面，都被 `<Layout>` 组件包裹，后者提供统一的头部导航栏、桌面侧边栏和右侧面板骨架，构成三栏响应式布局。

[来源](packages/pwa/src/main.tsx) | [来源](packages/pwa/src/App.tsx#L1-L2)

关于 Hash 路由的编码/解码细节（`#/feed?feed=at://...`、`#/thread?uri=at://...` 等格式），详见 [导航路由与视图管理](导航路由与视图管理.md)。

## Layout：三栏响应式布局

`Layout` 组件是整个 PWA 的页面框架，由三个纵向区域组成：

| 区域 | 断点 | 功能 |
|------|------|------|
| **左侧 Sidebar** | `md:` 及以上固定显示，`md:` 以下为 overlay | 7-tab 导航 |
| **中间主内容区** | `max-w-content` 居中 | 渲染页面组件 |
| **右侧面板** | `lg:` 及以上固定显示 | AI 建议区（TODO） |

**头部导航栏**（`sticky top-0 z-50`）从左到右依次是：移动端汉堡菜单按钮、后退按钮（桌面端）、Astroid 图标 + "Bluesky" 品牌名、用户 handle、连接状态指示灯（绿/灰圆点）、设置按钮、主题切换按钮、注销按钮。头部使用 `backdrop-blur-md` 毛玻璃效果，背景为半透明白/黑。

**移动端侧边栏** 使用 `fixed inset-0` 遮罩层 + `w-64` 抽屉，点击遮罩关闭。桌面端侧边栏使用 `flex-col w-sidebar h-[calc(100vh-3rem)] sticky top-12`，固定在头部下方。

[来源](packages/pwa/src/components/Layout.tsx)

## Sidebar：7-Tab 导航 + 徽章

`Sidebar` 组件定义了一个 **`SIDEBAR_TABS` 常量数组**，包含 7 个导航项，按顺序排列：

1. **首页**（`home` 图标）→ `feed` 视图
2. **通知**（`bell` 图标）→ `notifications` 视图
3. **搜索**（`compass` 图标）→ `search` 视图
4. **书签**（`bookmark` 图标）→ `bookmarks` 视图
5. **个人资料**（`at-sign` 图标）→ `profile` 视图（需要 `handle`）
6. **AI 聊天**（`astroid-as-AI-Button` 图标）→ `aiChat` 视图
7. **发帖**（`pen-line` 图标）→ `compose` 视图

**高亮策略**：当前 `currentView.type` 与 tab type 匹配时，应用 `bg-primary/10 text-primary font-semibold border-primary` 样式；否则使用默认灰色文字。profile tab 特殊处理：始终匹配 `currentView.type === 'profile'`。

**徽章系统**：通知 tab 支持 `notifCount > 0` 时显示红色圆角 badge（`99+` 时截断）；发帖 tab 支持 `draftCount > 0` 时显示黄色 badge。两个条件独立，可同时出现。

**点击处理**：profile tab 点击时使用 `client.getHandle()` 获取当前用户 handle，构造 `{ type: 'profile', actor: handle }` 视图对象；其他 tab 直接构造 `{ type: tab.type }`。

[来源](packages/pwa/src/components/Sidebar.tsx)

## PostCard：双模式渲染引擎

`PostCard` 是整个帖子渲染的核心基础设施，支持两种数据源模式：

### PostView 模式 vs FlatLine 模式

```typescript
type PostCardProps = PostCardWithPost | PostCardWithLine;
// PostCardWithPost:  { post: PostView;  line?: never }
// PostCardWithLine:  { post?: never;   line: FlatLine }
```

这两种模式通过 TypeScript 联合类型实现互斥。组件内部根据运行时判断 `if (post) { ... } else if (line) { ... }` 提取字段。**PostView** 是完整的 API 返回格式，包含嵌套的 `record`、`author`、`embed` 等结构；**FlatLine** 是经过 `useThread` 扁平化后的轻量结构，省去了解析开销。

### 嵌入内容提取

组件内置三个提取函数：

1. **`extractEmbeds(post)`** — 从 `post.record.embed` 中提取图片、外部链接、视频和 GIF 标记。支持 `app.bsky.embed.video`（视频）、`app.bsky.embed.images`（图片）、`app.bsky.embed.external`（链接卡片）、`app.bsky.embed.recordWithMedia`（带媒体的引用帖）四种嵌入类型。GIF 通过 MIME type 包含 `gif` 判定。

2. **`extractQuotedPost(post)`** — 从 API 解析后的顶层 `post.embed.record` 中提取被引用的帖子信息，包括作者、文字、图片等。仅处理 `app.bsky.embed.record#view` 和 `app.bsky.embed.recordWithMedia#view` 类型。

3. **`linkifyText(text)`** — 使用正则 `/(https?:\/\/[^\s<>"']+|@[\w.-]+(?:\.[\w]{2,})+|#[\p{L}\p{N}_]+)/gu` 将文本中的 URL、@提及和 #标签 转换为可点击链接。@提及和 #标签使用 hash 路由导航，URL 使用 `target="_blank"` 新窗口打开。

### 图片网格与灯箱

**`ImageGrid`** 组件根据图片数量选择网格布局：

| 图片数 | 布局 |
|--------|------|
| 1 | 单列全宽 |
| 2 | 两列等宽 |
| 3 | 两列，前两张一行，第三张 `col-span-2` |
| 4+ | 两列，最多显示 4 张，超出显示 "+N" 提示 |

点击任意图片触发 **`ImageLightbox`**，使用 `createPortal` 渲染到 `document.body`，全屏黑色遮罩 + 居中图片 + 右上角关闭按钮。灯箱内点击遮罩关闭。

### 视频卡片

`PostCard` 中检测到 `video` 嵌入时，渲染 `<VideoCard />` 组件。该组件使用 **hls.js** 动态加载 HLS 视频流，降级方案是原生 `<video>` 的 `canPlayType('application/vnd.apple.mpegurl')`。首次点击封面图触发加载，支持 `aspectRatio` 保持宽高比。

[来源](packages/pwa/src/components/PostCard.tsx#L1-L260) | [来源](packages/pwa/src/components/VideoCard.tsx)

## FeedTimeline：IntersectionObserver 虚拟滚动

`FeedTimeline` 是首页时间线组件，集成了三个关键技术：

### 1. @tanstack/react-virtual 虚拟滚动

使用 `useVirtualizer` 创建虚拟列表，`estimateSize` 设定为 120px。渲染时只渲染可见区域（+ overscan 5 项）的帖子，其余部分用绝对定位 + `transform: translateY()` 占位。每个虚拟项通过 `virtualizer.measureElement` 动态测量实际高度。

### 2. IntersectionObserver 自动加载

在列表底部放置一个 `sentinelRef`（1px 高的 div），通过 `IntersectionObserver` 监听其可见性。当哨兵进入视口且 `cursor` 存在时，自动调用 `loadMore()`。`rootMargin: '200px'` 提供提前 200px 的预加载窗口。

### 3. 滚动位置恢复

`FeedTimeline` 接收 `initialScrollIndex` 和 `onFirstVisibleIndexChange` 回调。当从 ThreadView 返回 Feed 时，`App.tsx` 中的 `feedScrollIndexRef` 记录上次可见位置，通过 `virtualizer.scrollToIndex(index, { align: 'start' })` 恢复滚动位置。`requestAnimationFrame` 确保 DOM 更新后再执行滚动。

### 状态处理

组件覆盖所有边缘状态：**加载中**时渲染 5 个骨架屏（`SkeletonCard`）；**错误**时显示红色错误信息条；**无帖子**时显示 🕊️ 空状态图标；**加载更多中**显示主色旋转动画。

[来源](packages/pwa/src/components/FeedTimeline.tsx)

## ThreadView：展开树 + 翻译按钮

`ThreadView` 使用 `useThread(client, uri)` Hook 获取扁平化帖子列表（`flatLines`），然后在组件内部通过 `useMemo` 将其拆分为 **parent chain**（`depth < 0`）和 **replies**（`depth > 0`，且不超过 `focused.depth + 1`）。

### 三层视觉层次

1. **讨论源（parent chain）**：左侧缩进 + `opacity-60` 半透明 + 点击可导航到该帖子。按 depth 升序排列。

2. **主题帖/当前帖（focused）**：左侧 `border-l-4 border-primary` 加粗竖线高亮。展示完整信息：头像、显示名、handle、时间、关注按钮、帖子正文、引用帖、图片网格、视频、外部链接、媒体标签。

3. **回复**：根据 `line.depth` 计算左侧缩进 `marginLeft = Math.min((depth - 1) * 16, 48)`，每个回复渲染为 `<PostCard line={line} />`，可点击跳转到该回复的独立线程视图。

### 翻译功能

`ThreadView` 通过 `useTranslation` Hook 接入 AI 翻译能力。翻译按钮位于帖子操作栏中，点击后调用 `translate(focused.text, targetLang)`，结果渲染在帖子下方带语言标签的翻译面板中。再次点击可收起翻译面板。详细翻译流程见 [智能翻译与草稿润色](智能翻译与草稿润色.md)。

### 关注/取关

当 focused 帖子变化时，通过 `client.getProfile(focused.handle)` 获取用户 `viewer.following` 状态。关注按钮 UI 根据状态切换样式：未关注时 `bg-primary text-white`，已关注时 `border border-red-400 text-red-500`。

### 截断回复展开

当回复列表中存在 `isTruncation` 标志的 line 时，渲染 "展开更多回复" 按钮，调用 `expandReplies()` 加载更多。

[来源](packages/pwa/src/components/ThreadView.tsx)

## ComposePage：图片上传 + 草稿存储

`ComposePage` 是一个完整的帖子编辑器，处理新建、回复和引用帖三种场景。

### 状态管理

通过 `useCompose(client, goBack, goHome)` 获取 `draft`、`submitting`、`submit` 等核心状态。同时使用 `useDrafts()` 管理草稿列表。

### 媒体上传

**图片处理**：通过 `<input type="file" accept="image/*,video/*" multiple>` 选择文件。选择后触发 `compressImage(file)` 函数，对超大图片进行压缩（阈值 2MB），压缩结果通过 `compressInfo` 状态提示用户。最多允许 4 张图片，与视频互斥。

**视频处理**：视频文件检测依据 `file.type.startsWith('video/')`，Bluesky 限制单视频且不能混搭图片。上传前校验 `MAX_VIDEO_SIZE`（100MB）。

**上传流程**：提交时先上传媒体到 Bluesky（`client.uploadBlob()`），获取 `blobRef` 后构造 `ComposeMedia[]` 数组。上传过程中每个媒体项显示 `uploading: true` 的旋转动画遮罩，失败显示错误信息。

### 引用帖预览

当 `quoteUri` 存在时，通过 `client.getPostThread(quoteUri, 0, 0)` 获取引用帖详情，渲染为带作者头像、文字、图片缩略图的预览卡片。

### 草稿系统

用户离开编辑器时，如果内容非空，通过 `confirm()` 提示保存草稿。草稿存储结构为 `{ id, text, replyTo, quoteUri }`。编辑器头部可展开草稿列表，加载或删除已有草稿。草稿数量在 Sidebar 的发帖 tab 上以黄色 badge 显示。

### 字符计数

文本区域限制 `maxLength={300}`，底部实时显示 `charLen/300`，当超过 280 字符时字体变黄。

[来源](packages/pwa/src/components/ComposePage.tsx)

## AIChatPage：历史列表 + 流式 Markdown 渲染

`AIChatPage` 是 AI 助手的交互界面，分为三个区域。

### 左侧历史会话列表

固定宽度 `w-[280px]` 的侧边栏，通过 `useChatHistory(storage)` 从 **IndexedDB** 读取会话列表。支持：
- **新建会话**：通过 `goTo({ type: 'aiChat', sessionId: crypto.randomUUID() })` 创建新 ID
- **选择会话**：点击切换，当前活跃会话高亮（`bg-primary/10 border-primary/30`）
- **删除会话**：hover 时显示垃圾桶按钮，确认后从 IndexedDB 删除，若删除的是当前会话则跳转到空会话
- 每条会话显示标题、消息数、最后更新时间

关于 IndexedDB 存储方案的实现，详见 [哈希路由、存储与离线支持](哈希路由-存储与离线支持.md)。

### 中间主消息区域

通过 `useAIChat()` Hook 驱动，渲染四种消息类型：

| 角色 | 视觉样式 | 描述 |
|------|----------|------|
| `user` | 蓝色气泡 + 右对齐 | 用户消息，hover 显示编辑按钮 |
| `assistant` | 白色气泡 + 左对齐 | AI 回复，使用 `react-markdown` + `remark-gfm` 渲染 Markdown，hover 显示复制按钮 |
| `thinking` | 灰色斜体 + 左缩进 | AI 思考过程，`💭 Thinking:` 前缀 |
| `tool_call` / `tool_result` | 居中灰色小字 | 工具调用记录，显示工具名称和结果摘要 |

**流式渲染**：当 `stream: true` 时，AI 的回复以流式方式逐块追加到消息列表中，用户可实时看到 Markdown 渐显。流式中断时显示 "Stop" 按钮替代发送按钮。

**引导问题**：消息为空时，显示 `guidingQuestions` 列表作为快速入口。

**写操作确认**：当 AI 尝试执行写操作（发帖、关注等）时，`pendingConfirmation` 触发确认弹窗，用户可确认或拒绝。——详细的确认门控机制见 [AIAssistant 核心对话架构](aiassistant-核心对话架构.md)。

### 底部输入区

- 自动拉伸的 `<textarea>`（`minHeight: 42px, maxHeight: 128px`）
- 图片上传按钮（支持 `visionEnabled` 模式，上传图片后注入 `[图片: index=N]` 标记）
- 已选图片预览 + 移除按钮
- Enter 发送、Shift+Enter 换行

[来源](packages/pwa/src/components/AIChatPage.tsx)

关于 `useAIChat` Hook 的深层次细节——上下文注入、流式回调、会话持久化策略——参见 [useAIChat: 深度解析](useaichat-深度解析.md)。

## 组件树全景

```
<App>
  ├─ (authLoading) → 加载动画
  ├─ (!isLoggedIn) → <LoginPage />
  └─ <Layout>
       ├─ header (sticky)
       │   ├─ hamburger (mobile) / back (desktop)
       │   ├─ brand + handle + status dot
       │   └─ settings + theme toggle + logout
       ├─ <Sidebar /> (desktop: fixed left, mobile: overlay)
       ├─ <main>
       │   └─ switch(currentView.type):
       │       ├─ feed        → <FeedTimeline>
       │       │                ├─ <FeedHeader> (feed switch + refresh)
       │       │                └─ virtual scroll <PostCard> × N
       │       │                     └─ <PostActionsRow>
       │       ├─ thread      → <ThreadView>
       │       │                ├─ parent chain
       │       │                ├─ focused post + translate + follow
       │       │                └─ reply tree <PostCard> × N
       │       ├─ compose     → <ComposePage>
       │       │                ├─ quote preview
       │       │                ├─ textarea + image/video upload
       │       │                └─ drafts list
       │       ├─ profile     → <ProfilePage>
       │       │                ├─ banner + avatar + bio
       │       │                ├─ follow/followers lists
       │       │                └─ virtual scroll posts
       │       ├─ notifications→ <NotifsPage>
       │       ├─ search      → <SearchPage>
       │       │                ├─ tabs: top / latest / users / feeds
       │       │                └─ results lists
       │       ├─ aiChat      → <AIChatPage>
       │       │                ├─ sidebar (history list)
       │       │                ├─ message area (Markdown + streaming)
       │       │                └─ input area (image upload + send/stop)
       │       └─ bookmarks   → <BookmarkPage>
       ├─ right panel (lg: AI suggestions, TODO)
       └─ <SettingsModal> (portal)
```
