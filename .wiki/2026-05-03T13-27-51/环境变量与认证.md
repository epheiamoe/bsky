# 环境变量与认证

## 两种配置方式概览

这个项目采用**双端架构**（[三层架构设计](三层架构设计.md)），因此配置方式也分为两条路：

| 维度 | TUI 终端客户端 | PWA 网页客户端 |
|------|---------------|---------------|
| 存储介质 | `.env` 文本文件 | 浏览器 `localStorage` |
| 配置入口 | 手动编辑 `.env` 或运行安装向导 | 应用内"设置"弹窗 |
| Bluesky 凭据 | `BLUESKY_HANDLE` + `BLUESKY_APP_PASSWORD` | 登录表单输入，会话持久化 |
| LLM 配置 | `LLM_*` 系列变量 | 设置页 → AI 标签页 |
| 翻译语言 | `TRANSLATE_TARGET_LANG` | 设置页 → General 标签页 |
| 用户界面语言 | 终端适配 | 设置页 → General 标签页（`locale`） |

## TUI 端：.env 文件

### 启动时加载流程

TUI 启动时，`cli.ts` 会从两个位置尝试加载 `.env` 文件：

1. `packages/tui/../../../.env`（项目根目录）
2. `process.cwd()/.env`（当前工作目录）

找到任意一个即加载，然后调用 `getConfigFromEnv()` 将环境变量映射为 `AppConfig` 对象。

```typescript
// cli.ts 简化逻辑
for (const envPath of envPaths) {
  dotenv.config({ path: envPath });
}
// 然后检查 BLUESKY_HANDLE 和 BLUESKY_APP_PASSWORD
const handle = process.env.BLUESKY_HANDLE;
const password = process.env.BLUESKY_APP_PASSWORD;
if (!handle || !password) return null;  // 缺失凭据 → 启动安装向导
```

如果 `getConfigFromEnv()` 返回 `null`（即凭据不全），TUI 会自动启动 **SetupWizard**（[安装向导与环境校验](安装向导与环境校验.md)），引导你通过交互式表单填写配置，完成后自动写入 `.env` 文件并重新加载。

### 逐变量详解

以下对应项目根目录的 `.env.example` 模板文件：

```env
BLUESKY_HANDLE=your-handle.bsky.social
BLUESKY_APP_PASSWORD=your-app-password
LLM_API_KEY=sk-your-api-key
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-v4-flash
LLM_THINKING_ENABLED=true
LLM_VISION_ENABLED=false
TRANSLATE_TARGET_LANG=zh
```

#### 🔐 Bluesky 凭据（必需）

| 变量 | 含义 | 默认值 | 说明 |
|------|------|--------|------|
| `BLUESKY_HANDLE` | Bluesky 账号标识 | **无（必填）** | 形如 `user.bsky.social` 或自定义域名 |
| `BLUESKY_APP_PASSWORD` | 应用专用密码 | **无（必填）** | 不能使用主账号密码 |

> **如何获取 App Password？**
> 1. 登录 [bsky.app](https://bsky.app)
> 2. 进入 **Settings → App Passwords**
> 3. 点击 **Add App Password**，取名如 "bsky-tui"
> 4. 复制生成的密码（形如 `xxxx-xxxx-xxxx-xxxx`）
>
> PWA 登录页面底部也有同样的指引链接。
>
> ⚠️ App Password 是 **16 位带横杠的随机字符串**，不要用你的邮箱密码。

[来源](packages/pwa/src/components/LoginPage.tsx#L68-L76)

#### 🤖 LLM / AI 配置

| 变量 | 含义 | 默认值 | 可选值 |
|------|------|--------|--------|
| `LLM_API_KEY` | API 密钥 | **（不设则 AI 功能不可用）** | 任何 OpenAI 兼容 API 的密钥 |
| `LLM_BASE_URL` | API 地址 | `https://api.deepseek.com` | 任何兼容端点，如 `https://api.openai.com/v1` |
| `LLM_MODEL` | 模型名称 | `deepseek-v4-flash` | DeepSeek、GPT、Claude 等模型标识 |
| `LLM_THINKING_ENABLED` | 是否显示思考过程 | `true` | `true` / `false` |
| `LLM_VISION_ENABLED` | 是否启用图片分析 | `false` | `true` / `false`（需模型支持视觉） |

代码中的默认值回退逻辑：

```typescript
aiConfig: {
  apiKey: process.env.LLM_API_KEY || '',           // 空字符串 = 不配置
  baseUrl: process.env.LLM_BASE_URL || 'https://api.deepseek.com',
  model: process.env.LLM_MODEL || 'deepseek-v4-flash',
  thinkingEnabled: process.env.LLM_THINKING_ENABLED !== 'false',  // 默认 true
  visionEnabled: process.env.LLM_VISION_ENABLED === 'true',       // 默认 false
}
```

注意 `thinkingEnabled` 的判定逻辑：只要不是显式设为 `'false'`，就默认为 `true`。而 `visionEnabled` 只有严格等于 `'true'` 时才开启。

[来源](packages/tui/src/cli.ts#L48-L56)

#### 🌐 翻译语言

| 变量 | 含义 | 默认值 | 可选值 |
|------|------|--------|--------|
| `TRANSLATE_TARGET_LANG` | 翻译目标语言 | `zh`（中文） | `zh` `en` `ja` `ko` `fr` `de` `es` |

如果未设置此变量，代码默认使用 `'zh'`。PWA 的设置中还有 **翻译模式**（`translateMode`）选项：`simple`（简洁模式）和 `json`（JSON 模式，提供更结构化结果），详见[智能翻译与草稿润色](智能翻译与草稿润色.md)。

## PWA 端：浏览器 localStorage

PWA 不在浏览器中读取 `.env` 文件（浏览器没有文件系统权限），而是将所有配置存储在 **`localStorage`** 中。

### 配置持久化

通过 `useAppConfig` Hook 管理，存储在 `localStorage` 的 key 为 `bsky_app_config`：

```typescript
// packages/pwa/src/hooks/useAppConfig.ts
const CONFIG_KEY = 'bsky_app_config';

// 默认值与 .env.example 保持一致
const DEFAULT_CONFIG: AppConfig = {
  aiConfig: {
    apiKey: '',
    baseUrl: 'https://api.deepseek.com',
    model: 'deepseek-v4-flash',
  },
  targetLang: 'zh',
  translateMode: 'simple',
  darkMode: false,
  thinkingEnabled: true,
  visionEnabled: false,
};
```

[来源](packages/pwa/src/hooks/useAppConfig.ts#L6-L25)

### 配置入口：设置弹窗

PWA 的 `SettingsModal` 提供三个标签页：

1. **Bluesky 标签** — 重新登录（更新 handle/password）、登出
2. **AI 标签** — API Key、Base URL、Model、思考模式开关、视觉模式开关
3. **General 标签** — 翻译目标语言、翻译模式（simple/json）、UI 界面语言、深色模式

点击"保存"按钮后调用 `updateAppConfig()` 将配置写入 `localStorage`：

```typescript
const saveAi = () => {
  const updated = {
    ...config,
    thinkingEnabled,
    visionEnabled,
    aiConfig: { apiKey, baseUrl, model },
  };
  updateAppConfig(updated);   // 写入 localStorage
  onConfigChange(updated);    // 通知父组件
};
```

[来源](packages/pwa/src/components/SettingsModal.tsx#L160-L168) · [来源](packages/pwa/src/hooks/useAppConfig.ts#L38-L43)

### 与 TUI 的关键差异

| 特性 | TUI | PWA |
|------|-----|-----|
| Bluesky 凭据 | 存储在 `.env`，明文 | 会话存储在 `localStorage`，登录页面输入 |
| API Key | 在 `.env` 中明文 | 在 `localStorage` 中明文 |
| 翻译模式 | 无（始终 simple） | 支持 `simple` / `json` 两种模式 |
| 深色模式 | 终端自适应 | 设置页开关，使用 CSS 变量切换 |
| UI 语言 | 无（终端英文） | 支持多语言切换（`locale`） |

## App Password 安全提示

- **永远不要提交 `.env` 到 Git**：`.env.example` 已加入版本控制，但真实的 `.env` 在 `.gitignore` 中
- **App Password ≠ 账号密码**：App Password 只能用于 API 访问，不能登录网页，泄露风险可控
- **如需取消授权**：在 Bluesky 的 App Passwords 页面删除对应条目即可立即失效
