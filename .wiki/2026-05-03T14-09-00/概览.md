# 🦋 Bluesky 客户端概览

**Bluesky 客户端**是一个双界面、AI 深度集成的第三方 [Bluesky](https://bsky.app) 客户端。它同时提供**终端界面 (TUI)** 和**可安装的网页应用 (PWA)**，底层共享同一套业务逻辑与 AI 引擎。

> 一句话定位：一个让你在终端里也能刷 Bluesky、并且随时召唤 AI 助手写帖、翻译、润色的全功能客户端。

[来源](README.md#L1-L5)

## 双界面：TUI vs PWA

这个项目最直观的特色是——**一个代码仓库，两套界面**。终端用户和浏览器用户各取所需，但操作的是同一组数据、同一个 AI。

| 维度 | TUI（终端） | PWA（网页） |
|------|:-----------:|:----------:|
| **运行环境** | 终端（Windows Terminal / iTerm2 / Kitty） | 浏览器（Chrome / Firefox / Safari） |
| **渲染引擎** | Ink（React-on-terminal） | React DOM + Tailwind CSS |
| **凭据管理** | `.env` 文件（Handle + App Password + API Key） | 浏览器登录表单 + localStorage |
| **路由方式** | 键盘事件驱动的视图切换 | Hash 路由 `#/feed?feed=...` |
| **输入方式** | 全键盘操作（`j/k` 上下，`l` 点赞，`R` 回复） | 鼠标/触控 + 键盘 |
| **聊天存储** | JSON 文件 (`ChatStorage`) | IndexedDB |
| **Markdown 渲染** | 自定义终端解析器 | react-markdown + GFM（表格、代码高亮） |
| **图片显示** | 终端字符画 / CDN 链接 | 图片 Lightbox + 本地保存 |
| **深色模式** | 终端自带（无独立切换） | CSS 变量切换 |
| **离线/安装** | 无 | Service Worker + manifest.json |
| **打开耗时** | 毫秒级 | 秒级（首次加载） |
| **适用场景** | 极客、SSH 远程、低配环境 | 日常浏览、移动端、视觉丰富 |

两个界面都支持完整的时间线浏览、发帖、通知、搜索、个人主页、书签以及全部 AI 功能，但在**交互范式和视觉呈现**上各有所长。

## 四大包的职责

项目采用 Monorepo 架构，分为四个 npm 包，依赖流向严格单向：

```
@bsky/core  →  @bsky/app  →  @bsky/tui（终端）
                          →  @bsky/pwa（网页）
```

### `@bsky/core` — 核心层（零 UI 依赖）

纯 TypeScript 模块，不依赖任何 UI 框架。封装了：
- **AT Protocol 客户端**（基于 `ky` HTTP 库，含自动 JWT 刷新）
- **AI 助手引擎**（DeepSeek V4 Flash 驱动）
- **31 个 Bluesky 工具**（读取/写入/搜索/分析）
- **提示词模板**与系统提示管理
- **全部类型定义**

这是整个项目的**逻辑心脏**，可被任何框架复用。

[来源](README.md#L131-L138) | [详情](bsky-core-核心层设计.md)

### `@bsky/app` — 共享逻辑层

React Hooks + 纯 Store 层，TUI 和 PWA 直接复用同一套 hooks：
- `useAuth` / `useTimeline` / `useThread` / `useAIChat` / `usePostActions` 等
- 导航状态机、Feed 配置持久化
- 国际化 (i18n) 单例 Store（中/英/日）
- 书签系统、聊天记录接口

**PWA 的 React 组件只需要引入这些 hooks，不需要重复编写业务逻辑。**

[来源](README.md#L131-L138) | [详情](bsky-app-共享逻辑与-hooks.md)

### `@bsky/tui` — 终端界面

基于 Ink（React-on-terminal）实现的终端 UI：
- 键盘驱动的视图路由（`App.tsx` 中单一 `useInput` 处理所有按键）
- 视口渲染的帖子列表（`PostList.tsx`）
- CJK 文本宽度计算与智能换行（`visualWidth` / `wrapLines`）
- ANSI 鼠标追踪（滚动支持）
- `tsx src/cli.ts` 直接启动

### `@bsky/pwa` — 网页应用

基于 React DOM + Vite + Tailwind CSS 构建的可安装 PWA：
- Hash 路由（兼容静态托管）
- 虚拟滚动（`@tanstack/react-virtual`）
- IndexedDB 聊天存储
- SVG 图标系统（50+ 图标）
- 主题切换、滚动位置恢复
- PWA Manifest + Service Worker

[来源](docs/ARCHITECTURE.md#L8-L50)

## AI 深度集成：超越官方客户端的亮点

本项目最区别于官方 Bluesky 客户端的能力——**AI 不是附加功能，而是内嵌在设计中的一等公民**。

### 🧠 AI 对话 + 31 个工具

在 AI Chat 界面中，你可以用自然语言操控 Bluesky。AI 助手背后绑定了 **31 个 Bluesky 工具**，分为三类：

| 类别 | 工具示例 | 数量 |
|------|---------|:----:|
| **读取** | 获取时间线、查看帖子、搜索用户、获取通知 | ~15 |
| **写入** | 发帖、回复、点赞、转发、关注 | ~10 |
| **分析** | 图片分析（视觉模型）、Feed 推荐 | ~6 |

AI 会判断你的意图，自动调用对应工具。例如你说"看看最近有什么热门帖"，AI 会调用 `getTimeline` 工具并返回结果。

[详情](ai-助手与工具调用系统.md)

### ⚡ 流式输出 + 思考模式

所有 AI 响应通过 **SSE（Server-Sent Events）流式传输**，逐字呈现，不需要等待完整响应。如果启用了**思考模式**，AI 的推理过程会以 `💭` 块显示在回答上方，让你看到"AI 在想什么"。

[详情](流式输出与思考模式.md)

### 🌐 智能翻译（双模式）

选中帖子后一键翻译，支持 **7 种语言**。提供两种模式：
- **简单模式**：直接给出翻译文本
- **JSON 模式**：返回结构化结果 `{translated, source_lang}`，自动检测源语言

内置指数退避重试机制，最多重试 3 次，应对网络波动或解析失败。

[来源](docs/ARCHITECTURE.md#L73-L75) | [详情](翻译与润色功能.md)

### ✨ 草稿润色

在写帖界面调用 AI 润色，指定风格要求（正式/轻松/简洁等），AI 会改写你的草稿但保留原意。

### 🚦 写操作确认门控

AI 执行写操作（发帖、点赞、转发）前必须经过用户确认，避免意外操作。这是 AI **安全护栏**的一部分。

[详情](ai-对话与智能助手.md)

## 黄金法则：一次编写，两处渲染

> **"业务逻辑只写一次，TUI 和 PWA 只是纯渲染层。"**

- `@bsky/core` 中的 `BskyClient` 和 `AIAssistant` 不知道也不关心它们在终端还是浏览器中运行
- `@bsky/app` 中的 React Hooks（`useTimeline`、`useAIChat` 等）被两个界面直接导入使用
- TUI 和 PWA 各自只负责**把数据渲染成对应的 UI 元素**——终端用 Ink 的 `<Box>``<Text>`，浏览器用 `<div>``<span>`

这意味着：添加一个新功能时，**核心逻辑改一次**，然后分别更新两个界面的渲染组件即可。
