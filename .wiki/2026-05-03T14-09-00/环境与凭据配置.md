# 环境与凭据配置

运行这个客户端之前，你只需要准备三样东西：一个 Bluesky 账号、一个 LLM API 密钥，以及把这两者告诉程序的方式。本页逐一拆解 `.env` 文件中的每个字段，并说明 TUI（终端）和 PWA（浏览器）两种模式在配置上的根本差异。

---

## 你的第一份 `.env` 文件

项目根目录下的 `.env.example` 是一份完整模板，直接复制为 `.env` 即可开始配置：

```env
BLUESKY_HANDLE=your-handle.bsky.social
BLUESKY_APP_PASSWORD=your-app-password
LLM_API_KEY=sk-your-api-key
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-v4-flash
LLM_THINKING_ENABLED=true
LLM_VISION_ENABLED=false
TRANSLATE_TARGET_LANG=zh
```

[来源](.env.example#L1-L18)

## 字段逐行解读

### Bluesky 凭据

**`BLUESKY_HANDLE`** — 你的 Bluesky 用户名。通常是 `用户名.bsky.social` 格式，也可以是自定义域名。**这是必填项**，TUI 模式下若缺少此项程序会拒绝启动，直接弹出交互式设置向导。

**`BLUESKY_APP_PASSWORD`** — **应用密码**，不是你的登录密码。前往 Bluesky 网站 **设置 → 应用密码 → 添加应用密码**，生成一串 `xxxx-xxxx-xxxx-xxxx` 格式的密码。使用 App Password 而非主密码是安全最佳实践——即使泄露也仅影响该客户端的访问权限。**这也是必填项**。

> **获取 App Password 的步骤**：打开 Bluesky 网页版 → 左下角「设置」→ 向下滚动到「应用密码」→ 点击「添加应用密码」→ 输入名称（如 `my-ai-client`），生成后复制保存，关闭后不再显示。

[来源](packages/tui/src/cli.ts#L42-L49)

### AI/LLM 配置

**`LLM_API_KEY`** — 调用大语言模型的 API 密钥。项目兼容所有 OpenAI 格式的 API，推荐使用 DeepSeek。**必填**，否则 AI 对话功能无法使用。

**`LLM_BASE_URL`** — API 地址。默认值为 `https://api.deepseek.com`，如果你使用兼容 OpenAI 格式的其他服务商（如 OpenRouter、Groq 或自部署的 Ollama 等），修改此项即可。

**`LLM_MODEL`** — 模型名称。默认 `deepseek-v4-flash`。可替换为任何你的 API Key 有权限调用的模型名，如 `gpt-4o`、`claude-3-opus` 等。

[来源](packages/tui/src/cli.ts#L50-L58)

### 功能开关

**`LLM_THINKING_ENABLED`** — 思考模式开关。设为 `true` 时，AI 的推理过程（Reasoning Content）会以 `💭` 思考块的形式展示出来；设为 `false` 则只显示最终回答。默认值根据代码逻辑判定：**只要不是 `'false'` 字符串，就视为开启**，所以即使不设置此项，思考模式也默认启用。

**`LLM_VISION_ENABLED`** — 视觉模式开关。这决定了 AI 是否能看到帖子中的图片内容。**仅当你的模型支持多模态视觉（如 GPT-4V、Claude Vision、DeepSeek VL）时才设为 `true`**。纯文本模型即使收到图片数据也无法理解，只会看到乱码。默认值为 `false`。

[来源](packages/tui/src/cli.ts#L52-L54)
[来源](packages/core/src/ai/assistant.ts#L335-L335)

### 翻译设置

**`TRANSLATE_TARGET_LANG`** — 翻译功能的目标语言。默认 `zh`（中文）。可选值包括 `zh`（中文）、`en`（英语）、`ja`（日语）、`ko`（韩语）、`fr`（法语）、`de`（德语）、`es`（西班牙语）。此字段影响 [`翻译与润色功能`](翻译与润色功能.md) 中自动翻译输出的语言。

[来源](docs/ENV.md#L18-L21)

## TUI vs PWA：配置方式的根本差异

这是初学者最容易困惑的地方。两种界面使用 **完全不同的配置机制**，理解这一点才能避免"我在 PWA 设置了为什么 TUI 不生效"之类的问题。

| 维度 | TUI（终端） | PWA（浏览器） |
|------|-------------|---------------|
| 配置载体 | `.env` 文件 | 表单输入 + `localStorage` |
| 加载时机 | 进程启动时 `dotenv` 自动读取 | 用户在登录页/设置页手动输入 |
| 生命周期 | 文件持久化，重启固定 | 存在浏览器本地，清空缓存后丢失 |
| 首次启动 | 无 `.env` 文件则弹出 **SetupWizard** 交互式引导 | 直接显示登录表单 |
| 安全边界 | 凭据明文存储在文件系统 | 凭据仅存于浏览器沙箱内 |

[来源](README.md#L118-L124)
[来源](packages/tui/src/cli.ts#L7-L19)

### TUI 的配置流

```
┌─────────────┐     dotenv.config()     ┌─────────────────────┐
│  .env 文件   │ ──────────────────────→ │  process.env.*      │
│             │                         │  ↓                   │
│  HANDLE=xxx │                         │  getConfigFromEnv()  │
│  PASSWORD=… │                         │  ↓                   │
│  API_KEY=…  │                         │  AppConfig 对象       │
└─────────────┘                         └─────────────────────┘
```

TUI 启动时从两个路径查找 `.env` 文件：项目根目录和 `process.cwd()`。如果找不到有效的 `BLUESKY_HANDLE` 和 `BLUESKY_APP_PASSWORD`，程序不会崩溃——而是自动切换到 **SetupWizard** 交互式界面，引导你逐项输入，然后写入 `.env` 文件并重启应用。

[来源](packages/tui/src/cli.ts#L77-L84)

### PWA 的配置流

PWA 没有 `.env` 的概念。用户在登录页输入 Bluesky Handle 和 App Password 后，session 存储在浏览器 `localStorage`。AI 相关的配置（API Key、Base URL、Model、思考/视觉开关）则在设置页单独填写，同样持久化到 `localStorage`。

这意味着 **PWA 可以部署为纯静态页面**，所有凭据只留在用户自己的浏览器中，从不发送到任何第三方服务器。

[来源](README.md#L121-L123)
