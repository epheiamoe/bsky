# 浏览时间线与帖子

Bluesky 的核心体验围绕两条线展开：**时间线（Feed）**——按算法或自定义规则聚合的帖子流，以及**讨论串（Thread）**——以一条帖子为根的对话树。bsky 项目在 TUI 和 PWA 两套界面中分别实现了这两条线，操作方式和视觉呈现各有特色。

---

## Feed 切换：选择你想看的内容

Bluesky 的时间线不像传统社交网络只有"关注"和"推荐"两个 tab。在 AT Protocol 的架构下，所有时间线都是 **Feed（动态源）**——一段以 AT URI 标识的算法规则。项目预设了三类：

| Feed 类型 | 说明 |
|---|---|
| **Following（关注）** | 你关注的人发布的帖子 |
| **Discover（发现）** | 算法推荐的热门/相关帖子 |
| **自定义 Feed** | 由社区或自己创建的算法源，例如"猫猫图""科技新闻" |

[来源](../packages/core/src/feeds.ts#L1-L30)

### TUI：按 `f` 打开 Feed 配置面板

在终端中，当焦点位于时间线视图时，按 **`f`** 键即可呼出 **Feed 配置覆盖层（FeedConfigOverlay）**。这个覆盖层会显示：

1. **当前已添加的 Feed 列表**——每个条目前有 `★` 标记默认 Feed
2. **推荐 Feed 列表**——通过 `client.getSuggestedFeeds()` 从网络加载
3. **添加自定义 Feed**——手动输入 AT URI 来订阅任何公开的 Feed

操作方式全键盘驱动：`j/k` 上下移动光标，`Enter` 选择并切换到该 Feed，`d` 删除该 Feed，`s` 设为默认。按 `Esc` 关闭面板。

```typescript
// TUI App.tsx — 按 f 键打开 Feed 配置
if (k === 'f') { setFeedConfigInput(''); setShowFeedConfig(true); }
```

[来源](../packages/tui/src/components/App.tsx#L319-L322)

### PWA：点击下拉菜单

PWA 的时间线头部（`FeedHeader` 组件）展示当前 Feed 的名称和一个 `▼` 箭头按钮。点击后通过 React Portal 在 `document.body` 上渲染一个下拉列表，罗列所有已配置的 Feed。

点击某个选项即时切换；底部的"配置 Feed"按钮打开一个 **FeedConfigModal** 弹窗，功能与 TUI 面板基本一致但多了鼠标操作——可点击"推荐"列表中的 `+` 按钮一键添加，也可在输入框中粘贴 AT URI 后点击"添加"按钮。

[来源](../packages/pwa/src/components/FeedHeader.tsx#L1-L90)

### 默认 Feed 的持久化

TUI 和 PWA 共享同一套配置状态：`getFeedConfig()` / `setDefaultFeed()` 等函数来自 `@bsky/app` 包。这意味着你在 TUI 中设为默认的 Feed，切换到 PWA 时仍然有效——反之亦然。

[来源](../packages/app/src/stores/timeline.ts#L1-L30)

## 帖子列表：虚拟滚动与视口渲染

时间线可能包含数百条帖子，全部渲染会造成严重的性能问题。两种界面用了不同的优化策略，但核心思想一致：**只渲染屏幕可见范围内的帖子**。

### PWA：@tanstack/react-virtual 虚拟滚动

`FeedTimeline` 组件利用 `@tanstack/react-virtual` 库实现了虚拟滚动：

```typescript
const virtualizer = useVirtualizer({
  count: posts.length,
  getScrollElement: () => scrollRef.current,
  estimateSize: () => 120, // 每条帖子估算高度 120px
  overscan: 5,             // 额外预渲染 5 条
});
```

关键机制：
- **估算高度**——每条帖子固定估算 120px，实际高度在渲染后通过 `virtualizer.measureElement` 修正
- **绝对定位**——虚拟列表的容器高度设为 `getTotalSize()`（所有帖子的总估算高度），每个虚拟项通过 `transform: translateY()` 定位到正确位置
- **IntersectionObserver 自动加载**——列表底部有一个 `sentinelRef` 哨兵元素，当它进入视口时自动触发 `loadMore()`，实现无限滚动

```typescript
const obs = new IntersectionObserver(
  ([entry]) => { if (entry?.isIntersecting) loadMore(); },
  { root: scrollRef.current, rootMargin: '200px' },
);
obs.observe(sentinelRef.current);
```

滚动位置会在导航返回时恢复：通过 `initialScrollIndex` 属性传入之前记录的第一个可见项索引，组件挂载后调用 `virtualizer.scrollToIndex(target, { align: 'start' })`。

[来源](../packages/pwa/src/components/FeedTimeline.tsx#L1-L120)

### TUI：手动视口计算

TUI 无法使用 DOM 的滚动能力（Ink 渲染的是终端字符），因此采用了一种更"古典"的方法：

**预计算所有行**。每条帖子被 `postToLines()` 函数展开为若干文本行（作者行、正文行、引用行、统计行、空行等），存入一个扁平的 `PostLine[]` 数组。

```typescript
const allLines = useMemo(() => {
  const lines: PostLine[] = [];
  for (let i = 0; i < posts.length; i++) {
    const postLines = postToLines(posts[i]!, i, i === selectedIndex, width, t, locale);
    for (const l of postLines) lines.push(l);
  }
  return lines;
}, [posts, selectedIndex, width, t, locale]);
```

**计算可见范围**。根据 `height`（终端行数）减去头部和边距占用，得到可用行数 `visibleLines`。找到当前选中项所在的行位置 `selectedLineStart`，以此为基准居中切取 `visibleSlice`。

**滚动指示器**。顶部和底部显示 `▲` / `▼` 箭头及百分比，让用户知道上方/下方还有内容。

```typescript
const hasAbove = viewStart > 0;
const hasBelow = viewStart + visibleLines < allLines.length;
const scrollPct = allLines.length > 0
  ? Math.round((viewStart / Math.max(1, allLines.length - visibleLines)) * 100) : 0;
```

[来源](../packages/tui/src/components/PostList.tsx#L1-L60)

## 讨论串：树形展开与折叠

点击时间线中的任意帖子，就会进入它的 **讨论串视图（Thread View）**。Bluesky 的讨论串实质是一棵树：每条帖子可以有父帖（被回复的对象）和子帖（回复它的内容）。

### 数据模型：FlatLine 扁平化

`useThread` 这个共享 Hook（来自 `@bsky/app`）将树形结构"拍平"成 `FlatLine[]` 数组。每条 `FlatLine` 包含：

| 属性 | 说明 |
|---|---|
| `depth` | 在树中的深度。根的 depth = 0，父帖 depth < 0，子帖 depth > 0 |
| `isRoot` | 是否为当前讨论串的根帖子 |
| `isTruncation` | 是否为"还有 X 条回复未显示"的占位行 |
| `uri` | 帖子的 AT URI |
| `quotedPost` | 引用的帖子数据（嵌套结构） |

**父帖链表**（themeLines）：depth < 0 的帖子，按 depth 升序排列，形成从根到当前帖子的引用链。

**回复列表**（replyLines）：depth 在 1 到 `focusedDepth + 1` 之间的帖子，按时间升序排列。每条回复画对应的缩进层级（`'  '.repeat(Math.min(line.depth - 1, 3))`）。

[来源](../packages/app/src/hooks/useThread.ts#L46-L130)

### 展开更多回复

当某条帖子的回复数超过 `maxSiblings`（默认 5 条），尾部会插入一条 `isTruncation: true` 的占位线，显示"还有 X 条回复未显示"。

**TUI**：光标移到该行后按 **Enter** 触发 `expandReplies()`，将 `maxSiblings` 增加 10，然后重新调用 API 获取完整的帖子树并重新扁平化。

**PWA**：点击"还有 X 条回复"按钮，同样调用 `expandReplies()`。

```typescript
const expandReplies = useCallback(() => {
  if (!client || !uri) return;
  setMaxSiblings(prev => prev + 10);
}, [client, uri]);
```

[来源](../packages/app/src/hooks/useThread.ts#L80-L95)

### TUI 讨论串的键盘导航

`UnifiedThreadView` 组件完全由键盘驱动：

| 按键 | 功能 |
|---|---|
| `↑` / `k` | 移动光标（高亮目标帖子） |
| `↓` / `j` | 移动光标 |
| `Enter` | 将光标指向的帖子设为新的"当前帖子"，重新加载讨论串 |
| `h` | 跳回主题帖（theme post） |
| `l` | 点赞光标处的帖子 |
| `r` | 转发（先选"转发/引用"，再确认） |
| `c` | 回复光标处的帖子 |
| `v` | 添加/移除书签 |
| `f` | 翻译光标处的帖子文本 |
| `u` | 关注/取关当前帖子的作者 |
| `g` | 跳转到当前帖子作者的资料页 |

[来源](../packages/tui/src/components/UnifiedThreadView.tsx#L60-L130)

### PWA 讨论串的鼠标操作

`ThreadView` 完全面向点击设计：

- 父帖链中的每条帖子可**点击**跳转到对应讨论串
- 当前帖子的作者头像可**点击**跳转资料页
- 引用卡片可**点击**跳转到被引帖子的讨论串
- 每个帖子底部有 `PostActionsRow`（点赞、转发、回复、书签等操作按钮）
- 翻译按钮、复制链接按钮、删除按钮（仅自己的帖子）一目了然

[来源](../packages/pwa/src/components/ThreadView.tsx#L1-L100)

## 引用帖子的卡片展示

当一条帖子引用了另一条帖子（类似 Twitter 的 Quote Tweet），两种界面分别用不同的视觉方案呈现。

### TUI：字符画风格的引用区

在 `postToLines()` 中，被引用的帖子以 **`│` 缩进线** 的形式渲染在帖子正文下方：

```
│ @被引用者昵称
│ 被引用的帖子内容文本...
```

每行前都有一个 `│` 符号，视觉上形成"嵌套卡片"的效果。引用内容包含作者名和正文，如果被引帖子有图片，则继续以 `│` 线跟随图片链接。

[来源](../packages/tui/src/components/PostItem.tsx#L60-L95)

### PWA：带圆角和阴影的卡片

PWA 的引用帖渲染为独立的 `<div>` 卡片，有圆角、边框和浅色背景：

```tsx
<div className="mt-2 border border-border rounded-xl p-3 bg-surface cursor-pointer ...">
  <div className="flex items-center gap-2 mb-1">
    {quotedPost.authorAvatar && <img className="w-4 h-4 rounded-full" />}
    <span className="text-xs font-semibold">{quotedPost.displayName}</span>
    <span className="text-xs text-text-secondary">@{quotedPost.handle}</span>
  </div>
  <p className="text-xs text-text-primary line-clamp-3">{linkifyText(quotedPost.text)}</p>
  {quotedPost.imageUrls && (
    <div className="mt-1 flex gap-1">
      {quotedPost.imageUrls.slice(0, 2).map((url, idx) => (
        <img className="w-16 h-16 object-cover rounded-md" />
      ))}
    </div>
  )}
</div>
```

关键差异：PWA 的引用卡片是**可点击**的，点击后跳转到被引帖子的讨论串。TUI 的引用行在终端中仅用于阅读，没有直接的跳转链接（但引用的 AT URI 可通过 `q` 键在时间线中跳转）。

在讨论串详情页（`ThreadView`）中，被引用帖的展示更进一步：包含了作者头像、分享链接的卡片式展示，以及最多 2 张缩略图。

[来源](../packages/pwa/src/components/PostCard.tsx#L410-L440)

## 两种界面的操作差异一览

| 操作 | TUI（终端） | PWA（网页） |
|---|---|---|
| **切换 Feed** | 按 `f` 打开键盘控制的面板，`j/k` 导航，`Enter` 选择 | 点击顶部 `▼` 下拉菜单，或点击"配置 Feed"打开弹窗 |
| **添加自定义 Feed** | 在面板中输入 AT URI 后按 `Enter` | 在弹窗的输入框中粘贴 URI，点击"添加"按钮 |
| **浏览帖子列表** | `↑↓`/`jk` 移动选中行，PgUp/PgDn 翻页 | 鼠标滚轮或触摸板滚动，支持触屏手势 |
| **进入帖子详情** | 在帖子列表选中某条帖子后按 `Enter` | 点击帖子的任意区域 |
| **展开更多回复** | 光标移到"还有 X 条回复"行按 `Enter` | 点击"还有 X 条回复"按钮 |
| **查看引用帖子** | 以 `│` 前缀的文本行展示，不可点击跳转 | 以卡片样式展示，点击跳转被引帖讨论串 |
| **关注作者** | `u` 键切换关注/取关 | 点击帖子顶部的"关注/取关"按钮 |
| **翻译帖子** | `f` 键触发翻译，结果显示在当前帖上方 | 点击翻译图标，翻译结果以内联卡片显示 |
| **滚动定位** | 选中项居中于可视区域 | 虚拟滚动 + 滚动位置恢复 |
| **无限滚动** | 手动按 `m` 加载更多 | 自动加载（IntersectionObserver 哨兵） |
