# 个人主页与书签

当你点击某个用户名或头像时，就会进入**个人主页**。这里集中展示一个用户的所有公开信息、帖子，以及关注/粉丝列表。与此同时，本客户端内置了一套**书签系统**——不同于浏览器收藏夹，它直接调用 Bluesky 官方 API 将书签保存在服务端，因此在 TUI 和 PWA 之间完全同步。

---

## 个人主页概览

个人主页由三部分组成：

1. **顶部资料卡**：头像、横幅背景、显示名、简介、统计数据（帖子数/关注者/关注中）
2. **操作栏**：关注/取消关注按钮、AI 对话入口
3. **标签页切换**：帖子（默认）和回复

> PWA 中可以点击头像/横幅放大浏览，点击关注者/关注中数字可弹出列表。[来源](../../packages/pwa/src/components/ProfilePage.tsx#L1-L10)

### 标签页切换

TUI 中使用 `Tab` 键（或 `←`/`→`）在「帖子」和「回复」间切换。[来源](../../packages/tui/src/components/ProfileView.tsx#L83-L88)

PWA 中点击顶部的标签按钮切换，当前标签底部有一条蓝色指示线。[来源](../../packages/pwa/src/components/ProfilePage.tsx#L241-L266)

标签变化时会重新请求对应数据。帖子标签（`posts`）过滤掉了回复，只显示原创内容；回复标签（`replies`）则显示该用户的所有回复。两种标签下均支持虚拟滚动（PWA 使用 `@tanstack/react-virtual`，TUI 使用箭头导航）。

```typescript
// useProfile 中根据 tab 决定 filter 参数
const filter = tab === 'posts' ? 'posts_no_replies' : '';
const res = await client.getAuthorFeed(actor, 20, cursor, filter || undefined);
```

[来源](../../packages/app/src/hooks/useProfile.ts#L45-L48)

### 关注与取消关注

关注状态通过 `profile.viewer?.following` 字段判断——如果该字段存在值，说明当前已关注，该值即为关注记录的 URI。[来源](../../packages/app/src/hooks/useProfile.ts#L26-L27)

**执行逻辑**（以 TUI 为例）：

- 在 ProfileView 中按 `u` 键触发关注或取关
- 调用 `handleFollow()` 时，客户端发送 `client.follow(profile.did)`，返回新创建的关注记录 URI
- 调用 `handleUnfollow()` 时，客户端发送 `client.unfollow(followUri)`，删除该关注记录
- 两种操作后都会重新拉取个人资料以刷新 `viewer` 状态

[来源](../../packages/app/src/hooks/useProfile.ts#L64-L77) [来源](../../packages/tui/src/components/ProfileView.tsx#L106-L109)

PWA 中点击按钮同样触发上述逻辑，关注按钮为蓝色实心，取关按钮为红色边框。[来源](../../packages/pwa/src/components/ProfilePage.tsx#L156-L170)

### 关注/粉丝列表

在 PWA 中点击「关注中」或「关注者」数字会弹出列表视图，支持滚动加载更多（使用 `IntersectionObserver` 监听哨兵元素）。点击列表中的用户可跳转到其个人主页。[来源](../../packages/pwa/src/components/ProfilePage.tsx#L96-L139)

TUI 中按 `p` 打开关注列表、`P` 打开粉丝列表，`j/k` 导航，`Enter` 进入该用户主页，`Esc` 返回，`m` 加载更多。[来源](../../packages/tui/src/components/ProfileView.tsx#L99-L101)

## 书签系统

书签是本客户端的一个特色功能。**它完全基于 Bluesky 的内置 API**，而非浏览器的 localStorage 或 IndexedDB。这意味着你添加的书签会保存在你的 Bluesky 账户中，在任何客户端（TUI、PWA 乃至第三方）上都能看到。

### 底层 API

核心层 `@bsky/core` 封装了三个端点：

| 方法 | 端点 | 说明 |
|------|------|------|
| `createBookmark(uri, cid)` | `app.bsky.bookmark.createBookmark` | 添加书签 |
| `deleteBookmark(uri)` | `app.bsky.bookmark.deleteBookmark` | 删除书签（静默忽略不存在的情况） |
| `getBookmarks(limit, cursor)` | `app.bsky.bookmark.getBookmarks` | 获取书签列表（分页） |

[来源](../../packages/core/src/at/client.ts#L368-L404)

返回的 `BookmarkResult` 结构包含 `subject`（帖子引用）和 `item`（完整的 `PostView`），因此书签列表直接展示帖子卡片，无需额外请求。[来源](../../packages/core/src/at/types.ts#L264-L272)

### 添加与删除书签

**在 TUI 中**：

- 浏览时间线时，选中一条帖子后按 `v` 即可**切换**书签状态（已收藏则取消，未收藏则添加）
- 进入书签列表后，选中一条按 `d` 可**删除**书签
- 按 `b` 从任何界面跳转到书签列表

[来源](../../packages/tui/src/components/App.tsx#L297-L299) [来源](../../packages/tui/src/components/App.tsx#L309-L314)

**在 PWA 中**：

- 帖子详情页（ThreadView）的操作栏中有一个**书签图标按钮**，实心表示已收藏，空心表示未收藏，点击切换
- 书签列表页（BookmarkPage）的每个帖子右上角有一个 `×` 按钮，点击删除

[来源](../../packages/pwa/src/components/PostActionsRow.tsx#L88-L93) [来源](../../packages/pwa/src/components/BookmarkPage.tsx#L46-L57)

**核心 Hook `useBookmarks`** 维护了一个 `bookmarkedUris` 集合用于快速判断某帖子是否已收藏，同时提供 `toggleBookmark(uri, cid)` 方法做一键切换。[来源](../../packages/app/src/hooks/useBookmarks.ts#L13-L15)

### 浏览书签列表

**TUI 中书签视图**：
- 按 `b` 进入，显示所有已收藏的帖子列表
- `j`/`k` 或上下箭头导航，`Enter` 打开帖子详情
- `r` 刷新列表，`d` 移除当前选中的书签
- `q` 打开当前帖子的引用帖子（如果有）

[来源](../../packages/tui/src/components/App.tsx#L301-L319)

**PWA 中书签页面**：
- 侧边栏点击书签图标进入
- 使用 `PostCard` 渲染每个帖子，支持点击进入详情
- 顶部有刷新按钮，右上角 `×` 可删除单条书签
- 自动恢复滚动位置（通过 `useScrollRestore`）

[来源](../../packages/pwa/src/components/BookmarkPage.tsx#L1-L70)

## 技术要点小结

- 个人资料和书签数据都通过 `@bsky/core` 的 `BskyClient` 请求，遵循 AT Protocol 标准
- `useProfile` 和 `useBookmarks` 两个 Hook 设计为与 UI 无关，TUI 和 PWA 共享同一套逻辑——详见 [@bsky/app 共享逻辑与 Hooks](bsky-app-共享逻辑与-hooks.md)
- 书签数据存储在 Bluesky 服务端，不是**浏览器本地存储**，这是与大部分社交应用的关键区别
- 关注操作使用 AT Protocol 的 `com.atproto.repo.createRecord`，取关使用 `com.atproto.repo.deleteRecord`，封装在 `client.follow()` 和 `client.unfollow()` 中
