# 搜索与通知管理

Bluesky 客户端提供全文搜索和多维度通知管理两大功能。搜索支持四种分类过滤和 Lucene 风格查询语法；通知系统按原因分类并实时显示未读计数，帮助你不遗漏任何互动。

---

## 交互方式总览

TUI 终端和 PWA 网页两种界面使用不同的交互范式，核心逻辑却共享同一套 Hook。

| 功能 | TUI 终端 | PWA 网页 |
|------|---------|----------|
| 搜索输入 | 键盘键入，回车提交 | 文本框输入 + 点击按钮 |
| 标签切换 | Tab 键轮换 | 点击标签栏按钮 |
| 结果导航 | `j`/`k`（或上下方向键）移动光标 | 点击任意结果卡片 |
| 查看详情 | 回车进入帖子/用户/动态源 | 点击卡片自动跳转 |
| 通知列表 | `j`/`k` 导航，`Enter` 跳转 | 点击通知条目 |
| 刷新通知 | 按 `R` 键 | 点击刷新按钮 |
| 未读标记 | 侧边栏显示数字徽章 | 侧边栏支持但当前版本未连线 |

## 搜索系统

### 四个标签页

搜索分为 **热门(Top)**、**最新(Latest)**、**用户(Users)**、**动态源(Feeds)** 四个维度：

- **热门 / 最新** — 调用 `client.searchPosts()`，通过 `sort: 'latest'` 参数区分排序方式，每次请求返回 25 条帖子
- **用户** — 调用 `client.searchActors()`，搜索用户句柄和显示名称
- **动态源** — 调用 `client.getPopularFeedGenerators()` 获取全局流行动态源，然后在客户端按名称/描述/创建者句柄过滤匹配项

```typescript
// 核心搜索逻辑
if (t === 'top' || t === 'latest') {
  const res = await client.searchPosts({ q, limit: 25, sort: t === 'latest' ? 'latest' : undefined });
  setPosts(res.posts);
} else if (t === 'users') {
  const res = await client.searchActors({ q, limit: 25 });
  setUsers(res.actors);
} else if (t === 'feeds') {
  const res = await client.getPopularFeedGenerators(25);
  // 客户端过滤
  const matches = res.feeds.filter(f =>
    f.displayName.toLowerCase().includes(qLower) ||
    f.description?.toLowerCase().includes(qLower) ||
    f.creator?.handle?.toLowerCase().includes(qLower)
  );
  setFeeds(matches);
}
```

### Lucene 风格搜索语法

`app.bsky.feed.searchPosts` 底层基于 AT Protocol 的搜索服务（通常是 OpenSearch/Elasticsearch），支持以下 Lucene 风格操作符：

| 语法 | 示例 | 说明 |
|------|------|------|
| `from:@handle` | `from:@jack.bsky.social` | 按发帖人过滤 |
| `to:@handle` | `to:@alice.bsky.social` | 按回复目标过滤 |
| `has:image` | `cat has:image` | 仅带图片的帖子 |
| `has:video` | `tutorial has:video` | 仅带视频的帖子 |
| `has:link` | `news has:link` | 仅带链接的帖子 |
| `has:embed` | `review has:embed` | 仅带嵌入内容的帖子 |
| `lang:xx` | `lang:zh` | 按语言过滤（ISO 639-1） |
| `"精确短语"` | `"climate change"` | 精确匹配短语 |
| `-排除词` | `apple -fruit` | 排除包含某词的帖子 |
| `OR` | `cat OR dog` | 逻辑或 |
| `since:timestamp` | `since:2024-01-01` | 起始时间过滤 |
| `until:timestamp` | `until:2024-06-30` | 截止时间过滤 |

结合使用：`from:@me has:image lang:zh` — 搜索"我发布的带图片的中文帖子"。

> **注意**：搜索需认证登录，公共 API 会返回 403。
> [来源](packages/core/src/at/client.ts#L170-L172)

### TUI 搜索操作

在终端界面的搜索视图里：

- **Tab 键** — 在四个标签页间依次切换
- **输入后回车** — 执行搜索，自动跳转到搜索结果
- **j / k** — 在结果列表中上下移动光标
- **回车** — 进入当前选中项（帖子→讨论串、用户→个人主页、动态源→动态源时间线）
- **V 键** — 查看当前选中的动态源详情
- **S 键** — 订阅当前选中的动态源
- **Esc 键** — 返回上一界面

搜索结果支持滚动位置保存：每个查询的浏览位置会被持久化到本地存储，再次搜索相同关键词时自动恢复。 [来源](packages/tui/src/components/SearchView.tsx#L32-L42)

### PWA 搜索操作

在网页界面中，浏览器地址栏的 Hash 路由会同步搜索参数：

```
#/search?q=关键词&tab=top|latest|users|feeds
```

搜索结果以完整的卡片列表呈现。帖子卡片包含交互按钮（点赞、转发等），用户卡片显示头像和句柄，动态源卡片支持一键订阅。点击任意结果直接导航到对应详情页。 [来源](packages/pwa/src/components/SearchPage.tsx#L80-L90)

## 通知系统

### 通知分类（Reason）

每条通知包含一个 `reason` 字段，标识触发通知的互动类型。系统支持 6 种原因：

| reason 值 | 中文含义 | 对应操作 |
|-----------|---------|---------|
| `like` | 赞了你的帖子 | `♥` |
| `repost` | 转发了你的帖子 | `♻` |
| `follow` | 关注了你 | `👤` |
| `reply` | 回复了你的帖子 | `💬` |
| `mention` | 提到了你 | `@` |
| `quote` | 引用了你的帖子 | `💬` |

如果通知包含 `reasonSubject`（即关联帖子的 URI），则表示该通知可点击跳转至对应讨论串。

### 未读计数与徽章

`useNotifications` Hook 加载 30 条最新通知，自动计算未读数：

```typescript
const { notifications, loading, unreadCount, refresh } = useNotifications(client);
// unreadCount = notifications.filter(n => !n.isRead).length
```

**TUI** 中，`unreadCount` 传递到侧边栏组件，在"通知"按钮旁显示数字徽章。未读数大于 0 时显示，超过 99 显示 `99+`。

```typescript
// Sidebar.tsx
const badge = tab.key === 'notifications' && notifCount > 0 ? ` ${notifCount}` : '';
```

**PWA** 侧边栏同样预留了徽章渲染逻辑，当前版本仅在通知页面内部加载数据；未读数随下次进入通知页面时同步更新。

[来源](packages/app/src/hooks/useNotifications.ts#L6-L21) · [来源](packages/tui/src/components/Sidebar.tsx#L69) · [来源](packages/pwa/src/components/Sidebar.tsx#L54-L57)

### TUI 通知操作

- **j / k** — 在通知列表中上下移动选择光标
- **回车** — 如果通知有关联帖子（`reasonSubject`），跳转到该帖子讨论串
- **R 键** — 手动刷新通知列表
- **状态标记**：已读通知显示 `○`，未读显示 `●`（青色高亮） [来源](packages/tui/src/components/NotifView.tsx#L23-L57)

### PWA 通知操作

- 每条通知显示用户头像、互动类型图标（如 `♥` 表示点赞）和相对时间（`3m`、`2h`、`5d`）
- 未读通知在右侧有一个蓝色圆点指示
- 点击可跳转的通知，自动进入对应帖子的讨论串视图
- 顶部刷新按钮可重新加载通知列表 [来源](packages/pwa/src/components/NotifsPage.tsx#L75-L110)

## 架构速览

```
用户交互层
  ├── TUI: SearchView.tsx / NotifView.tsx   (键盘导航)
  └── PWA: SearchPage.tsx / NotifsPage.tsx  (点击/触控)

共享 Hook 层 (@bsky/app)
  ├── useSearch.ts         → 管理 query / tab / 结果列表 / loading
  └── useNotifications.ts  → 管理 通知列表 / 未读计数 / 刷新

AT Protocol 客户端 (@bsky/core)
  ├── client.searchPosts()         → app.bsky.feed.searchPosts
  ├── client.searchActors()        → app.bsky.actor.searchActors
  ├── client.getPopularFeedGenerators()
  └── client.listNotifications()   → app.bsky.notification.listNotifications
```
