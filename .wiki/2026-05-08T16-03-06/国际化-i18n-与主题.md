# 国际化（i18n）与主题

这个项目同时支持**三语言界面切换**和**暗色/亮色主题**，两种能力独立运作但共享同一个设计目标：让用户以自己舒适的方式使用客户端。国际化系统位于 `packages/app/src/i18n/`，主题则依赖 `packages/pwa/src/index.css` 中的 CSS 变量层。

---

## 类型约束：Locale 与 LocaleMessages

整个 i18n 系统的类型根在 `types.ts`：

```typescript
export type Locale = 'zh' | 'en' | 'ja';
export type LocaleMessages = Record<string, string>;
```

`Locale` 是受支持的三种语言的字面量联合类型，`LocaleMessages` 即扁平的键值映射表——每条消息是一个点分路径（如 `'nav.feed'`），对应一个翻译字符串。

[来源](packages/app/src/i18n/types.ts#L1-L3)

## 翻译消息的组织

每个语言独立成一个模块文件，输出 `LocaleMessages` 对象：

| 文件 | 语言 | 键数量（约） |
|------|------|-------------|
| `locales/zh.ts` | 简体中文 | ~160 个键 |
| `locales/en.ts` | 英文 | ~160 个键 |
| `locales/ja.ts` | 日文 | ~160 个键 |

三个文件的键结构完全对齐，以点分命名空间划分：`nav.*` 导航、`action.*` 操作、`status.*` 状态、`compose.*` 发帖、`thread.*` 帖子线程、`notifications.*` 通知、`settings.*` 设置、`theme.*` 主题、`keys.*` 键盘提示等等。

`locales/index.ts` 聚合三者为一个 `Record<Locale, LocaleMessages>` 对象：

```typescript
export const messages: Record<Locale, LocaleMessages> = { zh, en, ja };
export const availableLocales: Locale[] = ['zh', 'en', 'ja'];
export const localeLabels: Record<Locale, string> = {
  zh: '中文',
  en: 'English',
  ja: '日本語',
};
```

`availableLocales` 和 `localeLabels` 直接导出供 UI 渲染语言选择器使用。

## I18nStore 单例模式

### 接口

`I18nStore` 接口定义了运行时状态和操作方法：

```typescript
export interface I18nStore {
  locale: Locale;
  messages: LocaleMessages;
  getLocale(): Locale;
  setLocale(locale: Locale): void;
  t(key: string, params?: Record<string, string | number>): string;
  _notify(): void;
  subscribe(fn: () => void): () => void;
  unsubscribe(fn: () => void): void;
}
```

[来源](packages/app/src/i18n/store.ts#L4-L15)

### 创建与单例

`createI18nStore(initialLocale)` 构造一个新 store，接受可选初始语言（默认 `'zh'`）。实例内部维护一个 `listeners` 集合（`Set<() => void>`），每次切换语言时通知所有订阅者。

关键模式是**模块级单例**：

```typescript
let _instance: I18nStore | null = null;

export function getI18nStore(initialLocale?: Locale): I18nStore {
  if (!_instance) {
    _instance = createI18nStore(initialLocale);
  }
  return _instance;
}

export function resetI18nStore(): void {
  _instance = null;
}
```

`getI18nStore` 保证全局只有一个 store 实例，首次调用时创建，后续复用。`resetI18nStore` 用于测试场景重置状态。

[来源](packages/app/src/i18n/store.ts#L70-L82)

### 切换语言

`setLocale(locale)` 执行三步操作：

1. 检查 `allMessages[locale]` 是否存在，不存在则静默返回（防止设置不存在的语言）
2. 更新 `store.locale` 和 `store.messages`
3. 遍历 `listeners` 逐一通知

```typescript
setLocale(locale: Locale) {
  if (!allMessages[locale]) return;
  store.locale = locale;
  store.messages = allMessages[locale];
  listeners.forEach(fn => fn());
},
```

[来源](packages/app/src/i18n/store.ts#L35-L40)

### 翻译函数与三层 fallback

`t(key, params)` 是核心翻译函数，含三层回退链：

1. **当前语言**：在 `store.messages` 中查找 `key`
2. **英文回退**：当前语言无此键，尝试 `allMessages.en[key]`
3. **中文回退**：英文也无此键，尝试 `allMessages.zh[key]`
4. **原始键**：全部缺失，返回 `key` 本身

找到文本后，调用 `interpolate` 做参数替换：将 `{current}/{max}` 中的占位符替换为实际值。

```typescript
function interpolate(text: string, params?: Record<string, string | number>): string {
  if (!params) return text;
  return text.replace(/\{(\w+)\}/g, (_, key) => {
    return params[key] !== undefined ? String(params[key]) : `{${key}}`;
  });
}
```

例如：`t('compose.charCount', { current: 42, max: 300 })` → `"42/300"`。

[来源](packages/app/src/i18n/store.ts#L17-L22)
[来源](packages/app/src/i18n/store.ts#L42-L50)

## useI18n Hook：桥接 Store 到 React

`useI18n` 是将纯 store 接入 React 响应式系统的桥接 Hook：

```typescript
export function useI18n(initialLocale?: Locale) {
  const store = getI18nStore(initialLocale);
  const [, force] = useState(0);
  const tick = useCallback(() => force(n => n + 1), []);

  useEffect(() => store.subscribe(tick), [store, tick]);

  return {
    t: useCallback((key: string, params?: Record<string, string | number>) => store.t(key, params), [store]),
    locale: store.locale,
    setLocale: useCallback((l: Locale) => store.setLocale(l), [store]),
    availableLocales,
    localeLabels,
  };
}

export type UseI18nReturn = ReturnType<typeof useI18n>;
```

核心机制：

- **`getI18nStore(initialLocale)`** 获取单例（若首次调用则初始化语言）
- **`useState(0)` + `subscribe(tick)`**：store 的任何变更（如 `setLocale`）触发 `tick` → `forceUpdate` → 组件重渲染
- 返回的 `t`、`setLocale` 均被 `useCallback` 包裹，保证引用稳定性

[来源](packages/app/src/i18n/useI18n.ts#L1-L22)

此模式与 [React Hooks 架构与 Store 模式](react-hooks-架构与-store-模式.md) 中描述的纯 Store + Hook 桥接模式完全一致。

### 返回值速查

| 属性 | 类型 | 说明 |
|------|------|------|
| `t` | `(key, params?) => string` | 带三层 fallback 和参数插值的翻译函数 |
| `locale` | `Locale` | 当前语言代码 |
| `setLocale` | `(locale: Locale) => void` | 切换语言并通知所有监听者 |
| `availableLocales` | `Locale[]` | 支持的语言列表 `['zh', 'en', 'ja']` |
| `localeLabels` | `Record<Locale, string>` | 每语言的显示名称 `{ zh: '中文', en: 'English', ja: '日本語' }` |

## PWA 中的使用方式

在 PWA 项目中，每个页面组件通过 `useI18n()` 获取翻译函数 `t`。例如 SettingsModal：

```typescript
const { t, locale, setLocale, localeLabels, availableLocales } = useI18n();

// 渲染语言选择器
<select value={locale} onChange={e => setLocale(e.target.value as Locale)}>
  {availableLocales.map(l => (
    <option key={l} value={l}>{localeLabels[l]}</option>
  ))}
</select>
```

[来源](packages/pwa/src/components/SettingsModal.tsx#L32-L386)

而大部分组件只取 `t`：

```typescript
const { t } = useI18n();
// 使用示例
<button aria-label={t('nav.back')}>...</button>
<span>{t('status.loading')}</span>
```

## CSS 变量：双主题设计

主题系统基于原生 CSS 变量 + Tailwind `dark:` 前缀，设计文件为 `packages/pwa/src/index.css`。

### :root（亮色主题）

```css
:root {
  --color-primary: #00A5E0;
  --color-primary-hover: #0095C9;
  --color-surface: #F8F9FA;
  --color-border: #E5E7EB;
  --color-text-primary: #0F172A;
  --color-text-secondary: #64748B;
}
```

### .dark（暗色主题）

```css
.dark {
  --color-primary: #00A5E0;
  --color-primary-hover: #00B5F0;
  --color-surface: #121212;
  --color-border: #27272A;
  --color-text-primary: #F1F5F9;
  --color-text-secondary: #A3B4C0;
}
```

注意 `--color-primary` 在两种主题下主色相同（`#00A5E0`），但 hover 状态有微调。其他色值在暗色下显著变暗/变亮，确保可读性。

### 使用方式

组件中通过 `var(--color-*)` 或 Tailwind 语义类引用。Tailwind 通过 `tailwind.config` 映射这些变量：

```jsx
// 用法示例
<div className="bg-white dark:bg-[#0A0A0A] text-text-primary">
  <span className="text-text-secondary">{t('status.loading')}</span>
</div>
```

Markdown 内容区（`.markdown-body`）中的标题、代码块、引用、表格等全部通过 `var(--color-*)` 引用主题变量，深色模式下自动适配。

## 暗色模式切换机制

### 核心操作

切换主题本质是对 `<html>` 元素添加/移除 `.dark` 类：

```typescript
document.documentElement.classList.toggle('dark', darkMode);
```

### 三种触发路径

**1. 启动时同步**（`App.tsx`）

```typescript
useEffect(() => {
  document.documentElement.classList.toggle('dark', getAppConfig().darkMode);
}, []);
```

从 `localStorage` 恢复用户的主题偏好。

[来源](packages/pwa/src/App.tsx#L133-L136)

**2. Header 快捷切换**（`Layout.tsx`）

```typescript
const [dark, setDark] = useState(() => {
  return document.documentElement.classList.contains('dark');
});
useEffect(() => {
  document.documentElement.classList.toggle('dark', dark);
}, [dark]);
const toggleDark = useCallback(() => setDark((d) => !d), []);
```

header 中的太阳/月亮图标按钮调用 `toggleDark`，即时生效但不持久化到配置。

[来源](packages/pwa/src/components/Layout.tsx#L53-L56)
[来源](packages/pwa/src/components/Layout.tsx#L134-L138)

**3. 设置页面保存**（`SettingsModal.tsx`）

```typescript
const [darkMode, setDarkMode] = useState(config.darkMode);
const saveGeneral = () => {
  const updated = { ...config, darkMode };
  updateAppConfig(updated);
  document.documentElement.classList.toggle('dark', darkMode);
};
```

设置页的 checkbox 控制 `darkMode`，点击"保存"时同时写入 `localStorage` 和 DOM。

[来源](packages/pwa/src/components/SettingsModal.tsx#L88-L129)
[来源](packages/pwa/src/hooks/useAppConfig.ts#L9)
[来源](packages/pwa/src/hooks/useAppConfig.ts#L32)

## 国际化与主题的关系

两者在代码层面完全解耦，但在用户体验层面交汇于**设置页面**。SettingsModal 同时包含语言选择器和暗色模式开关，共用同一条调用链路：

- 国际化：`useI18n().setLocale('ja')` → store 切换消息集 → 所有 `t()` 调用自动刷新
- 主题：`setDarkMode(true)` + `saveGeneral()` → `document.documentElement.classList.toggle('dark', true)` → CSS 变量切换 → 所有 `var(--color-*)` 自动刷新

两者都是**即时响应**的，无需页面刷新。
