# 🚀 快速开始

5 分钟搭建双界面 Bluesky 客户端——**终端版**给键盘党，**浏览器版**给所有人。两个界面共享同一套业务逻辑，选一个入口即可。

---

## 先认识两个界面

这个项目有两条使用路径，本质上是同一套代码的两种呈现方式：

| 路径 | 适合谁 | 核心体验 |
| --- | --- | --- |
| **TUI（终端）** | 命令行重度用户、Vim 流派 | 全键盘操作，Ink 渲染，Lightning 快 |
| **PWA（浏览器）** | 所有人 | 图形界面，可安装到桌面，离线可用 |

无论选哪条路，你的 Bluesky 凭据和 AI API Key 都不经过任何第三方服务器——所有请求直接从你的设备发出。[来源](../README.md#L4-L5)

---

## 一、TUI 路径（终端版）

### 前置条件

- **Node.js** ≥ 18
- **pnpm**（首选包管理器，与 `pnpm-lock.yaml` 一致）

### 第一步：克隆仓库

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
```

[来源](../README.md#L82-L84)

### 第二步：安装依赖

项目是 pnpm monorepo，包含 4 个包：`@bsky/core`、`@bsky/app`、`@bsky/tui`、`@bsky/pwa`。

```bash
pnpm install
```

这会安装所有包的依赖，耗时约 30 秒。[来源](../README.md#L84)

### 第三步：编译子包

```bash
pnpm -r build
```

编译 `core` 和 `app` 两个基础包。只需执行一次，后续改代码时会自动增量编译。[来源](../README.md#L84)

### 第四步：配置环境变量

复制模板文件，填入你的 Bluesky 凭据：

```bash
cp .env.example .env
```

`.env` 文件放在**项目根目录**，TUI 启动时自动从此路径加载。[来源](src/cli.ts#L21-L28)

打开 `.env`，填入以下内容：

```ini
# ============================================
# 必填项
# ============================================
BLUESKY_HANDLE=your-handle.bsky.social     # 你的 Bluesky 用户名
BLUESKY_APP_PASSWORD=your-app-password      # App Password（不是登录密码）

# ============================================
# 选填项
# ============================================
# BLUESKY_PDS=https://your-pds.example.com  # 自定义 PDS。留空自动发现 bsky.social

# ============================================
# AI 功能（非必需，不填也可正常使用 Bluesky）
# ============================================
LLM_API_KEY=sk-your-api-key                # LLM API Key（如 DeepSeek/Mistral）

# ============================================
# 翻译目标语言（非必需，默认 zh）
# ============================================
TRANSLATE_TARGET_LANG=zh  # 可选: zh/en/ja/ko/fr/de/es
```

[来源](../.env.example)

### 配置项详细说明

| 变量名 | 必需 | 说明 |
| --- | --- | --- |
| `BLUESKY_HANDLE` | ✅ | 你的 Bluesky 用户名，如 `alice.bsky.social` |
| `BLUESKY_APP_PASSWORD` | ✅ | 从 Bluesky 设置 → App Passwords 生成，**不是你的登录密码** |
| `BLUESKY_PDS` | ❌ | 自定义 PDS 地址。留空则自动解析到 `bsky.social`，详见[第三方 PDS 支持](第三方-pds-支持.md) |
| `LLM_API_KEY` | ❌ | LLM 提供商 API Key。TUI 优先读取此变量。如果要用多提供商，可以在 `bsky-tui.config.json` 中配置每个提供商的 key |
| `TRANSLATE_TARGET_LANG` | ❌ | 翻译目标语言，支持 `zh`/`en`/`ja`/`ko`/`fr`/`de`/`es` |

[来源](src/cli.ts#L55-L73)

> **注意**：`LLM_API_KEY` 环境变量优先级最高。如果设置了它，会覆盖 `bsky-tui.config.json` 中所有 per-provider 的 key。如果要用多个 LLM 提供商（如 DeepSeek 用于聊天、Mistral 用于翻译），可以不设 `LLM_API_KEY`，改在配置文件中指定。[来源](src/cli.ts#L55-L62)

### 第五步（可选）：配置 AI 模型

非凭据性的 AI 设置存储在 `bsky-tui.config.json` 中（项目根目录）。参考模板：

```bash
cp bsky-tui.config.example.json bsky-tui.config.json
```

默认配置已写好 DeepSeek 的常用参数，你可以按需修改：

```json
{
  "targetLang": "zh",
  "translateMode": "simple",
  "aiConfig": {
    "baseUrl": "https://api.deepseek.com",
    "model": "deepseek-v4-flash",
    "provider": "deepseek",
    "reasoningStyle": "reasoning_content",
    "thinkingEnabled": true,
    "visionEnabled": false
  },
  "apiKeys": {
    "deepseek": "",
    "mistral": ""
  },
  "scenarioModels": {
    "aiChat": "",
    "translate": "",
    "polish": ""
  }
}
```

[来源](../bsky-tui.config.example.json)

配置文件的完整说明请见[环境配置详解](环境配置详解.md)。

### 第六步：启动 TUI

```bash
cd packages/tui
npx tsx src/cli.ts
```

首次启动时，如果未检测到 `.env` 配置，会自动进入 **SetupWizard** 交互式引导界面，按顺序完成：Bluesky 账号 → App Password → PDS（可选）→ LLM 提供商 → 模型 → API Key → 语言 → 场景模型配置。[来源](src/cli.ts#L95-L112)

启动成功后，你将看到终端内的全功能 Bluesky 界面，按 `?` 查看全部快捷键。

## 二、PWA 路径（浏览器版）

### 方式 A：本地开发（推荐体验完整功能）

```bash
cd packages/pwa
pnpm dev
```

浏览器访问 **http://localhost:5173**。[来源](docs/PWA_GUIDE.md#L5)

在打开的页面中：

1. 点击 **Login** 按钮
2. 输入你的 Bluesky 用户名
3. 输入 **App Password**（不是登录密码）
4. 点击登录——凭据保存在浏览器 `localStorage`，不会上传到任何服务器

[来源](docs/PWA_GUIDE.md#L180-L198)

AI 功能可在页面右上角的 ⚙️ 设置中配置 API Key。

### 方式 B：直接访问在线版（零安装）

打开 **[https://ai-bsky.pages.dev](https://ai-bsky.pages.dev)**，直接在浏览器内登录即可使用。无需克隆仓库，无需 `.env` 文件。[来源](../README.md#L95)

### 特性一览

- **可安装**：Chrome/Edge 地址栏右侧点击安装图标，即可变成桌面应用
- **离线可用**：Service Worker 缓存资源，断网后仍可查看已加载内容
- **无需后端**：纯前端架构，所有 API 请求直接从浏览器发往 Bluesky [来源](docs/PWA_GUIDE.md#L26-L27)
- **深色模式**：跟随系统主题 [来源](packages/pwa/src/main.tsx#L7-L15)

## 三、首次登录详解

### 获取 App Password

无论走 TUI 还是 PWA，都需要 **App Password**（应用密码）：

1. 登录 [bsky.app](https://bsky.app)
2. 进入 **Settings** → **App Passwords**
3. 点击 **Add App Password**，取个名字如 `bsky-client`
4. 复制生成的 16 位密码——**这是唯一一次看到它**
5. 将此密码填入 `.env` 的 `BLUESKY_APP_PASSWORD`（TUI）或登录表单（PWA）

> ⚠️ **安全提醒**：App Password 与你的登录密码不同。它只能用于一个应用，可以随时撤销。**永远不要**把你的登录密码填入 `.env`。

### 获取 AI API Key（可选）

AI 功能需要 LLM 提供商的 API Key。当前支持两个提供商：

| 提供商 | 获取地址 | 默认模型 | 特点 |
| --- | --- | --- | --- |
| **DeepSeek** | [platform.deepseek.com](https://platform.deepseek.com) | `deepseek-v4-flash` | 高性价比，支持思维链 |
| **Mistral** | [console.mistral.ai](https://console.mistral.ai) | `mistral-small-latest` | 支持多模态（图片识别） |

[来源](../packages/core/src/ai/providers.json)

**TUI 路径**：把 Key 填入 `.env` 的 `LLM_API_KEY`，或在 SetupWizard 中输入。

**PWA 路径**：登录后点击 ⚙️ 设置，在 AI 配置区域填入 Key。

> 没有 API Key 也能正常使用所有 Bluesky 核心功能（时间线、发帖、搜索、私信等），只有 AI 对话和翻译功能会受限。

## 四、常见问题排查

### Q1：启动 TUI 时提示"模块未找到"

```bash
Error: Cannot find module '@bsky/app'
```

**原因**：monorepo 子包未编译。运行一次构建：
```bash
pnpm -r build
```

### Q2：TUI 启动后一片空白/闪退

**原因**：终端不支持 raw mode。检查：
- 确保终端支持 TTY（Windows 推荐 **Windows Terminal**，不要用 CMD）
- 尝试在 Git Bash 或 WSL 中运行

### Q3：登录报错 "Invalid handle or password"

- 确认 `.env` 中 `BLUESKY_HANDLE` 写的是完整的 handle（如 `user.bsky.social`），不是仅用户名
- 确认 App Password 中**没有空格**
- 确认 Bluesky 服务器状态：https://bsky.statuspage.io

### Q4：PWA 在线版无法登录

- 检查是否被广告拦截器屏蔽了请求
- 尝试在无痕模式下打开 [ai-bsky.pages.dev](https://ai-bsky.pages.dev)
- 如果使用自定义 PDS，请确保 PDS 的 CORS 头已正确配置，详见[第三方 PDS 支持](第三方-pds-支持.md)

### Q5：AI 功能提示 "API key not configured"

**TUI**：检查 `.env` 中 `LLM_API_KEY` 是否正确设置，或在 `bsky-tui.config.json` 中填写对应提供商的 `apiKeys`。[来源](src/cli.ts#L55-L62)

**PWA**：点击 ⚙️ 设置，在 AI API Key 输入框中填入 Key 并保存。

### Q6：翻译功能不工作

- 确认 `.env` 中的 `TRANSLATE_TARGET_LANG` 或配置文件中的 `targetLang` 已设置
- 确认 AI API Key 有效且余额充足
- 翻译使用 LLM 完成，需要配置 AI 功能
