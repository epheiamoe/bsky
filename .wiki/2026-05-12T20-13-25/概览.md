# 概览

## 一句话说清这个项目

一个 Bluesky 社交媒体客户端，但你可以在**终端**和**浏览器**里同时使用它。背后有 AI 助手的全部能力——分析帖子、总结讨论串、管理列表、润色草稿——所有操作都在你的设备上完成，不经过任何第三方服务器。

[来源](README.md#L1-L5)

---

## 双界面：同一套大脑，两个身体

这个项目最独特的设计：**TUI（终端界面）** 和 **PWA（渐进式网页应用）** 共享同一套业务逻辑和 React Hooks，只是渲染层不同。

| 维度 | TUI | PWA |
|------|-----|-----|
| **运行环境** | 终端/SSH | 浏览器/手机 |
| **渲染引擎** | Ink（React for terminal） | React DOM |
| **安装方式** | `pnpm tui` | 访问 URL 或安装到桌面 |
| **输入方式** | 键盘快捷键 | 鼠标/触屏 + 键盘 |
| **存储实现** | JSON 文件 | IndexedDB |
| **适合谁** | CLI 爱好者、SSH 远程操作 | 日常浏览、手机用户 |
| **启动入口** | `packages/tui/src/cli.ts` | `packages/pwa/src/main.tsx` |

无论你从哪个界面登录，你的 Bluesky 数据、AI 对话历史、草稿、书签都通过统一的接口访问。TUI 和 PWA 的关系就像同一台车的方向盘和触控屏——操作方式不同，引擎是同一台。

[来源](docs/ARCHITECTURE.md#L5-L61)

## 三大核心价值

### 🤖 AI 驱动

内置一个完整的 AI 对话引擎（`AIAssistant` 类），支持多轮工具调用。AI 可以直接读取你的时间线、分析讨论串、搜索用户、管理列表。每次写操作（发帖、点赞、拉黑）都需要你手动确认——AI 只有建议权，你才是最终决策者。

支持 OpenAI、Anthropic、Google Gemini 等多个 LLM 提供商，BYOK（自带密钥），密钥只存在你的浏览器/终端里。

详见 [AI 对话引擎](ai-对话引擎.md)、[38 个 AI 工具系统](33-个-ai-工具系统.md)、[AI 系统提示词与多提供商](ai-系统提示词与多提供商.md)。

[来源](README.md#L33-L38)，[来源](packages/core/src/ai/assistant.ts#L96)

### 🔒 隐私优先

这是一个**纯前端**应用。没有后端服务器，没有数据库，没有任何中间代理。你的 Bluesky 凭证、AI API 密钥、聊天记录全部存储在本地：

- TUI 模式：存储在 `.bsky-tui/` 目录的 JSON 文件
- PWA 模式：存储在浏览器的 IndexedDB

每一次网络请求都直接从你的设备发往 Bluesky PDS 或 AI 提供商。项目本身不做任何数据中转。

详见 [存储抽象层](存储抽象层.md)、[第三方 PDS 支持](第三方-pds-支持.md)。

[来源](README.md#L99-L102)，[来源](docs/ARCHITECTURE.md#L100-L105)

### 🏗 纯前端零服务器

不需要搭建服务器、不需要配置数据库、不需要担心运维。TUI 模式只需 `git clone` + `pnpm install` 即可运行；PWA 模式直接通过 `pnpm dev` 启动开发服务器，或访问托管在 Cloudflare Pages 的在线版本。

详见 [快速开始](快速开始.md)。

[来源](README.md#L78-L95)

## 六大核心功能模块

```
          ┌─────────────────────────────────┐
          │           AI 对话引擎            │
          │    (AIAssistant + 38 个工具)      │
          └────────────┬────────────────────┘
                       │
┌──────────┐   ┌──────▼──────┐   ┌──────────┐
│ 时间线/   │   │             │   │ 私信系统  │
│ 信息流    │◄──┤  AT Protocol ├──►│ (Direct  │
│ (Feeds)   │   │  客户端     │   │ Messages)│
└──────────┘   │ (BskyClient) │   └──────────┘
               │   + 类型系统  │
┌──────────┐   └──────┬──────┘   ┌──────────┐
│ 讨论串    │          │          │ 列表/书签 │
│ (Threads) │◄─────────┴────────►│ (Lists &  │
│ + 翻译    │                    │ Bookmarks)│
└──────────┘                    └──────────┘
```

1. **AT Protocol 客户端** — `BskyClient` 类封装了所有 Bluesky API 调用，包括会话管理、JWT 自动刷新、PDS 发现、三重 `ky` 实例架构。这是整个应用的网络基石。

2. **时间线与信息流** — 支持 Following、Discover、自定义 Feed 三种视图，采用虚拟滚动（`@tanstack/react-virtual`）保证海量数据下的流畅滚动。每个帖子卡片展示作者头像、富文本、图片、外部链接。

3. **讨论串与翻译** — 将 Bluesky 的嵌套回复结构展平为线性列表（`FlatLine` 格式），支持一键翻译整条讨论串到目标语言，7 种语言可选。

4. **AI 对话引擎** — 34 个读工具 + 4 个写工具，AI 可以读写 Bluesky 的绝大部分功能。写操作前触发确认门（confirmation gate），防止误操作。支持流式 SSE 响应和可见式思考过程。

5. **私信系统** — 基于 `chat.bsky.convo.*` API 的直接消息实现，支持表情反应、引用帖子嵌入、后台轮询新消息、消息删除与静音。

6. **列表与书签** — 创建策展列表（自定义信息源）或 moderation lists（批量静音），管理成员、浏览列表时间线。书签功能让你收藏任何帖子以备后用。

详见 [AT Protocol 客户端](at-protocol-客户端.md)、[AT Protocol 类型系统](at-protocol-类型系统.md)、[38 个 AI 工具系统](33-个-ai-工具系统.md)、[Direct Messages 私信系统](direct-messages-私信系统.md)、[虚拟滚动与滚动恢复](虚拟滚动与滚动恢复.md)。

## 四层架构：core → app → tui/pwa

项目采用 monorepo（单一仓库）组织，**pnpm workspace** 管理四个包之间的依赖关系：

```
@bsky/core  ──►  @bsky/app  ──►  @bsky/tui  (Ink 终端 UI)
                              └─►  @bsky/pwa  (React DOM 浏览器 UI)
```

### Layer 0: `@bsky/core`（零 UI 依赖）

纯 TypeScript 类与函数，不涉及任何 React 或 UI 框架。包含：

- `BskyClient` — AT Protocol 的完整封装
- `AIAssistant` — 多轮工具调用的 AI 对话引擎
- `createTools` — 38 个 AI 工具的工厂函数
- 所有类型定义（`parseAtUri`、响应类型、会话类型）
- AI 系统提示词与多提供商注册表

这是整座大楼的地基。

### Layer 1: `@bsky/app`（React Hooks + 纯 Store）

20+ 个 React Hooks，每个 hook 封装一个功能领域的数据获取与状态管理。同时提供纯对象 Store（`createNavigation`、`feedConfig`、`widgetStore`）和存储接口（`ChatStorage`、`DraftStorage`）。

这一层是**可直接在 PWA 中复用**的——PWA 只需写渲染组件，所有业务逻辑由 `@bsky/app` 提供。

### Layer 2: `@bsky/tui` 与 `@bsky/pwa`（双渲染终端）

- **TUI**：使用 Ink 在终端渲染 React 组件，键盘驱动的导航系统，CJK 感知的文本换行，ANSI 鼠标追踪
- **PWA**：Vite 构建，Tailwind CSS 主题系统，哈希路由（兼容静态托管），Service Worker 离线支持

### 数据流方向

```
用户操作  ──►  视图组件  ──►  React Hook  ──►  BskyClient  ──►  Bluesky PDS / AI API
(tui/pwa)       (UI 层)       (@bsky/app)       (@bsky/core)       (远端)

                 ◄──── 响应 ────  ◄──── 响应 ────  ◄──── 响应 ────
```

所有数据从远端流向 core，经过 app 层变成 React 可消费的状态，最终在 tui 或 pwa 中渲染。反向亦然——用户操作从 UI 层向下传递到 core 层发起网络请求。

详见 [三层架构详解](三层架构详解.md)、[React Hooks 体系](react-hooks-体系.md)、[Store 订阅模式](store-订阅模式.md)、[TUI 入口与配置加载](tui-入口与配置加载.md)、[PWA 应用架构](pwa-应用架构.md)。

## 国际化与可访问性

内置三语言支持：**中文**、**English**、**日本語**。运行时可在界面中即时切换，无需刷新。所有 UI 文本通过键值对管理，支持插值语法。

PWA 模式遵循系统深色/浅色主题，安装后可作为独立应用离线运行。

详见 [国际化 i18n 系统](国际化-i18n-系统.md)。
