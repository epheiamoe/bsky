# 导航与状态管理

**核心文件**：`packages/app/src/state/navigation.ts`
**React Hook**：`packages/app/src/hooks/useNavigation.ts`
**PWA 哈希路由**：`packages/pwa/src/hooks/useHashRouter.ts`

导航系统采用**纯状态机 + React Hook 桥接**架构：`createNavigation()` 是一个与框架无关的纯函数，返回一个订阅式导航控制器；`useNavigation()` 是一个薄封装层，将控制器状态同步到 React 渲染周期。PWA 在此基础上额外交付 `useHashRouter()`，将导航状态编码到 URL 哈希中，以适配静态托管的部署场景。

## createNavigation：纯状态机

`createNavigation()` 是一个零依赖的工厂函数，维护一个 **导航堆栈** 和 **发布-订阅** 机制。

```
┌─────────────────────────────────────┐
│         createNavigation()          │
│                                     │
│  stack: AppView[]                   │
│  ┌──────────────┐                   │
│  │ [{type:'feed'}]  ← 初始状态     │
│  │       │                          │
│  │ goTo(v) → push stack            │
│  │ goBack()  → pop stack           │
│  │ goHome()  → reset stack         │
│  └──────────────┘                   │
│                                     │
│  subscribe(fn) → unsubscribe()      │
│  notify() → 遍历 listeners          │
└─────────────────────────────────────┘
```

### 核心行为

| 方法 | 行为 | 边界条件 |
|------|------|----------|
| `goTo(view)` | 将新视图推入堆栈 | 总是成功 |
| `goBack()` | 弹出栈顶元素 | 当 `stack.length > 1` 时执行，否则无操作 |
| `goHome()` | 重置为 `[{ type: 'feed' }]` | 无条件重置 |
| `subscribe(fn)` | 注册状态变更监听器 | 返回取消注册函数 |

`canGoBack` 由 `stack.length > 1` 计算得出，当且仅当堆栈深度大于 1 时才可返回。[来源](packages/app/src/state/navigation.ts#L30-L74)

### 为什么是纯状态机？

核心动机是**跨界面复用**。TUI 终端和 PWA 浏览器共享同一套导航状态逻辑，但渲染层完全不同：

- TUI 直接调用 `createNavigation()`，状态存在于内存中
- PWA 通过 `useHashRouter` 将状态编码到 URL，确保刷新不丢失位置

## useNavigation：React Hook 桥接

`useNavigation()` 是对 `createNavigation()` 的 React 封装，采用懒初始化模式：

```typescript
// 来源：packages/app/src/hooks/useNavigation.ts#L5-L20
export function useNavigation() {
  const [nav] = useState(() => createNavigation());
  const [state, setState] = useState(() => nav.getState());
  const tick = useCallback(() => setState(nav.getState()), [nav]);
  useEffect(() => nav.subscribe(tick), [nav, tick]);
  return { currentView, canGoBack, goTo, goBack, goHome };
}
```

关键设计决策：

- **`useState(() => createNavigation())`** — `createNavigation` 只在组件挂载时执行一次，不会因重渲染而丢失堆栈状态。`nav` 的引用在组件整个生命周期内保持不变。
- **`subscribe(tick)`** — 每次导航状态变更触发 `notify()`，驱动 React 重新渲染。订阅在 `useEffect` 清理阶段自动取消。
- **返回值仅暴露快照** — 外部调用 `goTo/goBack/goHome` 时状态变更通过订阅反馈，而非直接修改 `state`。[来源](packages/app/src/hooks/useNavigation.ts#L5-L20)

这个 Hook 是 `@bsky/app` 包公共 API 的一部分，被 TUI 和 PWA 共同引用。[来源](packages/app/src/index.ts#L4)

## TUI vs PWA：两种导航策略

同一套 `AppView` 联合类型，TUI 和 PWA 采用截然不同的存储和恢复策略。

| 维度 | TUI | PWA |
|------|-----|-----|
| **状态载体** | 内存变量（闭包堆栈） | URL 哈希（`#/path?params`） |
| **导航驱动** | `useNavigation()` → `createNavigation()` | `useHashRouter()` → `window.history.pushState` |
| **刷新持久性** | ❌ 丢失（终端无需） | ✅ 保留（浏览器 F5 可恢复） |
| **后退/前进** | 栈内 `pop` 操作 | 浏览器 `history.back()`/`forward()` |
| **深层链接** | ❌ 不支持 | ✅ 支持（可通过 URL 直接跳转到任意视图） |
| **订阅机制** | `subscribe/notify` | `popstate` 事件监听 |

### PWA 哈希路由设计（`useHashRouter`）

PWA 放弃 `useNavigation()` 而采用独立的 `useHashRouter()`，原因见架构文档：

> **Hash-based routing (useHashRouter)** — PWA uses `history.pushState` + `popstate` with `#/path` format for static hosting compatibility; `useHashRouter` hook encodes/decodes `AppView` to/from hash URLs

[来源](docs/ARCHITECTURE.md#L111)

哈希路由的核心是两个互为逆函数的转换器：

```
parseHash(hash) → AppView
encodeView(view) → hash
```

#### 默认 Feed 重定向

进入 `#/feed` 或空哈希时，`useHashRouter` 会自动解析默认 Feed URI：

```typescript
// 来源：packages/pwa/src/hooks/useHashRouter.ts#L29-L33
const defFeed = getFeedConfig().defaultFeedUri ?? BUILTIN_FEEDS.following;
window.history.replaceState(null, '', `#/feed?feed=${encodeURIComponent(defFeed)}`);
```

优先级链：`getFeedConfig().defaultFeedUri` → `BUILTIN_FEEDS.following`。[来源](packages/pwa/src/hooks/useHashRouter.ts#L29-L33)

#### goTo 中的 Feed 消歧

当 `goTo({ type: 'feed' })` 不带 `feedUri` 时，`useHashRouter` 自动补全为上次活跃的 Feed URI 或默认 Feed。[来源](packages/pwa/src/hooks/useHashRouter.ts#L46-L58)

## 设计约束与架构决策

1. **`@bsky/app` 层零 UI 依赖** — `createNavigation()` 是纯函数，不依赖 React 或任何渲染框架，可在任何 JS 环境复用。[来源](docs/ARCHITECTURE.md#L100-L101)
2. **PWA 用 `useHashRouter` 而非 `useNavigation`** — 虽然两者都暴露 `{ currentView, canGoBack, goTo, goBack, goHome }` 接口，但 PWA 需要 URL 同步。`useNavigation` 的内存堆栈无法在页面刷新后恢复。[来源](packages/pwa/src/hooks/useHashRouter.ts#L6-L11)
3. **`goHome()` 总是回到 Feed** — 无论当前在哪个视图，`goHome()` 重置堆栈为 `[{ type: 'feed' }]`，在 PWA 中还会自动补全默认 Feed URI。[来源](packages/pwa/src/hooks/useHashRouter.ts#L66-L71)
4. **`canGoBack` 的语义差异** — TUI 基于 `stack.length > 1`，PWA 基于 `window.history.length` 和当前哈希值，两者行为在页面级导航时一致但在边界条件上略有差别。[来源](packages/app/src/state/navigation.ts#L37) 和 [来源](packages/pwa/src/hooks/useHashRouter.ts#L23-L24)
