# 概览

## 一句话说清这个项目

一个 Bluesky 社交媒体客户端，但你可以在**终端**和**浏览器**里同时使用它。AI 助手能分析帖子、总结讨论串、管理列表、润色草稿——甚至搜索网页、查 Wikipedia、抓取任意 URL——所有操作都在你的设备上完成，不经过任何第三方服务器。

## 双界面：同一套大脑，两个身体

这个项目最独特的设计：**TUI（终端界面）** 和 **PWA（渐进式网页应用）** 共享同一套业务逻辑和 React Hooks，只是渲染层不同。

| 维度 | TUI | PWA |
|------|-----|-----|
| **运行环境** | 终端/SSH | 浏览器/手机 |
| **渲染引擎** | Ink（React for terminal） | React DOM + Tailwind CSS |
| **安装方式** | `pnpm install && cd packages/tui && npx tsx src/cli.ts` | 访问 `ai-bsky.pages.dev` 或安装到桌面 |
| **输入方式** | 键盘快捷键 | 鼠标/触屏 + 键盘 |
| **存储实现** | JSON 文件（`~/.bsky-tui/`） | IndexedDB |
| **适合谁** | CLI 爱好者、SSH 远程操作 | 日常浏览、手机用户 |
| **启动入口** | `packages/tui/src/cli.ts` | `packages/pwa/src/main.tsx` |

无论从哪个界面登录，Bluesky 数据、AI 对话历史、草稿、书签都通过统一接口访问。TUI 和 PWA 的关系就像同一台车的方向盘和触控屏——操作方式不同，引擎是同一台。

## 三大核心价值

### 🤖 AI 驱动

内置一个完整的 AI 对话引擎（`AIAssistant` 类），支持多轮工具调用（无限轮次，用户可通过暂停/停止控制）。AI 可以直接读取你的时间线、分析讨论串、搜索用户、管理列表，还能搜索网页（DuckDuckGo，无需 API 密钥）、查 Wikipedia、提取任意 URL 内容。**33 个工具**（27 个读工具 + 6 个写工具）覆盖了 Bluesky 的绝大部分功能。每次写操作（发帖、点赞、拉黑）都需要你手动确认——AI 只有建议权，你才是最终决策者。

支持 OpenAI、Anthropic、Google Gemini、DeepSeek、Mistral 等多个 LLM 提供商，BYOK（自带密钥），密钥只存在你的浏览器/终端里。

详见 [AI 对话引擎](ai-对话引擎.md)、[33 个 AI 工具系统](33-个-ai-工具系统.md)、[AI 系统提示词与多提供商](ai-系统提示词与多提供商.md)。

[来源](README.md#L23-L26)，[来源](packages/core/src/ai/assistant.ts#L96)，[来源](packages/core/src/ai/tools.ts#L75-L1120)

### 🔒 隐私优先

这是一个**纯前端**应用。没有后端服务器，没有数据库，没有任何中间代理。你的 Bluesky 凭证、AI API 密钥、聊天记录全部存储在本地：

- TUI 模式：存储在 `~/.bsky-tui/` 目录的 JSON 文件
- PWA 模式：存储在浏览器的 IndexedDB

每一次网络请求都直接从你的设备发往 Bluesky PDS 或 AI 提供商。项目本身不做任何数据中转。

## 六大核心功能模块

```
          ┌─────────────────────────────────┐
          │           AI 对话引擎            │
          │  (AIAssistant + 33 个工具)       │
          └────────────┬────────────────────┘
                       │
┌──────────┐   ┌──────▼──────┐   ┌──────────┐
│ 时间线/   │   │             │   │ 私信系统  │
│ 信息流    │◄──┤  AT Protocol ├──►│ (Direct  │
│ (Feeds)   │   │  客户端     │   │ Messages)│
└──────────┘   │ (BskyClient) │   └──────────┘
               │   + 类型系统  │
┌──────────┐   └──────┬──────┘   ┌──────────┐
│ 讨论串    │          │          │ 列表/书签 │
│ (Threads) │◄─────────┴────────►│ (Lists &  │
│ + 翻译    │                    │ Bookmarks)│
└──────────┘                    └──────────┘
```

1. **AT Protocol 客户端** — `BskyClient` 类封装了所有 Bluesky API 调用，包括会话管理、JWT 自动刷新、PDS 发现、三重 `ky` 实例架构。这是整个应用的网络基石。

2. **时间线与信息流** — 支持 Following、Discover、自定义 Feed 三种视图，采用虚拟滚动（`@tanstack/react-virtual`）保证海量数据下的流畅滚动。每个帖子卡片展示作者头像、富文本、图片、外部链接。

3. **讨论串与翻译** — 将 Bluesky 的嵌套回复结构展平为线性列表（`FlatLine` 格式），支持一键翻译整条讨论串到目标语言，7 种语言可选，支持 simple（纯文本）和 json（结构化 + 源语言检测）双模式。

4. **AI 对话引擎** — 27 个读工具 + 6 个写工具，AI 可以读写 Bluesky 的绝大部分功能，还能独立搜索网页（DuckDuckGo）、查 Wikipedia、提取任意 URL 内容（无需任何 API 密钥）。写操作前触发确认门（confirmation gate），防止误操作。支持流式 SSE 响应和可见式思考过程，支持暂停/停止。

5. **私信系统** — 基于 `chat.bsky.convo.*` API 的直接消息实现，支持表情反应（可配置 Emoji 列表）、引用帖子嵌入、后台轮询新消息、消息删除与静音。

6. **列表与书签** — 创建策展列表（自定义信息源）或 moderation lists（批量静音），管理成员、浏览列表时间线。书签功能让你收藏任何帖子以备后用。

详见 [AT Protocol 客户端](at-protocol-客户端.md)、[AT Protocol 类型系统](at-protocol-类型系统.md)、[33 个 AI 工具系统](33-个-ai-工具系统.md)、[Direct Messages 私信系统](direct-messages-私信系统.md)、[虚拟滚动与滚动恢复](虚拟滚动与滚动恢复.md)。

## 五包架构：core → app → tui/pwa

项目采用 monorepo（单一仓库）组织，**pnpm workspace** 管理五个包 + `contracts/` 之间的依赖关系：

```
@bsky/ddg-search  (零依赖工具包)
       │
@bsky/core  ──►  @bsky/app  ──►  @bsky/tui  (Ink 终端 UI)
                              └─►  @bsky/pwa  (React DOM 浏览器 UI)
contracts/  (JSON 定义 + 系统提示词)
```

### Layer 0: `@bsky/core`（零 UI 依赖）

纯 TypeScript 类与函数，不涉及任何 React 或 UI 框架。包含：

- `BskyClient` — AT Protocol 的完整封装
- `AIAssistant` — 多轮工具调用的 AI 对话引擎（无限轮次）
- `createTools` — 33 个 AI 工具的工厂函数
- 所有类型定义（`parseAtUri`、响应类型、会话类型）
- AI 系统提示词与多提供商注册表
- `translateText` / `polishDraft` / `generateChatTitle` 等 AI 辅助函数

### Layer 1: `@bsky/app`（React Hooks + 纯 Store）

28 个 React Hooks，每个 hook 封装一个功能领域的数据获取与状态管理。同时提供纯对象 Store（`createNavigation`、`feedConfig`、`widgetStore`）和存储接口（`ChatStorage`、`DraftStorage`）。

这一层是**可直接在 PWA 中复用**的——PWA 只需写渲染组件，所有业务逻辑由 `@bsky/app` 提供。

### Layer 2: `@bsky/tui` 与 `@bsky/pwa`（双渲染终端）

- **TUI**：使用 Ink 在终端渲染 React 组件，键盘驱动的导航系统，CJK 感知的文本换行，ANSI 鼠标追踪
- **PWA**：Vite 构建，Tailwind CSS 主题系统，哈希路由（兼容静态托管），Service Worker 离线支持，framer-motion 动画，HLS 视频播放，Mermaid 图表渲染

### 工具包: `@bsky/ddg-search`

零依赖纯函数包：解析 DuckDuckGo Lite 搜索结果 HTML，提取真实 URL。由 `@bsky/core` 的 `search_web_ddg` AI 工具在 jina.ai 不可用时作为备用方案。

### 共享定义: `contracts/`

包含 `tools.json`（全部 33 个工具的 JSON Schema 定义）和 `system_prompts.md`（AI 系统提示词集中管理）。

### 数据流方向

```
用户操作  ──►  视图组件  ──►  React Hook  ──►  BskyClient  ──►  Bluesky PDS / AI API
(tui/pwa)       (UI 层)       (@bsky/app)       (@bsky/core)       (远端)

                 ◄──── 响应 ────  ◄──── 响应 ────  ◄──── 响应 ────
```

所有数据从远端流向 core，经过 app 层变成 React 可消费的状态，最终在 tui 或 pwa 中渲染。反向亦然——用户操作从 UI 层向下传递到 core 层发起网络请求。
