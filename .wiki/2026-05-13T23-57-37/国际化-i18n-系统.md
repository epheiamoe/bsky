# 国际化（i18n）系统

国际化系统为这个双界面（TUI/PWA）Bluesky 客户端提供 **三语言运行时切换** 能力，覆盖中文、English、日本語。整个系统只有 8 个源文件，采用极简的「键值对字典 + 单例 Store + React Hook 桥接」三明治架构。

## 2. `useI18n` Hook 使用方法

### 2.1 返回值签名

```typescript
const {
  t,                 // (key: string, params?: Record<string, string | number>) => string
  locale,            // 当前语言 'zh' | 'en' | 'ja'
  setLocale,         // (l: Locale) => void
  availableLocales,  // ['zh', 'en', 'ja']
  localeLabels,      // { zh: '中文', en: 'English', ja: '日本語' }
} = useI18n(initialLocale?);
```

[来源](packages/app/src/i18n/useI18n.ts#L6-L20)

### 2.2 基础用法

**纯文本键** — 直接传入键名获取翻译：

```tsx
const { t } = useI18n();
return <Button>{t('action.like')}</Button>;
// 中文环境 → "赞"，English → "Like"，日本語 → "いいね"
```

**带参数的键** — 传入占位符参数：

```tsx
const { t } = useI18n();
return <span>{t('thread.replyCount', { n: 5 })}</span>;
// 中文 → "回复 (5)"，English → "Replies (5)"
```

**切换语言**：

```tsx
const { t, locale, setLocale, availableLocales, localeLabels } = useI18n();
// 渲染语言选择器
availableLocales.map(l => (
  <button key={l} onClick={() => setLocale(l)}>
    {localeLabels[l]}
  </button>
));
```

**TUI 中指定初始语言** — 从 config 读取：

```tsx
const { t, locale } = useI18n(config.targetLang as Locale);
```

[来源](packages/app/src/i18n/useI18n.ts#L6-L20), [来源](packages/tui/src/components/App.tsx#L75-L75)

### 2.3 Hook 实现原理

`useI18n` 没有使用 React Context。它基于模块级单例 `I18nStore` + `useState(0)` 触发重渲染：

```typescript
export function useI18n(initialLocale?: Locale) {
  const store = getI18nStore(initialLocale);
  const [, force] = useState(0);            // 计数器的唯一用途是触发重渲染
  const tick = useCallback(() => force(n => n + 1), []);

  useEffect(() => store.subscribe(tick), [store, tick]);
  // ...
}
```

当 `setLocale()` 被调用时，Store 遍历所有已订阅的 listener（即各个组件的 `tick` 函数），触发 `force(n+1)` 让组件重渲染，组件重新调用 `store.t(key)` 获取新语言文本。

这种模式的优势是不需要 Provider 包裹，任何组件随时可调用 `useI18n()`；缺点是没有 Provider 层的自动透传，每个消费组件都需要显式调用 hook。这在双界面架构中是可接受的权衡——见 [Store 订阅模式](store-订阅模式.md) 的详细说明。

[来源](packages/app/src/i18n/useI18n.ts#L6-L12)

## 3. 插值语法与陷阱

### 3.1 插值引擎

`interpolate()` 函数使用正则 `/\\{(\\w+)\\}/g` 匹配 `{key}` 格式：

```typescript
function interpolate(text: string, params?: Record<string, string | number>): string {
  if (!params) return text;
  return text.replace(/\\{(\\w+)\\}/g, (_, key) => {
    return params[key] !== undefined ? String(params[key]) : `{${key}}`;
  });
}
```

逻辑：替换 `{变量名}` 为对应参数值，若参数缺失则保留原样。

[来源](packages/app/src/i18n/store.ts#L17-L22)

### 3.2 语言文件中的插值示例

| 键 | 模板 | 调用 | 结果 |
|----|------|------|------|
| `compose.charCount` | `{current}/{max}` | `t('compose.charCount', {current:120, max:300})` | `120/300` |
| `thread.replyCount` | `回复 ({n})` | `t('thread.replyCount', {n:5})` | `回复 (5)` |
| `compose.maxImages` | `最多 {n} 个媒体` | `t('compose.maxImages', {n:4})` | `最多 4 个媒体` |
| `compose.imageCompressed` | `图片已自动压缩: {name} {orig} → {comp} ({ratio}%)` | 多参数 | 完整压缩信息 |
| `lists.memberCount` | `{n} 人` | `t('lists.memberCount', {n:12})` | `12 人` |

### 3.3 ⚠️ 单大括号陷阱（Lesson 12）

**「Lesson 12: `{{n}}` vs `{n}` — i18n 模板插值陷阱」** 记录了项目早期的一个典型 Bug：

> **问题**：列表人数显示为 `{1}` 而非 `1`（花括号留在屏幕上）。
>
> **根因**：i18n 的 `interpolate()` 使用正则 `/\\{(\\w+)\\}/g`，匹配单大括号。`'{{n}}'` 在模板中 —— 外括号不匹配 `\\{` 和 `\\}`（因为正则期待 `\\{` 后紧跟 `\\w+`），内括号被匹配 → 替换为 `1`，外括号残留 → `{1}`。
>
> ```json
> // ❌ "{{n}} members"  → 显示为 "{5} members"
> // ✅ "{n} members"     → 显示为 "5 members"
> ```

**教训**：i18n 插值必须使用 `{n}`（单大括号），不要使用 `{{n}}`（双大括号）。新增任何带参数的键时必须遵循这一约定。项目中所有现有插值（`ai.messageCount`、`thread.replyCount`、`compose.charCount` 等）都使用单大括号。

[来源](docs/LESSONS.md#L263-L275), [来源](docs/CONTEXT.md#L401-L401)

## 4. I18nStore：单例引擎

`createI18nStore` 创建的对象包含四个核心方法：

| 方法 | 行为 |
|------|------|
| `t(key, params?)` | 翻译：先查当前语言 → fallback 到 en → fallback 到 zh → 返回 key 本身 |
| `setLocale(locale)` | 切换语言并通知所有订阅者 |
| `subscribe(fn)` / `unsubscribe(fn)` | 发布-订阅机制，供 hook 使用 |
| `getLocale()` | 返回当前语言 |

**三级 fallback 链**确保任何语言文件缺失的键都会有兜底值：

```typescript
t(key: string, params?: Record<string, string | number>): string {
  const raw = store.messages[key];
  if (raw) return interpolate(raw, params);       // 1. 当前语言
  const en = allMessages.en[key];
  if (en) return interpolate(en, params);          // 2. fallback English
  const zh = allMessages.zh[key];
  if (zh) return interpolate(zh, params);          // 3. fallback 中文
  return key;                                       // 4. 返回键名本身
}
```

## 5. 新增语言键的完整流程

以新增一个「草稿同步中」提示为例：

### Step 1：在三个语言文件中添加键

```typescript
// packages/app/src/i18n/locales/zh.ts
'drafts.syncingWithProgress': '同步中 ({current}/{total})',

// packages/app/src/i18n/locales/en.ts
'drafts.syncingWithProgress': 'Syncing ({current}/{total})',

// packages/app/src/i18n/locales/ja.ts
'drafts.syncingWithProgress': '同期中 ({current}/{total})',
```

### Step 2：在组件中使用

```tsx
const { t } = useI18n();
const label = t('drafts.syncingWithProgress', { current: done, total: all });
```

### 要点检查清单

- [x] 键名遵循 `{领域}.{具体名}` 点号命名法
- [x] 参数占位符使用 `{n}` 而非 `{{n}}`
- [x] 三个语言文件都添加了键（至少 en.ts 必须有——作为 fallback 中枢）
- [x] 如果键不需要翻译变化（如仅数字差异），中文/日文可直接复用英文模板
