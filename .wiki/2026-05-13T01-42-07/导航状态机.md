# 导航状态机

导航是 bsky 双端（TUI + PWA）共用的核心子系统。它不依赖 React Router 或 Ink Router，而是一个 **纯 JS 的 Stack 状态机**，再分别用 Hook 桥接到 React 和 Ink 的渲染循环。

## AppView 联合类型：16 种视图

所有路由目标统一为 `AppView` 联合类型，每个成员通过 `type` 字面量区分：

```typescript
type AppView =
  | { type: 'feed'; feedUri?: string }
  | { type: 'detail'; uri: string }
  | { type: 'thread'; uri: string }
  | { type: 'compose'; replyTo?: string; quoteUri?: string; draftId?: string; initialText?: string }
  | { type: 'profile'; actor: string; profileTab?: string }
  | { type: 'notifications' }
  | { type: 'search'; query?: string; searchTab?: string }
  | { type: 'aiChat'; contextUri?: string; sessionId?: string; contextPost?: string; contextProfile?: string }
  | { type: 'bookmarks' }
  | { type: 'drafts' }
  | { type: 'components' }
  | { type: 'dm' }
  | { type: 'dmChat'; conversationId: string }
  | { type: 'lists'; actor?: string }
  | { type: 'listDetail'; uri: string; tab?: 'posts' | 'members' }
  | { type: 'about' }
  | { type: 'atplay' }
  | { type: 'atplaySocialCircle' };
```

联合类型的每个分支都携带该视图必需的数据载荷——`thread` 需要 `uri`，`compose` 可带 `replyTo` / `quoteUri` / `draftId`，`profile` 可指定 `profileTab` 等。TypeScript 的 **discriminated union** 保证了视图切换时绝不会传递不兼容的参数。

`feed` 变体支持 `feedUri?` 字段以切换自定义 Feed 源；省略时 TUI 使用默认 Feed 源，PWA 则自动解析为 last active feed 或默认 Feed 源。

## Stack 栈机制：push / pop / reset

导航状态机核心是一个 **不可变数组栈**：

```typescript
let stack: AppView[] = [{ type: 'feed' }];
```

三个操作函数：

| 操作 | 效果 | 边界条件 |
|------|------|----------|
| `goTo(view)` | `stack = [...stack, view]` 入栈 | 无限制，可无限深入 |
| `goBack()` | `stack = stack.slice(0, -1)` 弹出栈顶 | `stack.length > 1` 时才执行 |
| `goHome()` | `stack = [{ type: 'feed' }]` 重置为初始栈 | 无条件重置 |

**当前视图**由栈顶元素定义：`currentView = stack[stack.length - 1]`。**能否返回**由栈深度判断：`canGoBack = stack.length > 1`。

状态变更后通过 **单监听器发布-订阅模式** 通知所有订阅者：

```typescript
function notify() {
  listeners.forEach(fn => fn());
}
```

每个 `subscribe()` 返回取消函数，组件卸载时自动清理。

## 状态转换图

```
feed ──Enter──▶ thread ──c──▶ compose (replyTo=uri)
  ▲                │  ▲            │
  │──Esc───────────┘  │──Esc───────┘
  │                   │
  │                   ──Enter──▶ thread (切换焦点帖子)
  │                               ▲  │
  │──────────────────Esc──────────┘  │
  │
  ├──a / Ctrl+G──▶ aiChat
  │                  ▲  │
  │                  └──Esc──▶ goBack()
  │
  ├──n──▶ notifications ──Esc──▶ goBack()
  ├──p──▶ profile ──Esc──▶ goBack()
  ├──b──▶ bookmarks ──Enter──▶ thread
  ├──L──▶ lists ──Enter──▶ listDetail
  ├──m──▶ dm ──Enter──▶ dmChat
  ├──s──▶ search ──Enter──▶ thread/profile/feed
  ├──c──▶ compose (thread 模式下创建引用)
  ├──,──▶ settings (仅 TUI 侧)
  └──?──▶ about
```

关键设计点：

- **feed 是唯一锚点**：`goHome()` 会回收所有栈帧直接回到 `feed`，其他视图最终都通过 `Esc` 链式回退到 feed。
- **compose 上锁**：compose 模式下，全局导航快捷键（a/s/b 等）全部静默，防止用户正在编辑时意外跳转。compose 必须通过 `goBack()` 或提交后 `goHome()` 退出。
- **aiChat 双层 Esc**：第一层 Esc 解除 AI 面板聚焦（切回主面板），第二层 Esc 执行 `goBack()`。Tab 在 aiChat 中切换 `'main' ↔ 'ai'` 聚焦目标。

### PWA 视图差异

PWA 通过 Hash URL 承载状态（`#/thread?uri=...`等），没有键盘快捷键系统。导航通过按钮点击和浏览器原生返回键驱动。`goBack()` 直接调用 `window.history.back()` 以保持与浏览器历史的一致性。

## PWA 的 Hash 路由 vs TUI 的 Ink 路由

两者共享同一个 `createNavigation()` 状态机，但在渲染层有不同的路由适配策略。

| 维度 | TUI (Ink) | PWA (Hash) |
|------|-----------|------------|
| **渲染框架** | Ink `<Box>` 组件树 | React DOM `<div>` 组件树 |
| **路由实现** | `useNavigation()` Hook + `createNavigation()` | `useHashRouter()` Hook + `window.history.pushState` |
| **地址表示** | 无 URL——纯内存栈 | Hash URL：`#/thread?uri=at://...` |
| **返回行为** | `goBack()` 弹出栈顶 | `goBack()` → `window.history.back()` 触发 popstate |
| **视图渲染** | `switch(currentView.type)` 选择 Ink 组件 | `switch(currentView.type)` 选择 React 组件 |
| **浏览器历史** | 不存在 | 使用 `pushState` / `popstate` 同步 |
| **刷新保持** | 终端关闭后丢失 | Hash URL 可被浏览器持久化，刷新后 `parseHash()` 重建状态 |
| **裸 Feed 路由** | 直接使用 `{ type: 'feed' }` | 自动重写为 `#/feed?feed=<defaultFeedUri>` |

**核心差异**：PWA 的 `useHashRouter` 包裹了一层 `createNavigation` 不存在的 `parseHash/encodeView` 序列化层：

```typescript
// PWA goTo 逻辑：
const hash = encodeView(view);              // { type: 'thread', uri } → '#/thread?uri=...'
window.history.pushState(null, '', hash);    // 更新浏览器地址栏
setCurrentView(view);                        // 更新 React 状态
setCanGoBack(true);                          // 手动标记可返回
```

这使得 PWA 支持 **页面刷新后恢复**——浏览器加载 `#/thread?uri=...` 时，`parseHash()` 将其反序列化为 `AppView` 对象。而 TUI 不具备持久化能力，启动时始终回到 `[{ type: 'feed' }]`。

另外 PWA 的 `goHome()` 有一个关键差异：它会 push 一个带默认 Feed URI 的新地址到历史栈，而非像 TUI 那样直接 `stack = [{ type: 'feed' }]`。这是为了让 **浏览器后退键能回到之前的视图**。

## goTo / goBack / goHome 的典型调用链

### goTo：从列表点击到帖子详情

```
用户按 Enter (TUI) 或点击 (PWA)
  → App.tsx: key.return handler
    → goTo({ type: 'thread', uri: post.uri })
      → navigation.goTo(): stack = [...stack, view], notify()
        → useNavigation().tick(): setState(nav.getState())
          → React re-render, currentView.type === 'thread'
            → TUI: <UnifiedThreadView uri={post.uri} ... />
            → PWA: <ThreadView uri={post.uri} ... />
```

### goTo：从帖子进入带上下文的 AI 对话

```
用户按 a (TUI) 或点击 AI 图标 (PWA)
  → goTo({ type: 'aiChat', sessionId: crypto.randomUUID(), contextPost: threadUri })
    → createNavigation().goTo() → notify()
      → PWA 额外执行：hash = '#/ai?session=...&post=...', history.pushState
```

### goBack：Esc 链式回退

```
用户按 Esc
  → App.tsx: if (key.escape)
    → 检查各种 overlay 状态（feed config / draft save prompt / 等）
    → if (currentView.type !== 'feed') { goBack(); return; }
      → navigation.goBack(): stack = stack.slice(0, -1), notify()
        → TUI: React re-render 展示上一个视图
        → PWA: window.history.back() → popstate 事件 → parseHash() → setCurrentView()
```

```
用户按 t (TUI) 或点击 Home (PWA)
  → goHome()

```

## React Hook 桥接

`useNavigation()` 是连接纯状态机到 React 渲染的桥梁：

```typescript
export function useNavigation() {
  const [nav] = useState(() => createNavigation());  // 惰性初始化，组件生命周期内只创建一次
  const [state, setState] = useState(() => nav.getState());

  const tick = useCallback(() => setState(nav.getState()), [nav]);

  useEffect(() => nav.subscribe(tick), [nav, tick]);  // 订阅状态变更

  return {
    currentView: state.currentView,
    canGoBack: state.canGoBack,
    goTo: (v: AppView) => nav.goTo(v),
    goBack: () => nav.goBack(),
    goHome: () => nav.goHome(),
  };
}
```

这个 Hook 在 `packages/app` 层定义，TUI 和 PWA 都能导入使用。TUI 的 `App.tsx` 直接调用它，PWA 的 `App.tsx` 调用的是 `useHashRouter()`——后者为保持浏览器地址栏同步包裹了额外逻辑，但暴露的接口签名相同。
