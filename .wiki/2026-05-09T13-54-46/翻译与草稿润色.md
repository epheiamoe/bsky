# 翻译与草稿润色

翻译与草稿润色是应用中两个轻量级的 **单轮 AI 工具调用**，不依赖完整的 `AIAssistant` 会话管线。两者都通过 `singleTurnAI` 函数直接调用 LLM API，适用于嵌入式 UI 场景——例如发帖编辑器中的"翻译"按钮或"润色"按钮。

[来源](packages/core/src/ai/assistant.ts#L60-L61)

## 架构全景

```
┌─────────────────────────────────────────────────┐
│                   UI 层                          │
│  useTranslation (React Hook)                    │
│  ├─ translate() → 调用 core/translateText       │
│  └─ 内部缓存 Map<cacheKey, TranslationResult>   │
├─────────────────────────────────────────────────┤
│              Core 层 (packages/core)            │
│  translateText(config, text, lang, mode)        │
│    ├─ PF_TRANSLATE_SIMPLE / PF_TRANSLATE_JSON   │
│    ├─ 最多 3 次指数退避重试                      │

│  polishDraft(config, draft, requirement)        │
│    └─ singleTurnAI(P_POLISH_SYSTEM, PF_POLISH...)│
├─────────────────────────────────────────────────┤
│           LLM Provider (OpenAI-compatible)      │

```

两个函数均在 `packages/core/src/ai/assistant.ts` 中定义，通过 `@bsky/core` 的公共 API 导出，UI 层通过动态 `import('@bsky/core')` 按需加载。

## Dual-mode 翻译：translateText

`translateText` 是整个翻译能力的核心函数，接受两个关键参数：**mode**（`'simple' | 'json'`）控制输出格式；**targetLang** 指定目标语言。

### simple 模式

`simple` 模式返回纯文本翻译结果，适用于嵌入式小窗口或快速预览场景。内部使用 `PF_TRANSLATE_SIMPLE` 提示词：

```text
Translate the following text to {targetLang}. Keep the original meaning,
output only the translation, no explanations.
```

**行为特征：**
- LLM 只回复翻译后的文本，无任何附加说明
- 返回的 `TranslationResult` 只包含 `translated` 字段
- 适用于 UI 中"一键翻译"的轻量场景

[来源](packages/core/src/ai/prompts.ts#L144-L146)

### json 模式

`json` 模式通过 `response_format: { type: 'json_object' }` 约束 LLM 输出为结构化 JSON，返回包含 **源语言检测** 的结果。

`PF_TRANSLATE_JSON` 提示词明确要求输出格式：

```text
You are a translator. Translate the user's text to {targetLang}.
Output valid JSON with these keys:
{"source_lang": "<ISO 639-1 code, use 'und' if unsure>",
"translated": "<the translation>"}.
Output ONLY the JSON object.
```

**行为特征：**
- API 请求体添加 `response_format: { type: 'json_object' }`
- 返回的 `TranslationResult` 包含 `translated` 和 `sourceLang` 两个字段
- `sourceLang` 为 ISO 639-1 语言代码，无法确定时回退为 `'und'`
- JSON 解析失败时会触发重试，若重试耗尽则直接返回原始文本并标记 `sourceLang: 'und'`

[来源](packages/core/src/ai/assistant.ts#L752-L804)
[来源](packages/core/src/ai/prompts.ts#L152-L161)

### 两种模式对比

| 维度 | simple | json |
|------|--------|------|
| 输出格式 | 纯文本 | `{"translated": "...", "source_lang": "..."}` |
| response_format | 无 | `{ type: 'json_object' }` |
| 源语言检测 | 不支持 | 支持（ISO 639-1 代码） |
| 适用场景 | 快速预览、内嵌翻译 | 需要保留源语言信息的结构化场景 |
| 解析失败处理 | 无（直接返回文本） | 重试 → 回退为原始文本 + 'und' |

[来源](packages/core/src/ai/assistant.ts#L725-L818)

## 重试机制

`translateText` 内置了最多 3 次的 **指数退避重试**，应对以下三类失败场景：

1. **空内容** — LLM 返回了空字符串或纯空白内容
2. **JSON 解析失败**（仅 json 模式）— LLM 输出的 JSON 格式不正确或缺少 `translated` 字段
3. **网络/API 错误** — HTTP 状态码非 200，或 `fetch` 抛出异常

**退避策略：**

| 尝试次数 | 等待时间 |
|----------|----------|
| 第 1 次失败 → 第 2 次重试 | 800ms（第一次）或 1000ms（catch 分支） |
| 第 2 次失败 → 第 3 次重试 | 1600ms（第二次）或 2000ms（catch 分支） |
| 第 3 次失败 | 抛出异常或返回退化结果 |

代码路径上有一个 **二阶段退避** 设计：`content.trim()` 为空或 JSON 缺失 `translated` 字段时等待 `800 * (attempt + 1)` 毫秒；`catch` 分支捕获的网络异常等待 `1000 * (attempt + 1)` 毫秒。所有重试耗尽后，json 模式会退化返回 `{ translated: rawContent, sourceLang: 'und' }` 而非崩溃。

## 草稿润色：polishDraft

与翻译不同，润色功能是一个 **单轮调用**，不涉及重试或模式切换。

```typescript
export async function polishDraft(
  config: AIConfig,
  draft: string,
  requirement: string,
  modelOverride?: string
): Promise<string> {
  return singleTurnAI(config, P_POLISH_SYSTEM, PF_POLISH_USER(requirement, draft), 0.7, 2000, modelOverride);
}
```

**参数说明：**
- `draft` — 待润色的帖子草稿正文
- `requirement` — 用户的润色要求，如"更口语化""缩短到 100 字以内"等自然语言描述
- `modelOverride` — 可选，覆盖默认模型

**提示词设计：**

系统提示词 `P_POLISH_SYSTEM` 定义了助手的角色边界：

```
你是一个文字润色助手，根据用户要求调整以下帖子草稿，
只返回润色后的文本。
```

用户提示词 `PF_POLISH_USER` 将要求与草稿拼接：

```
用户要求：{requirement}

草稿：
{draft}
```

关键约束是 **"只返回润色后的文本"**，确保 LLM 不会附加解释、评价或任何元文本，保证了函数返回的字符串可以直接填充到编辑器中。

## React Hook：useTranslation

`useTranslation` 是 `packages/app` 层提供的 React Hook，封装了 `translateText` 的调用逻辑，并加入了 **内存缓存** 以避免重复翻译相同内容。

```typescript
export function useTranslation(
  aiKey: string,
  aiBaseUrl: string,
  aiModel = 'deepseek-v4-flash',
  targetLang: TargetLang = 'zh',
  initialMode: 'simple' | 'json' = 'simple',
)
```

**核心设计：**
- **缓存机制** — 使用 `useState(() => new Map())` 创建惰性初始化的 `Map<string, TranslationResult>`，缓存键为 `{mode}::{lang}::{text}` 的拼接字符串。相同内容的翻译只会在首次命中 API，后续请求直接返回缓存结果
- **按需加载** — `translate` 方法内部通过 `await import('@bsky/core')` 动态导入 `translateText`，避免了主包体积膨胀
- **状态暴露** — 返回 `{ translate, loading, cache, lang, setLang, mode, setMode, LANG_LABELS }`，消费方可以自由切换语言和模式，同时保持 UI 响应状态

[来源](packages/app/src/hooks/useTranslation.ts#L22-L54)

## 兼容性层：translateToChinese

作为便捷方法，`translateToChinese` 是一个无模式的简写——硬编码了 `targetLang='zh'` 和 `mode='simple'`，用于向后兼容。

```typescript
export async function translateToChinese(config: AIConfig, text: string): Promise<string> {
  const result = await translateText(config, text, 'zh', 'simple');
  return result.translated;
}
```
