# 快速开始

两条路径，一个代码库。本文带你分别在 **5 分钟内** 启动 TUI（终端）和 PWA（浏览器）客户端。

---

## 前置条件

- **Node.js >= 18**
- **pnpm**（推荐 `npm i -g pnpm` 安装）
- 一个 **Bluesky 账号**（注册于 [bsky.app](https://bsky.app)）
- 一个 **Bluesky App Password**：进入 Bluesky 设置 → **App Passwords** → 新建一个，权限选择默认即可
- 一个 **LLM API Key**（如 [DeepSeek](https://platform.deepseek.com) / [OpenAI](https://platform.openai.com) / [Mistral](https://console.mistral.ai) 等）

> 如果你只想浏览时间线和发帖，**LLM API Key 不是必须的**——只是 AI 聊天/翻译/润色等功能无法使用。

---

## 路径一：TUI（终端客户端）

TUI 是一个基于 **Ink**（React 终端渲染引擎）的全键盘操作界面，适合键盘党使用。

### 第 1 步：克隆仓库

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
```

### 第 2 步：安装依赖并构建

```bash
pnpm install
pnpm -r build
```

`pnpm -r build` 会依次构建 `@bsky/core` → `@bsky/app` → `@bsky/tui` 三个包。

### 第 3 步：配置环境变量

复制示例环境变量文件：

```bash
cp .env.example .env
```

编辑 `.env`，填入你的 Bluesky 凭证和 LLM API Key：

```env
# Bluesky 账号（必填）
BLUESKY_HANDLE=your-handle.bsky.social
BLUESKY_APP_PASSWORD=your-app-password

# LLM API Key（AI 功能需要）
LLM_API_KEY=sk-your-api-key

# 翻译目标语言（可选，默认 zh）
TRANSLATE_TARGET_LANG=zh
```

此外，TUI 还会读取项目根目录下的 `bsky-tui.config.json`（非凭证设置，如模型、提供商等）。如果文件不存在，会自动使用默认配置。完整配置项见 [`bsky-tui.config.example.json`](../bsky-tui.config.example.json) 和 [环境变量与配置](环境变量与配置.md)。

> **安全提示**：`.env` 和 `bsky-tui.config.json` 包含你的凭据，已列入 `.gitignore`，**不要提交到 Git**。

[来源](../.env.example#L1-L13) · [来源](../bsky-tui.config.example.json#L1-L22)

### 第 4 步：启动 TUI

```bash
cd packages/tui
npx tsx src/cli.ts
```

如果 `.env` 中缺少 `BLUESKY_HANDLE` 或 `BLUESKY_APP_PASSWORD`，TUI 会自动进入 **设置向导（SetupWizard）**，引导你逐步填写 Bluesky 账号、选择 LLM 提供商和模型、输入 API Key 以及选择界面语言。

启动后的界面结构如下：

- **顶部状态栏**：当前视图名称、连接状态
- **主内容区**：时间线、帖子详情、AI 聊天等
- **底部栏**：快捷键提示 + 当前时间
- **raw mode 警告条**（如果终端不兼容 raw mode）

[来源](../packages/tui/src/cli.ts#L54-L149) · [来源](../packages/tui/src/components/SetupWizard.tsx#L1-L302)

### 常见问题：raw mode 不兼容

部分终端（如 Windows 的 cmd.exe）不支持 **raw mode**（逐键输入模式）。程序启动时会自动检测：

```typescript
// src/cli.ts
try {
  if (stdin.isTTY) {
    stdin.setRawMode(true);
    isRawMode = true;
  }
} catch {}
```

如果检测失败，程序会**自动降级**：创建一个模拟的输入流管道，并将原始 `stdin` 数据桥接过去。此时：

- 大部分键盘操作仍可工作
- 鼠标点击不会拦截文本选择
- 屏幕底部会显示橙色警告条：**"当前终端不支持 raw mode。请在 Windows Terminal / iTerm2 中运行。"**

**建议**使用 Windows Terminal、iTerm2 或任何现代终端模拟器以获得最佳体验。

[来源](../packages/tui/src/cli.ts#L114-L149) · [来源](../packages/app/src/i18n/locales/zh.ts#L299)

## 路径二：PWA（浏览器客户端）

PWA 是一个基于 **React DOM + Tailwind CSS** 的浏览器界面，支持离线安装。**无需任何 `.env` 配置**——所有登录和 AI 设置都在浏览器内完成。

### 第 1 步：克隆并安装

如果你已按 TUI 路径完成了克隆和安装，跳过此步。否则：

```bash
git clone https://github.com/epheiamoe/bsky.git
cd bsky
pnpm install
pnpm -r build
```

### 第 2 步：启动开发服务器

```bash
cd packages/pwa
pnpm dev
```

Vite 开发服务器默认在 `http://localhost:5173` 启动，并自动打开浏览器。

### 第 3 步：在浏览器中完成配置

PWA 没有 `.env`，所有配置通过浏览器 UI 完成：

1. **登录**：页面顶部输入你的 Bluesky 句柄（如 `user.bsky.social`）和 App Password
2. **AI 配置**（可选）：进入设置页面，选择 LLM 提供商，填写 API Key 和模型名称
3. **个性化**：切换语言（中文/English/日本語）、切换深色模式等

配置会自动保存到浏览器的 `localStorage`，刷新页面不会丢失。

### 构建生产版本（可选）

```bash
pnpm build
npx wrangler pages deploy dist --project-name ai-bsky --commit-dirty=true
```

生产构建会生成静态文件到 `packages/pwa/dist/`，可部署到 Cloudflare Pages、Netlify 等静态托管平台。详见 [PWA 部署指南](pwa-部署指南.md)。

## 配置对比速查

| 配置项 | TUI 配置方式 | PWA 配置方式 |
|--------|-------------|-------------|
| Bluesky 句柄 | `.env` 文件 | 浏览器 UI 登录 |
| Bluesky App Password | `.env` 文件 | 浏览器 UI 登录 |
| LLM API Key | `.env` 文件 或 `bsky-tui.config.json` | 浏览器 UI 设置页 |
| AI 模型/提供商 | `bsky-tui.config.json` | 浏览器 UI 设置页 |
| 翻译目标语言 | `.env` 或 `bsky-tui.config.json` | 浏览器 UI 设置页 |
| 界面语言 | 自动或 SetupWizard 选择 | 浏览器 UI 设置页 |
| 凭证存储位置 | 本地文件系统 | 浏览器 `localStorage` |

详细配置说明见 [环境变量与配置](环境变量与配置.md)。
