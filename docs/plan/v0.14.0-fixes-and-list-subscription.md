# Plan: v0.14.0 Bug Fixes and List Subscription

## Issues to Fix

### 1. Empty `[@ ]` Card (Bug)
**Problem**: `extractQuotedPost()` returns non-null for list embeds because it only checks `$type === 'app.bsky.embed.record#view'` without verifying the record type. Lists (`app.bsky.graph.defs#listView`) are incorrectly rendered as empty quoted posts showing `@`.

**Fix**: Add type check in `extractQuotedPost` to skip `app.bsky.graph.defs#listView` records.

### 2. `/i/` Redirect Not Working (Bug)
**Problem**: `vite.config.ts` uses `base: './'` (relative paths). When page loads at `/i/...`, browser resolves `./assets/...` to `/i/.../assets/...` which 404s.

**Fix**: Change `base: './'` to `base: '/'` (absolute paths).

### 3. List Subscription (Feature)
**Problem**: Lists cannot be subscribed as timeline sources. Feed config only supports feeds.

**Implementation**:
- Core: Add `getListFeed(listUri, limit?, cursor?)` API
- App: Extend `FeedConfigData` with `subscribedLists: Array<{uri, name}>`
- App: Modify `useTimeline` to support list URIs as source
- PWA: FeedTimeline dropdown shows feeds + subscribed lists
- PWA: ListDetailPage "Subscribe to Timeline" button
- PWA: Settings "Configure Timeline" manages feeds + lists
