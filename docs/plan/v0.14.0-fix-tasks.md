## 修复 6: useModerationBatch 增量解析

**文件**: `packages/app/src/hooks/useModeration.ts`
**问题**: `useEffect` 依赖 `posts`，每次 `loadMore` 后全量重新查询标签
**修复**:
1. 使用 `useRef` 维护跨渲染的 `LabelCache` 实例
2. 使用 ref 维护已处理过的 URI 集合
3. 只对新出现的 URI 调用 `resolveModerationBatch`
4. 合并新的 decisions 和 failedLabelers 到现有结果

**参考实现思路**:
```typescript
const cacheRef = useRef(new LabelCache());
const processedUrisRef = useRef(new Set<string>());
// ...
const newPosts = posts.filter(p => !processedUrisRef.current.has(p.uri));
if (newPosts.length === 0) return;
newPosts.forEach(p => processedUrisRef.current.add(p.uri));
const result = await resolveModerationBatch(newPosts.map(...), config, client, cacheRef.current);
setResult(prev => ({
  decisions: new Map([...prev.decisions, ...result.decisions]),
  failedLabelers: mergeFailedLabelers(prev.failedLabelers, result.failedLabelers),
  isLoading: false,
}));
```

**文件**:

- `packages/pwa/src/components/FeedTimeline.tsx`

- `packages/pwa/src/components/SearchPage.tsx`

- `packages/pwa/src/components/ThreadView.tsx`

**问题**: 所有调用方都没传 `onDismiss`/`onRetry`，用户无法关闭 banner 或手动重试
**修复**:
1. 在每个调用组件中用 `useState` 跟踪 `dismissedLabelerDids`
2. `onDismiss(did)` 将 did 加入 dismissed 集合
3. `onRetry` 调用 `refresh?.()`（重新 moderation batch）
4. 过滤 `failedLabelers` 时排除已 dismiss 的 did
5. 当 `failedLabelers` 变化（有新的 did 或旧的恢复），自动重置 dismissed 状态
